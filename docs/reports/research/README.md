# Research — 技術リサーチ

**用途**: 外部ライブラリ調査、論文サーベイ、市場調査、競合タイトル分析等。

## 入る内容例

- Unity 新パッケージの評価
- 論文（arXiv 等）の SisterGame 適用可能性検討
- 競合 2D アクションゲームの仕様調査

## ファイル形式

- ファイル名: `YYYY-MM-DD_<slug>.md`
- 出典 URL / 出版日 / 結論を冒頭に明記
- 採用判断が出たら関連 Phase の Session 0 読み物リスト（WAVE_PLAN.md）に登録
- 新規作成時は `_registry.md` の research/ セクション最上部に 1 行追加
