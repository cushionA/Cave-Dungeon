# Arch — アーキテクチャ調査・設計検討

**用途**: アーキテクチャ調査結果や設計トレードオフの検討メモ。
**`Architect/` ディレクトリへの昇格候補**を一時的に置く場所。

## 入る内容例

- 新システム導入時のトレードオフ分析（ADR 簡易版）
- アーキテクチャパターン比較
- リファクタリング検討メモ

## ファイル形式

- ファイル名: `YYYY-MM-DD_<slug>.md`
- 採用が確定したら `Architect/` に正式版を移し、本ファイルは削除 or アーカイブ
- 新規作成時は `_registry.md` の arch/ セクション最上部に 1 行追加
