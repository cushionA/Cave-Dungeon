# Bugs — バグ調査の記録

**用途**: `docs/FUTURE_TASKS.md` 登録前段階のバグ調査メモ。
再現手順・原因仮説・関連コード位置を残しておくことで、後で本格対応に入る時に rediscover コストを下げる。

## 入る内容例

- 散発的に出ている怪しい挙動の調査
- ステージング環境で再現したが本番未確認のバグ
- 原因が複数に分散していそうな調査ログ

## ファイル形式

- ファイル名: `YYYY-MM-DD_<slug>.md`
- 本格対応に入ったら FUTURE_TASKS.md にエスカレート、本ファイルは関連 PR 番号付きで残す
- 新規作成時は `_registry.md` の bugs/ セクション最上部に 1 行追加
