# データコンテナ

## SoA（Structure of Arrays）

従来のAoS（Array of Structures）ではキャラクターごとにデータが混在し、キャッシュ効率が悪い。SoAでは同種データを連続配置する。

```
AoS: [HP,MP,Pos], [HP,MP,Pos], [HP,MP,Pos]  ← キャッシュミス多発
SoA: [HP,HP,HP], [MP,MP,MP], [Pos,Pos,Pos]    ← キャッシュヒット率向上
```

## SourceGenerator

情報クラスに属性を付けるだけで、SoAコンテナ・アクセサが自動生成される。

```csharp
// 開発者が書くコード
[SoAContainer]
public class CharacterInfo
{
    public int hp;
    public int mp;
    public Vector3 position;
}

// SourceGeneratorが自動生成するコード
public class SoACharacterInfoContainer
{
    public int[] hp;
    public int[] mp;
    public Vector3[] position;

    public CharacterInfo Get(int index) { ... }
    public void Set(int index, CharacterInfo value) { ... }
}
```

## ハッシュベースアクセス

GameObjectのhashCodeをキーとしてO(1)でデータにアクセス。

```csharp
// 初期化時（Awake/Start）
int hash = gameObject.GetHashCode();

// ランタイム（毎フレーム）
CharacterInfo info = GameManager.Data.GetCharacterInfo(hash);  // O(1)
```

### GetComponent排除

| 従来 | 本設計 |
|------|--------|
| `GetComponent<Health>().hp` | `GameManager.Data.GetCharacterInfo(hash).hp` |
| 毎フレーム検索コスト | O(1) Dictionary参照 |
| コンポーネント依存 | データ指向 |

**ルール**: Awake/Start以外でのGetComponent/FindObjectOfType使用は禁止。

## SoACharaDataDic

全キャラクターデータを管理する中央辞書。

```csharp
public class SoACharaDataDic
{
    // hashCode → 各種情報
    public CharacterInfo GetCharacterInfo(int hashCode);
    public CombatStats GetCombatStats(int hashCode);
    public AIInfo GetAIInfo(int hashCode);

    // 登録・解除
    public void Register(int hashCode, CharacterInfo info);
    public void Unregister(int hashCode);
}
```

GameManagerが唯一のSoACharaDataDicインスタンスを保持し、全システムに提供する。
