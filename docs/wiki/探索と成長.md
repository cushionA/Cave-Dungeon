# 探索と成長

## マップ構成

### レベルストリーミング

- **GameScene（永続）**: プレイヤー・仲間・UI
- **エリアシーン（Additive）**: ステージ固有のコンテンツ
- エリア境界のTriggerでAdditive Scene Loading/Unloading

### マップ仕様（asset-spec.json）

| 項目 | 値 |
|------|-----|
| 画面解像度 | 1920 x 1080 (16:9) |
| Camera Ortho Size | 7（表示高さ14ユニット） |
| PPU | 100 pixels/unit |
| タイルサイズ | 100px = 1.0ユニット |
| チャンクサイズ | 25 x 14 タイル（約1画面分） |
| プレイヤーサイズ | 120 x 180 px (1.2 x 1.8 ユニット) |

### 移動パラメータ

| 項目 | 値 |
|------|-----|
| 重力倍率 | 3.0x |
| 最大ジャンプ高さ | 3.5 タイル |
| 最大ジャンプ距離 | 5 タイル |

## 装備と成長

### 能力値スケーリング

武器はSTR/DEX/INTのAnimationCurveを持つ:

| 能力値 | 対応属性 |
|--------|---------|
| STR | 斬撃・聖 |
| DEX | 刺突・闇 |
| INT | 炎・雷 |

レベルアップ時にセーブポイントで能力値ポイントを振り分け。

### AbilityFlag

拡張移動（壁蹴り等）はAbilityFlagで管理:

- 装備（武器 + 盾 + コア + コンビネーション効果）から合算
- 新能力獲得でバックトラック報酬が連鎖的に解放

## ゲートシステム

4種のゲートで進行を制御:

| GateType | 開放条件 | 永続性 |
|----------|---------|--------|
| Clear | ボス撃破・イベントクリア | グローバルフラグ |
| Ability | AbilityFlag所持 | グローバルフラグ |
| Key | キーアイテム使用 | マップローカルフラグ |
| Elemental | 属性攻撃でヒット | マップローカルフラグ |

### 属性ゲート（環境パズル）

- 7属性すべてに対応
- multiHitRequired: 弱点を数回殴るギミック
- 仲間の属性攻撃でも開放可能（連携パズル）

## セーブシステム

### ISaveableパターン

```csharp
public interface ISaveable
{
    string SaveId { get; }
    object Serialize();
    void Deserialize(object data);
}
```

- SaveManager.Register(ISaveable) で一元管理
- セーブポイントで回復 + セーブ + ファストトラベル

### フラグ管理

| 種別 | 対象 | 管理 |
|------|------|------|
| グローバルフラグ | ストーリー進行、ボス撃破、実績 | 全マップ共有 |
| マップローカルフラグ | イベント再生済み、宝箱開封 | マップ遷移時に自動切替 |

## バックトラック報酬

1. AbilityFlag獲得時にBacktrackRewardManagerが全報酬を再評価
2. 能力獲得前はマップマーカー非表示（ネタバレ防止）
3. AbilityOrb回収 → 新能力 → さらにバックトラック報酬解放（連鎖）
4. エリアごとのScriptableObject（BacktrackRewardTable）で定義

## 経済システム

| 項目 | 仕様 |
|------|------|
| 通貨取得 | 敵撃破・探索 |
| ショップ | 消耗品・装備品の購入・売却 |
| デスペナルティ | 死亡時に所持通貨の20%ロスト |
