# アーキテクチャ概要

## 設計原則

### 1. SoA（Structure of Arrays）+ SourceGenerator

キャラクターデータをフィールドごとの配列で管理し、キャッシュ効率を最大化。

```
従来（AoS）: Character[] → 各キャラにHP, MP, Pos等が混在
本設計（SoA）: HP[], MP[], Pos[] → 同種データが連続配置
```

- SourceGeneratorにより情報クラスに属性を付けるだけでSoAコンテナ・アクセサが自動生成
- 手書きボイラープレートを排除

### 2. GameManager中央ハブ

```csharp
// 全システムへの唯一のアクセス点
GameManager.Data.GetCharacterInfo(hashCode);
GameManager.Events.OnDamageDealt;
```

- 散在するシングルトンを排除
- 依存関係を一元管理
- IGameSubManagerインターフェースで初期化順序を制御

### 3. ハッシュベースO(1)データアクセス

```csharp
// GameObjectのhashCodeをキーとしてO(1)アクセス
int hash = gameObject.GetHashCode();
CharacterInfo info = GameManager.Data.GetCharacterInfo(hash);
```

- GetComponent/FindObjectOfTypeの毎フレーム検索コストを完全排除
- Awake/Start以外でのGetComponent使用を禁止

### 4. Ability拡張（コンポーネントパターン）

```csharp
public interface IAbility
{
    AbilityFlag Flag { get; }
    void Initialize();
    void Execute();
}
```

- ベースキャラクターは最小限の責務のみ
- ジャンプ・ダッシュ・壁蹴り等はAbilityコンポーネントとして追加
- 新行動の追加が既存コードに影響しない

### 5. データ駆動（ScriptableObject）

- ゲームバランスに関わる値はすべてScriptableObject上で定義
- ハードコーディング禁止。Inspectorから調整可能にする

## レイヤー構成

```
[Input Layer]        — 入力受付、イベント発行
    ↓ events
[Game Logic Layer]   — メカニクス、ルール、状態管理（純粋C#クラス）
    ↓ data (SoAコンテナ経由)
[Presentation Layer] — MonoBehaviour統合、表示、アニメーション、サウンド
```

- 上位レイヤーは下位を知らない
- 下位レイヤーはイベント購読で上位の変化を受け取る
- Game Logic Layerは完全にMonoBehaviour非依存（テスタビリティ確保）

## Assembly構成

```
Game.Core（共通型・インターフェース・基盤ロジック）
    ↑
GameCode（Character, Combat, AI, World, Economy, UI）
    ├── Unity.InputSystem
    ├── UniTask
    ├── R3.Unity
    └── LitMotion
```

- 循環参照禁止。上位は下位を参照しない
- テスト用asmdefはGame.Coreを参照

## IGameSubManager パターン

```csharp
public interface IGameSubManager
{
    int InitOrder { get; }
    void Initialize(SoACharaDataDic data, GameEvents events);
    void Dispose();
}
```

- GameManagerCoreが全SubManagerをInitOrder順に初期化
- 各マネージャーはこのインターフェースを実装

## コルジエンジンからの移行

| Corgi Engine | 本設計 |

| CorgiController | BaseCharacter + Ability |
| CharacterAbility（継承） | IAbility（インターフェース） |
| Health | DamageReceiver + SoAコンテナ |
| AIBrain（Corgi） | AIBrain + AIInfo（3層判定） |
| InventoryEngine | 独自InventorySystem |
