# 戦闘システム

## 属性システム（7属性統一）

物理タイプも含めてElement enumに統合。`[Flags] byte` で複合属性を表現。

| 属性 | スケーリング能力値 | 分類 |
|------|-------------------|------|
| Slash（斬撃） | STR | 物理 |
| Strike（打撃） | STR | 物理 |
| Pierce（刺突） | DEX | 物理 |
| Fire（炎） | INT | 魔法 |
| Thunder（雷） | INT | 魔法 |
| Light（聖） | STR | 魔法 |
| Dark（闇） | DEX | 魔法 |

### ElementalStatus

7属性別のint値で攻撃力・防御力を管理:

```csharp
public struct ElementalStatus
{
    public int slash, strike, pierce, fire, thunder, light, dark;
}

public struct CombatStats
{
    public ElementalStatus attack;
    public ElementalStatus defense;
}
```

## ダメージ計算

### 基本式

各属性チャネルごとに計算し合算:

```
channelDamage = (atk² × motionValue) / (atk + def)
totalDamage = Σ channelDamage（全7属性）
```

### DamageResult

```csharp
public struct DamageResult
{
    public int totalDamage;
    public int armorDamage;
    public bool isArmorBroken;
    public bool isCritical;
    public Element attackElement;
}
```

### ガード

- **通常ガード**: 属性別カット率（slashCut, strikeCut, ...）で軽減
- **ジャストガード**: タイミング合わせで大幅軽減 + justGuardResistance でアーマー削り軽減
- **ガードブレイク**: アーマー値が0になると発生
- **JG不可攻撃**: `AttackFeature.JustGuardImmune` フラグ

## コンボシステム

### AttackMotionData

各モーションに以下を定義:

| フィールド | 説明 |
|-----------|------|
| motionValue | ダメージ倍率 |
| isAutoChain | 入力なしで次モーションへ自動遷移 |
| maxHitCount | モーションごとの最大ヒット数 |

### ActionExecType

全行動を5分類で統一管理:

| Type | 説明 | 例 |
|------|------|-----|
| Attack | ヒットボックス・コンボ管理 | 弱攻撃、強攻撃 |
| Cast | 詠唱→発動→飛翔体 | 魔法、チャージ攻撃 |
| Instant | アニメ1回再生 | 回避、アイテム使用 |
| Sustained | 開始→Tick→終了条件 | ガード、追従 |
| Broadcast | 他キャラAI状態操作 | ターゲット指示、挑発 |

## 装備システム

### スロット構成

| スロット | 部位 | 説明 |
|---------|------|------|
| 武器 | 右手 | メイン攻撃手段 |
| 盾 | 左手 | ガード + 盾スキル |
| コア | 1枠 | 魔物の核。基礎補正 + 特殊効果 |

### GripMode

| モード | 盾 | 武器スキル |
|--------|-----|-----------|
| OneHanded | 併用可 | 盾スキル優先 |
| TwoHanded | 無効化 | 武器スキル使用可 |

### 重量システム

```
weightRatio = totalEquipWeight / maxWeightCapacity
```

AnimationCurveで性能変動（回避速度、スタミナ回復、攻撃速度、回避距離）。過負荷（1.0超過）も可能だが大ペナルティ。

## キャラクター当たり判定

### 衝突モード（AttackContactType）

| モード | 動作 | レイヤー |
|--------|------|---------|
| PassThrough | すり抜け（通常時） | CharaPassThrough (12) |
| StopOnHit | 物理衝突で止まる | CharaCollide (13) |
| Carry | 衝突＋条件付き運搬 | CharaCollide (13) |

`gameObject.layer` の切替だけで制御。`Physics2D.IgnoreCollision`のループは使わない。

### 運搬条件

Carry接触時、対象が**敵**かつ以下のActStateの場合のみ運搬開始:
- Flinch / Stunned / GuardBroken / Knockbacked

詳細: [物理レイヤー定義](物理レイヤー定義)

## 状態異常

蓄積モデル: ヒットごとに蓄積値が増加 → 閾値超過で発症。

- **混乱**: 陣営フラグ反転（Enemy → Ally）。ボスは完全耐性
- 蓄積はStatusEffectManager.AccumulateEffectで管理
