# アセット管理

## Addressables基本方針

- すべてのゲームアセットはAddressableで管理
- ランタイム参照は `AssetReference` / `AssetReferenceT<T>` を使用
- `AssetDatabase` はEditor専用コードのみ許可
- 直接参照（Inspector drag & drop）はシーン内常駐オブジェクト間のみ

## アドレス命名規則

```
[種別]/[カテゴリ]/[アセット名]
```

| プレフィックス | 対象 | 例 |
|---------------|------|-----|
| sprite | スプライト | `sprite/player/idle` |
| anim | アニメーション | `anim/player/run` |
| sfx | SE | `sfx/attack/slash` |
| bgm | BGM | `bgm/stage/forest` |
| prefab | プレハブ | `prefab/enemy/slime` |
| tile | タイル | `tile/terrain/grass` |

- すべて小文字、区切りは `/`
- 単語間はハイフン `-`
- ファイルパスではなく論理名を使用

## グループ設計

| グループ | パッキング | ロードタイミング |
|---------|-----------|----------------|
| Core | Pack Together | 起動時プリロード |
| Player | Pack Together | タイトル画面先行ロード |
| Enemies | Pack Separately | ステージロード時 |
| Stage_[ID] | Pack Together | ステージ選択時 |
| UI | Pack Separately | 画面遷移時 |
| Audio_BGM | Pack Separately | シーン遷移時 |
| Audio_SFX | Pack Together | 起動時プリロード |
| Placeholder | Pack Together | 開発中のみ |

## ラベル体系

### 用途別

| ラベル | ビルド除外 |
|--------|-----------|
| preload | No |
| on-demand | No |
| placeholder | Yes（リリース時） |
| debug | Yes（リリース時） |

### コンテンツ種別

sprite, animation, audio-bgm, audio-sfx, prefab, tilemap, material, timeline

### ゲームコンテキスト

stage-[ID], event-[ID], boss, common

## プレースホルダー規約

### ルール
- アセット未準備の場合は `[PLACEHOLDER]` プレフィックスで仮実装
- `PlaceholderAsset` コンポーネントを付与
- Sprite: マゼンタ色四角形 / Audio: 無音クリップ

### 差替え手順

```
1. アセットをディレクトリ規則に従って配置
2. Addressableに登録
3. /bind-assets 実行
4. PlaceholderAssetが自動検出・参照差替え
5. [PLACEHOLDER] プレフィックス除去
6. PlaceholderAssetコンポーネント削除
```

## アセット配置ディレクトリ

```
Assets/
├── Sprites/[カテゴリ]/
├── Animations/[カテゴリ]/
├── Audio/BGM/ , Audio/SFX/
├── Materials/[カテゴリ]/
└── Prefabs/[カテゴリ]/
```

## ロード・解放の原則

- ロードしたアセットは必ず解放（参照カウント管理）
- `OnDestroy()` / `OnDisable()` で解放処理
- `InstantiateAsync()` → `ReleaseInstance()` で破棄
