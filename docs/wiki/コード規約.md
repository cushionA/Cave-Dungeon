# コード規約

## 命名規則

| 対象 | スタイル | 例 |
|------|---------|-----|
| クラス名 | PascalCase | `PlayerMovement` |
| インターフェース | I + PascalCase | `IDamageable` |
| メソッド名 | PascalCase | `CalculateDistance()` |
| プロパティ | PascalCase | `MaxHealth` |
| publicフィールド | PascalCase | `MovementSpeed` |
| ローカル変数・引数 | camelCase | `moveDirection` |
| privateフィールド | _camelCase | `_currentHealth` |
| 定数 | k_ + PascalCase | `k_MaxSpeed` |
| SerializeField | camelCase | `[SerializeField] private float moveSpeed` |
| enum名 | PascalCase（単数形） | `WeaponType` |
| enumフラグ | PascalCase（複数形） | `[Flags] AttackModes` |
| 名前空間 | PascalCase | `MyGame.PlayerSystems` |

### 命名の原則

- 変数名は名詞: `playerScore`, `targetObject`
- bool値は動詞で始める: `isAlive`, `hasWeapon`, `canJump`
- メソッドは動詞: `FireWeapon()`, `CalculateDistance()`
- bool返却メソッドは疑問文: `IsPlayerAlive()`, `HasAmmo()`
- イベント名は動詞句: `PointsScored`, `DoorOpened`

## コンポーネント設計

- 1コンポーネント = 1責務（Single Responsibility）
- `[SerializeField] private` を使用（publicフィールド禁止）
- `[Header("Section")]` でInspector設定値をグループ化
- `[RequireComponent]` で依存コンポーネントを明示
- `GetComponent` は `Awake`/`Start` でキャッシュ

## クラス構成順序

```csharp
public class Example : MonoBehaviour
{
    // 1. Fields
    // 2. Properties
    // 3. Events / Delegates
    // 4. MonoBehaviour Methods (Awake, Start, OnEnable, ...)
    // 5. Public Methods
    // 6. Private Methods
}
```

## MonoBehaviourライフサイクル

```
Awake()     → コンポーネント参照の取得
OnEnable()  → イベント登録
Start()     → 初期化ロジック
Update()    → 毎フレーム処理
OnDisable() → イベント解除
OnDestroy() → リソース解放
```

## パフォーマンス規約

### Update内のアロケーション禁止

```csharp
// NG
void Update() { List<GameObject> temp = new List<GameObject>(); }

// OK
private List<GameObject> _reusable = new List<GameObject>();
void Update() { _reusable.Clear(); /* 再利用 */ }
```

### その他

- Update内での `string +` 連結禁止 → `StringBuilder` 使用
- `Vector3.Distance` → `sqrMagnitude` 比較を検討
- `obj.tag == "Player"` → `obj.CompareTag("Player")`
- enum型は値が少ない場合 `byte` 型指定
- `var` 不使用（型を明示）

## フォーマット

- インデント: スペース4つ
- 中括弧: Allmanスタイル（新しい行に開き括弧）
- 1行でも中括弧を付ける
- 1ファイル = 1クラス
