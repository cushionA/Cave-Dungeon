# 開発ワークフロー

## TDD駆動開発

### 基本フロー

```
1. テストファイル作成（Red）
2. テスト全Fail確認
3. 実装コード作成
4. テスト全Pass確認（Green）
5. feature-db に完了記録
```

### テスト命名規則

```
[機能名]_[条件]_[期待結果]
```

例:
- `PlayerMovement_WhenSpeedIsZero_ShouldNotMove`
- `HealthSystem_WhenDamageTaken_ShouldReduceHealth`

### テスト配置

| 種別 | パス | 用途 |
|------|------|------|
| EditMode | `Tests/EditMode/[機能名]Tests.cs` | ロジック単体テスト |
| PlayMode | `Tests/PlayMode/[機能名]PlayTests.cs` | MonoBehaviour連携テスト |
| 結合テスト | `Tests/EditMode/Integration_[名前]Tests.cs` | クロスシステムテスト |

### 結合テスト必須観点

1. **既存ロジック呼び出し検証**: 新コードが既存ユーティリティを正しく経由しているか
2. **状態シーケンス検証**: 同じ操作を複数回行った時に状態が壊れないか
3. **境界値・不変条件検証**: HP < 0 にならない、リソースリークしない等

## feature-db（機能管理）

SQLiteベースの機能追跡データベース。

```bash
# 機能追加
python tools/feature-db.py add "機能名" --tests test1.cs test2.cs --impl impl1.cs

# ステータス更新
python tools/feature-db.py update "機能名" --status complete --test-passed 5 --test-failed 0

# 一覧・サマリー
python tools/feature-db.py list [--status complete|in_progress|failed]
python tools/feature-db.py summary
```

## AI支援パイプライン

Claude Codeのスキルによる対話型開発パイプライン。

### パイプライン全体フロー

```
/design-game         ゲーム設計（GDD作成）
    ↓
/design-systems      システム設計 + 機能分解 + feature-db登録（セクション単位。旧 plan-sprint 統合）
    ↓
/create-feature      機能実装（1機能ずつTDD）
    ↓
/run-tests           テスト実行・確認
```

### 主要スキル一覧

| スキル | 用途 |
|--------|------|
| `/build-pipeline` | 全工程を対話しながら自動進行 |
| `/design-game` | 対話型GDD作成 |
| `/design-systems [section]` | システム設計書 + 機能分解 + feature-db登録 |
| `/create-feature` | 1機能TDD実装 |
| `/run-tests` | テスト実行 |
| `/simplify` | コードレビュー（品質・効率・再利用） |
| `/design-stage` | 2Dステージ設計 |
| `/create-event` | イベントシーン構築 |
| `/generate-assets` | 画像生成 + 音声マッチング + 音声インデックス管理 |
| `/debug-assist` | MCP経由デバッグ支援 |

### セクション単位の進行

```
Section 1 設計 → 計画 → 実装 → 完了
    ↓
Section 2 設計 → 計画 → 実装 → 完了
    ↓
Section 3 ...
```

各セクション完了時にゲームとして動作確認可能。

## ディレクトリ規約

| パス | 内容 |
|------|------|
| `Assets/MyAsset/Core/` | ゲームロジック（MonoBehaviour非依存） |
| `Assets/MyAsset/Runtime/` | MonoBehaviour統合レイヤー |
| `Assets/MyAsset/UI/` | UI関連 |
| `Assets/Tests/EditMode/` | 単体・結合テスト |
| `Assets/Tests/PlayMode/` | ゲームプレイテスト |
| `designs/systems/` | システム設計書 |
| `designs/sprints/` | スプリント計画 |
| `Architect/` | アーキテクチャ設計文書 |
