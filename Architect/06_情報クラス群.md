# 06 - 情報クラス群

## 1. 概要

本ドキュメントでは、ゲーム内キャラクターの定義に使用する **3つの情報クラス** を定義する。
いずれも `ScriptableObject` ベースであり、エディタ上でアセットとして作成・編集する。
実行時にはこれらのデータが **SourceGenerator** によって生成された SoA データコンテナに展開され、
`GameManager.Data`（`SoACharaDataDic`）経由で高速にアクセスされる。

| 情報クラス | 役割 | コンテナ展開先 |
|---|---|---|
| **CharacterInfo** | キャラクターの識別・能力定義 | SoACharaDataDic 内の複数構造体 |
| **AttackInfo** | 攻撃・魔法の完全定義 | AttackInfoContainer（別コンテナ） |
| **AIInfo** | AI行動パターンの設定 | SoACharaDataDic（フラット化AI判定データ） |

```
CharacterInfo → CharacterVitals, CombatStats, CharacterFlags, MoveParams,
                EquipmentStatus, CharacterStatusEffects
AttackInfo    → AttackInfoContainer（別コンテナ。武器/魔法定義用）
AIInfo        → SoACharaDataDic（フラット化されたAI判定データ）
```

`BaseCharacter` は objectHash（`int`）を保持し、
このハッシュを使って `GameManager.Data`（`SoACharaDataDic`）から自身のデータにアクセスする。

## 2. CharacterInfo（キャラクター情報）

キャラクターの **「何者であるか」** と **「何ができるか」** を定義する ScriptableObject。
ボス・雑魚・NPC・プレイヤーなど、すべてのキャラクターがこのクラスで定義される。

```csharp
[CreateAssetMenu(fileName = "NewCharacterInfo", menuName = "Game/CharacterInfo")]
public class CharacterInfo : ScriptableObject
{
    // ─────────────────────────────────────────────
    //  キャラクター識別
    // ─────────────────────────────────────────────
    public CharacterFeature feature;      // Player / Boss / NPC / Minion 等（[Flags]）
    public CharacterBelong belong;        // 陣営（味方・敵・中立、[Flags]）
    public CharacterRank rank;            // 強さランク（AI脅威度評価に使用）
    public bool canFly;                   // 飛行可能か
    public int targetingLimit;            // 同時にターゲットされる上限数

    // ─────────────────────────────────────────────
    //  基礎ステータス
    // ─────────────────────────────────────────────
    public int maxHp;                     // 最大HP
    public int maxMp;                     // 最大MP
    public float maxStamina;              // 最大スタミナ
    public float staminaRecoveryRate;     // スタミナ回復速度（/秒）
    public float staminaRecoveryDelay;    // 消費後の回復開始までの遅延（秒）

    // ─────────────────────────────────────────────
    //  属性攻撃力・防御力（7属性: 斬撃/打撃/刺突/炎/雷/聖/闇）
    // ─────────────────────────────────────────────
    public ElementalStatus baseAttack;    // 7属性の基礎攻撃力
    public ElementalStatus baseDefense;   // 7属性の基礎防御力

    // ─────────────────────────────────────────────
    //  弱点・攻撃属性
    // ─────────────────────────────────────────────
    public Element weakPoint;             // 弱点属性（[Flags] で複数指定可）
    public Element attackElement;         // メイン攻撃属性

    // ─────────────────────────────────────────────
    //  アクション設定
    // ─────────────────────────────────────────────
    public float moveSpeed;               // 通常移動速度
    public float walkSpeed;               // 歩行速度
    public float dashSpeed;               // ダッシュ速度
    public float jumpHeight;              // ジャンプ力（高さ）

    // ─────────────────────────────────────────────
    //  アーマー
    // ─────────────────────────────────────────────
    public float maxArmor;                // ベースアーマー最大値
    public float armorRecoveryRate;       // アーマー自然回復速度（/秒）
    public float armorRecoveryDelay;      // 被弾後のアーマー回復開始までの遅延（秒）

    // ─────────────────────────────────────────────
    //  耐性
    // ─────────────────────────────────────────────
    public float statusResistance;        // 基礎状態異常耐性（蓄積閾値のグローバル倍率）
    public PhysicalResistance physicalResistance; // 物理タイプ別耐性（斬撃/打撃/刺突）

    // ─────────────────────────────────────────────
    //  初期状態
    // ─────────────────────────────────────────────
    public ActState initialActState;      // スポーン時の初期行動状態
}
```

> **注意**: 旧版にあった初期装備フィールド（`initialWeapon`, `initialShield`, `initialAccessory`）は現在の実装には存在しない。装備は `EquipmentStatus` 構造体で管理され、ランタイムに設定される。

### 2.1 関連 Enum 定義

```csharp
/// <summary>キャラクターの種別（[Flags] でビット演算によるグループ判定に対応）</summary>
[Flags]
public enum CharacterFeature : ushort
{
    Player    = 1 << 0,
    Companion = 1 << 1,
    Summon    = 1 << 2,
    NPC       = 1 << 3,
    Minion    = 1 << 4,
    MiniBoss  = 1 << 5,
    Boss      = 1 << 6,
    Ghost     = 1 << 7,
    Flying    = 1 << 8,
}

/// <summary>陣営（[Flags] でビット演算による同陣営判定に対応）</summary>
[Flags]
public enum CharacterBelong : byte
{
    Ally    = 1 << 0,
    Enemy   = 1 << 1,
    Neutral = 1 << 2,
}

/// <summary>強さランク（AI脅威度評価に使用）</summary>
public enum CharacterRank : byte
{
    F, E, D, C, B, A, S, SS,
}

/// <summary>
/// 属性の種類（物理3種 + 魔法4種 = 7属性）。
/// [Flags] で弱点属性の複数指定やビット演算に対応。
/// 旧版の {None,Fire,Water,Wind,Earth,Light,Dark} から変更。
/// </summary>
[Flags]
public enum Element : byte
{
    None    = 0,
    Slash   = 1 << 0,  // 斬撃
    Strike  = 1 << 1,  // 打撃
    Pierce  = 1 << 2,  // 刺突
    Fire    = 1 << 3,  // 炎
    Thunder = 1 << 4,  // 雷
    Light   = 1 << 5,  // 聖
    Dark    = 1 << 6,  // 闇
}
```

### 2.2 ElementalStatus（属性ステータス構造体）

7属性（斬撃/打撃/刺突/炎/雷/聖/闇）それぞれの値を保持する構造体。
攻撃力・防御力・ダメージの各チャネルで共用する。

```csharp
[Serializable]
public struct ElementalStatus
{
    public int slash;     // 斬撃
    public int strike;    // 打撃
    public int pierce;    // 刺突
    public int fire;      // 炎
    public int thunder;   // 雷
    public int light;     // 聖
    public int dark;      // 闇

    public int Total => slash + strike + pierce + fire + thunder + light + dark;
    public int PhysicalTotal => slash + strike + pierce;
    public int MagicalTotal => fire + thunder + light + dark;

    public int Get(Element element) => element switch
    {
        Element.Slash   => slash,
        Element.Strike  => strike,
        Element.Pierce  => pierce,
        Element.Fire    => fire,
        Element.Thunder => thunder,
        Element.Light   => light,
        Element.Dark    => dark,
        _               => 0,
    };
}
```

### 2.3 SoA コンテナへのマッピング

`CharacterInfo` の各フィールドは、実行時に以下の構造体へ分解・格納される。
SourceGenerator がこのマッピングコードを自動生成する。

| CharacterInfo フィールド | 格納先構造体 | 説明 |
|---|---|---|
| `maxHp`, `maxMp`, `maxStamina`, `staminaRecoveryRate`, `staminaRecoveryDelay`, `maxArmor` | **CharacterVitals** | HP/MP/スタミナ/アーマー/位置 |
| `baseAttack`, `baseDefense` | **CombatStats** | 7属性攻撃力+防御力+ガードステータス |
| `feature`, `belong`, `initialActState` 等 | **CharacterFlags** (ulong) | 全状態フラグをビットパック |
| `moveSpeed`, `walkSpeed`, `dashSpeed`, `jumpHeight` | **MoveParams** | 移動パラメータ |
| （ランタイム設定） | **EquipmentStatus** | 装備由来のステータス合計 |
| （ランタイム蓄積） | **CharacterStatusEffects** | 状態異常蓄積値・アクティブスロット |

> **構造体統合の詳細**: [07_AI判定システム再設計](./07_AI判定システム再設計.md) セクション6 参照

```csharp
// SourceGenerator が生成するマッピングコードのイメージ
public static void MapCharacterInfo(CharacterInfo info, int index, SoACharaDataDic data)
{
    data.vitals[index] = new CharacterVitals
    {
        maxHp               = info.maxHp,
        currentHp           = info.maxHp,
        maxMp               = info.maxMp,
        currentMp           = info.maxMp,
        maxStamina          = info.maxStamina,
        currentStamina      = info.maxStamina,
        maxArmor            = info.maxArmor,
        currentArmor        = info.maxArmor,
        staminaRecoveryRate = info.staminaRecoveryRate,
        staminaRecoveryDelay = info.staminaRecoveryDelay,
        hpRatio             = 100,
        mpRatio             = 100,
    };

    data.combatStats[index] = new CombatStats
    {
        attack  = info.baseAttack,
        defense = info.baseDefense,
    };

    // CharacterFlags は ulong にビットパック
    data.flags[index] = CharacterFlags.Pack(
        info.belong, info.feature, info.initialActState,
        AbilityFlag.None
    );

    data.moveParams[index] = new MoveParams
    {
        moveSpeed  = info.moveSpeed,
        dashSpeed  = info.dashSpeed,
        jumpForce  = info.jumpHeight,
    };
}
```

### 2.4 AI 認識用データ

`CharacterFlags`（ulongビットパック）にAIがターゲットを評価する際に必要な情報が集約されている。
AI判定はこのフラグデータを直接参照して判断を行う。

| CharacterFlags ビット | AI での用途 |
|---|---|
| `Feature` (bits 9-17) | ボスかどうかの判定、優先ターゲット選択 |
| `Belong` (bits 0-2) | 敵味方判定（攻撃対象の決定） |
| `ActState` (bits 3-8) | 行動状態の判定（攻撃中・ガード中等） |
| `BrainEventFlags` (bits 36-43) | AIイベントフラグ |
| `AbilityFlags` (bits 44-59) | 能力フラグ（壁蹴り・二段ジャンプ等） |

> **注意**: 旧版の `SolidData` 構造体は廃止され、これらの情報は `CharacterFlags` のビットフィールドに統合された。`rank`, `canFly`, `targetingLimit` 等の情報は `CharacterInfo` SO から直接参照するか、`CharacterFlags.Feature` のビット（`Flying` 等）で判定する。

## 3. AttackInfo（攻撃情報）

攻撃・魔法・スキルを **1つのアセットで完全に定義** する ScriptableObject。
発動前モーション、発動時モーション、溜め時間、使用エフェクト、サウンド、
飛翔体の軌道、吹き飛ばし、状態異常など、攻撃に必要なすべての情報を保持する。

```csharp
[CreateAssetMenu(fileName = "NewAttackInfo", menuName = "Game/AttackInfo")]
public class AttackInfo : ScriptableObject
{
    // ─────────────────────────────────────────────
    //  基本情報
    // ─────────────────────────────────────────────
    [Header("基本情報")]
    public string attackName;             // 攻撃名（表示用）
    public AttackCategory category;       // 近接 / 遠距離 / 魔法 / スキル

    // ─────────────────────────────────────────────
    //  モーション
    // ─────────────────────────────────────────────
    [Header("モーション")]
    public MotionInfo motionInfo;         // 発動前・発動時・硬直・溜めのモーション設定

    // ─────────────────────────────────────────────
    //  ダメージ
    // ─────────────────────────────────────────────
    [Header("ダメージ")]
    public ElementalStatus baseDamage;    // 基礎属性別ダメージ（ElementalStatus で統一）
    public float damageMultiplier;        // 武器攻撃力への倍率（motionValue）

    // ─────────────────────────────────────────────
    //  エフェクト・サウンド
    // ─────────────────────────────────────────────
    [Header("エフェクト・サウンド")]
    public EffectSoundInfo effectSoundInfo;

    // ─────────────────────────────────────────────
    //  飛翔体
    // ─────────────────────────────────────────────
    [Header("飛翔体")]
    public ProjectileInfo projectileInfo;

    // ─────────────────────────────────────────────
    //  吹き飛ばし
    // ─────────────────────────────────────────────
    [Header("吹き飛ばし")]
    public KnockbackInfo knockbackInfo;

    // ─────────────────────────────────────────────
    //  状態異常
    // ─────────────────────────────────────────────
    [Header("状態異常")]
    public StatusEffectInfo statusEffectInfo;

    // ─────────────────────────────────────────────
    //  コスト
    // ─────────────────────────────────────────────
    [Header("コスト")]
    public float mpCost;                  // MP消費量
    public float staminaCost;             // スタミナ消費量

    // ─────────────────────────────────────────────
    //  AI認識用
    // ─────────────────────────────────────────────
    [Header("AI認識用")]
    public AIAttackEvaluation aiEvaluation;  // AI判断に使用する攻撃評価データ

    // ─────────────────────────────────────────────
    //  攻撃移動
    // ─────────────────────────────────────────────
    public float attackMoveDistance;           // 攻撃中の移動距離
    public float attackMoveDuration;          // 攻撃移動の持続時間
    public AttackContactType contactType;     // PassThrough / StopOnHit / Carry

    // ─────────────────────────────────────────────
    //  パリィ・ガード
    // ─────────────────────────────────────────────
    public bool isParriable;                  // パリィ可能か
    public float armorBreakValue;             // アーマー削り値

    // ─────────────────────────────────────────────
    //  属性
    // ─────────────────────────────────────────────
    public Element attackElement;             // 攻撃属性（[Flags]）

    // ─────────────────────────────────────────────
    //  コンボ
    // ─────────────────────────────────────────────
    public bool isAutoChain;                  // 自動連鎖するか
    public bool isChainEndPoint;              // コンボ終端か
    public float inputWindow;                 // コンボ入力受付時間
    public float justGuardResistance;         // ジャストガード耐性（0-100）
}
```

### 3.2 MotionInfo（モーション情報）

攻撃の時間軸を完全に定義する構造体。
発動前モーション → 溜め（オプション） → 発動時モーション → 硬直 の流れを制御する。

```csharp
[System.Serializable]
public struct MotionInfo
{
    // ─── タイミング ───
    public float preMotionDuration;       // 発動前モーション時間（秒）
    public float activeMotionDuration;    // 発動時モーション時間（秒）
    public float recoveryDuration;        // 硬直時間（秒）

    // ─── 溜め（チャージ）───
    public float chargeTimeMin;           // 最小溜め時間（0 = 溜め無し）
    public float chargeTimeMax;           // 最大溜め時間
    public float chargeDamageMultiplier;  // 最大溜め時のダメージ倍率

    // ─── アニメーション ───
    public AnimationClip preMotionClip;   // 発動前アニメーション
    public AnimationClip activeClip;      // 発動時アニメーション
    public AnimationClip chargeLoopClip;  // 溜めループアニメーション（null=溜め無し）
    public AnimationClip recoveryClip;    // 硬直アニメーション（null=デフォルト）
}
```

**攻撃のタイムライン:**

```
[発動前モーション] → [溜め（任意）] → [発動時モーション] → [硬直]
 preMotionDuration   chargeTime       activeMotionDuration  recoveryDuration
 preMotionClip       chargeLoopClip   activeClip            recoveryClip
```

- `chargeTimeMin == 0` の場合、溜めフェーズはスキップされる
- 溜め中はプレイヤー入力で溜め時間を制御できる（AI は最適な溜め時間を判断する）
- 溜め時間に応じて `chargeDamageMultiplier` が線形補間される

### 3.5 ProjectileInfo（飛翔体情報）

遠距離攻撃・魔法の弾道を定義する構造体。
`hasProjectile == false` の場合、他のフィールドは無視される（近接攻撃等）。

```csharp
[System.Serializable]
public struct ProjectileInfo
{
    public bool hasProjectile;            // 飛翔体の有無
    public GameObject projectilePrefab;   // 飛翔体 Prefab
    public BulletMoveType trajectory;     // 弾道の種類
    public float speed;                   // 飛翔速度
    public float lifetime;                // 生存時間（秒、超過で自動消滅）
    public float homingStrength;          // 追尾強度（Homing 型のみ有効）
    public int pierceCount;               // 貫通回数（0 = 貫通なし、最初の対象で消滅）
    public float aoeRadius;               // 着弾時の範囲（0 = 単体攻撃）
    public int projectileCount;           // 同時発射数（1 = 単発）
    public float spreadAngle;             // 複数発射時の拡散角度（度）
}
```

### 3.7 KnockbackInfo（吹き飛ばし情報）

攻撃ヒット時の吹き飛ばし・ヒットストップを定義する構造体。

```csharp
[Serializable]
public struct KnockbackInfo
{
    public bool hasKnockback;             // 吹き飛ばしの有無
    public Vector2 force;                 // 吹き飛ばし力（方向+強さをVector2で定義）
    public float stunDuration;            // ヒットストップ時間（秒）
    public bool groundBounce;             // 地面バウンドするか
    public bool wallBounce;               // 壁バウンドするか
    public float gravityScale;            // 吹き飛ばし中の重力スケール（1.0 = 通常）
}
```

**吹き飛ばし方向の処理:**

```
攻撃者が右向き: direction をそのまま使用
攻撃者が左向き: direction.x を反転

例: direction = (1.0, 0.5) の場合
  右向き攻撃 → (1.0, 0.5)  = 右上に吹き飛ばし
  左向き攻撃 → (-1.0, 0.5) = 左上に吹き飛ばし
```

### 3.8 StatusEffectInfo（状態異常情報 — 蓄積モデル対応）

攻撃に付随する状態異常の定義。旧 `SpecialEffect` enum + 確率判定方式を廃止し、
**蓄積モデル**に対応した定義に変更。詳細は 04_ダメージ・被弾 §8 参照。

```csharp
/// <summary>
/// 攻撃に付随する状態異常情報（蓄積モデル対応版）。
/// 1ヒットあたりの蓄積量を定義し、閾値超過で発症する。
/// </summary>
[System.Serializable]
public struct StatusEffectInfo
{
    public StatusEffectId effect;           // 付与する状態異常
    public float accumulateValue;           // 1ヒットあたりの蓄積量
    public float duration;                  // 発症時の持続時間（秒）
    public float tickDamage;               // tick毎のダメージ量（DoT用）
    public float tickInterval;             // tick の間隔（秒, DoT用）
    public float modifier;                 // 効果量（速度低下率、被ダメ増加率等）
    public byte maxStack;                  // 最大スタック数（0=スタック不可）
}
```

### 3.9 StatusEffectId（状態異常種別 — 旧 SpecialEffect を置換）

```csharp
/// <summary>
/// 状態異常ID。ビットフラグではなく個別IDで管理する。
/// 04_ダメージ・被弾 §8.2 で定義。
/// </summary>
public enum StatusEffectId : byte
{
    None = 0,

    // 持続ダメージ系 (DoT)
    Poison,       // 毒: 毎秒固定ダメージ
    Burn,         // 炎上: 毎秒固定ダメージ + 防御低下
    Bleed,        // 出血: 移動時に追加ダメージ

    // 行動阻害系
    Stun,         // スタン: 完全行動不能（短時間）
    Freeze,       // 凍結: 移動不能 + 被ダメージ増加
    Paralyze,     // 麻痺: 確率で行動失敗
    Slow,         // 鈍足: 移動速度低下

    // 能力低下系
    Blind,        // 暗闇: AI索敵範囲低下
    Silence,      // 沈黙: 魔法使用不可
    Weakness,     // 脆弱: 被ダメージ増加
    Curse,        // 呪い: HP回復不可
}
```

### 3.10 AIAttackEvaluation（AI認識用攻撃評価）

AIが攻撃を認識・回避するために使用するデータ。
敵の攻撃に対してAIが適切に反応するために必要な情報を集約する。

```csharp
[System.Serializable]
public struct AIAttackEvaluation
{
    public float effectiveRangeMin;       // 有効射程（最小距離）
    public float effectiveRangeMax;       // 有効射程（最大距離）
    public float threatLevel;             // 脅威度（0.0〜1.0、AI回避判断の優先度）
    public bool isBlockable;              // ガード可能か
    public bool isParriable;              // パリィ可能か
    public float totalWindupTime;         // 発動前合計時間（AI回避タイミング計算用）
    public float preferredUseHpThreshold; // HP閾値（AI がこの攻撃を選ぶHP条件、0=無条件）
    public byte priority;                 // AI行動選択時の優先度（0〜255）
}
```

**AI の攻撃回避判断フロー:**

```
1. 敵の攻撃を検知（AttackInfo を参照）
2. aiEvaluation.threatLevel を確認
   → 高脅威: 回避行動を最優先
   → 低脅威: 他の行動と比較して判断
3. isBlockable / isParriable を確認
   → ガード可能 かつ ガード中: ガード継続
   → パリィ可能 かつ タイミング合致: パリィ実行
4. totalWindupTime を確認
   → 回避に十分な時間があるか判断
5. effectiveRangeMin / Max を確認
   → 射程外に退避するか、射程内に留まるか判断
```

### 3.11 AttackInfoContainer（攻撃情報コンテナ）

`AttackInfo` はキャラクターデータとは別のコンテナで管理される。
武器・魔法からインデックスで参照する。

```csharp
/// <summary>
/// 全攻撃情報を保持するコンテナ。
/// SoACharaDataDic とは独立して管理される。
/// 武器や魔法がインデックスを保持し、このコンテナから AttackInfo を取得する。

{
    private AttackInfo[] attacks;

    /// <summary>攻撃情報の総数</summary>
    public int Count => attacks.Length;

    /// <summary>ScriptableObject 配列から初期化</summary>
    public void Initialize(AttackInfo[] sourceAttacks)
    {
        attacks = sourceAttacks;
    }
}
```

### 4.3 判定データ群

AIの行動決定に使用される判定条件群。判定順序は **トリガー → 行動選択 → ターゲット選択** であり、
行動の種類に応じて最適なターゲットを選択するため、ターゲット選択は行動選択の直後に実行される。

```csharp
/// <summary>
/// 発動トリガー判定 - 行動を検討し始める条件。
/// </summary>
[System.Serializable]
public struct TriggerJudgeData
{
    public TriggerConditionType condition;  // トリガー条件の種類
    public float value;                     // 条件値
    public ComparisonOperator comparison;   // 比較演算子
}

/// <summary>
/// ターゲット選択判定 - 行動対象の選択基準。
/// ActData 内に保持され、行動選択後にこの条件でターゲットを決定する。
/// </summary>
[System.Serializable]
public struct TargetJudgeData
{
    public TargetSelectionType selectionType; // ターゲット選択方式
    public float maxRange;                    // 最大射程
    public float minRange;                    // 最小射程
    public CharacterBelong targetBelong;      // ターゲット陣営
    public TargetFilter targetFilter;         // 詳細フィルター条件（AND/OR組み合わせ）
}

/// <summary>
/// 行動実行判定 - 最終的に行動を実行するかの条件。
/// </summary>
[System.Serializable]
public struct ActJudgeData
{
    public float requiredMp;                  // 必要MP
    public float requiredStamina;             // 必要スタミナ
    public float minHpRatio;                  // 最低HP割合（これ以上ないと使わない）
    public float maxHpRatio;                  // 最大HP割合（これ以下でないと使わない）
    public bool requireLineOfSight;           // 視線が通っている必要があるか
    public bool requireGrounded;              // 接地している必要があるか
}

/// <summary>トリガー条件の種類</summary>
public enum TriggerConditionType : byte
{
    Always,               // 常時
    EnemyInRange,         // 敵が射程内
    HpBelow,              // HPが閾値以下
    MpAbove,              // MPが閾値以上
    AllyHpBelow,          // 味方HPが閾値以下
    CoolTimeReady,        // クールタイム完了
    TargetStatusEffect,   // ターゲットが特定状態異常
}

/// <summary>ターゲット選択方式</summary>
public enum TargetSelectionType : byte
{
    Nearest,      // 最も近い対象
    Farthest,     // 最も遠い対象
    LowestHp,     // 最もHPが低い対象
    HighestThreat,// 最も脅威度が高い対象
    Random,       // ランダム
    Self,         // 自分自身（回復・バフ用）
    NearestAlly,  // 最も近い味方（支援用）
}

/// <summary>比較演算子</summary>
public enum ComparisonOperator : byte
{
    LessThan,
    LessOrEqual,
    Equal,
    GreaterOrEqual,
    GreaterThan,
}
```

/// <summary>

## 6. SoA コンテナでの管理まとめ

すべてのデータがどのように `GameManager.Data` 配下で管理されるかの全体像。

```
GameManager.Data (SoACharaDataDic)
│
├── CharacterInfo 由来
│   ├── UnsafeList<CharacterBaseInfo>     [HP / MP の最大値]
│   ├── UnsafeList<SolidData>             [識別・属性・不変データ]
│   ├── UnsafeList<CharacterAtkStatus>    [属性別攻撃力]
│   ├── UnsafeList<CharacterDefStatus>    [属性別防御力]
│   ├── UnsafeList<MoveStatus>            [移動・アクション設定]
│   └── UnsafeList<StaminaInfo>           [スタミナ設定]
│
├── 実行時生成（CharacterInfo からの初期化 + ランタイム更新）
│   ├── UnsafeList<CharacterStateInfo>    [現在HP/MP/スタミナ等の動的状態]
│   ├── UnsafeList<CharacterColdLog>      [判定履歴・クールタイム残り等]
│   ├── UnsafeList<RecognitionData>       [認識データ・ターゲット情報]
│   ├── UnsafeList<EquipmentStatus>       [装備由来ステータス合計] ← NEW
│   └── UnsafeList<CharacterStatusEffects>[状態異常管理データ]     ← NEW
│
└── AIInfo 由来
    └── UnsafeList<AIBrainData>            [フラット化されたAI判定データ]

AttackInfoContainer（別コンテナ）
└── AttackInfo[]                          [武器・魔法からインデックス参照]

ProjectileManager（別コンテナ）                                    ← NEW
├── NativeList<ProjectileInstance>         [アクティブ弾丸データ]
└── NativeArray<float2>                    [ターゲット位置キャッシュ]
```

### 6.1 ライフサイクル

```
1. エディタ     : CharacterInfo / AttackInfo / AIInfo を ScriptableObject として作成
2. 初期化時     : SourceGenerator 生成コードが ScriptableObject → SoA 構造体へマッピング
3. スポーン時   : BaseCharacter に entityIndex を割り当て、コンテナにデータを追加
4. 実行時      : AI判定が UnsafeList を直接参照して処理
5. デスポーン時 : entityIndex を解放（スワップバック方式でリスト末尾と入れ替え）
```

### 6.2 設計上の注意点

- **CharacterInfo と AIInfo は 1:1 対応ではない** — 同じ AIInfo を複数の CharacterInfo で共有可能（例: 雑魚敵が共通AIを使用）
- **AttackInfo は多対多** — 複数のキャラクターが同じ AttackInfo を参照可能。1つのキャラクターが複数の AttackInfo を持つことも可能
- **ScriptableObject は読み取り専用** — 実行時の変更は SoA コンテナ側のデータに対して行う。元の ScriptableObject は変更しない
- **インデックスの一貫性** — `entityIndex` はキャラクターの生存期間中は変化しない。デスポーン時のスワップバックでは影響を受けるキャラクターのインデックスも更新する

## 7. AI拡張構造体群

本セクションでは、AI の知性・人間性・適応性を向上させるために追加する構造体群を定義する。
すべて `unmanaged` 型（`half`, `byte`, `BitableBool` 等）で構成し、SoA コンテナとの互換を保証する。

---

### 7.1 PersonalityData（性格ベクトル）

LLM が3ラウンドで出力する性格パラメータを保持する構造体。
AI判定の入力として直接使用し、全プレイスタイルを滑らかに補間する。

```csharp
/// <summary>
/// LLM が生成する性格ベクトル（10D）。
/// AI判定の観測ベクトルに直接含まれ、行動選択に影響する。
/// Blender は条件配列のdeltaのみを適用する（ベース値はこの構造体）。
/// </summary>
public struct PersonalityData
{
    // ─── LLM action_policy 由来 ───
    /// <summary>攻撃性。0.0（消極的）〜 1.0（超攻撃的）</summary>
    public half aggression;
    /// <summary>生存優先度。0.0（無視）〜 1.0（最優先で生き残る）</summary>
    public half survivalPriority;
    /// <summary>距離嗜好。0.0（インファイト）〜 1.0（アウトレンジ）</summary>
    public half distancePreference;
    /// <summary>リソース節約。0.0（全力消費）〜 1.0（温存重視）</summary>
    public half resourceSaving;

    // ─── LLM reaction_policy 由来 ───
    /// <summary>復讐傾向。0.0（スルー）〜 1.0（必ず報復）</summary>
    public half revengeTendency;
    /// <summary>割り込み優先。0.0（無視）〜 1.0（即座にカット）</summary>
    public half interruptPriority;
    /// <summary>味方保護。0.0（自分優先）〜 1.0（味方最優先）</summary>
    public half allyProtection;

    // ─── LLM action_detail_policy 由来 ───
    /// <summary>リソース開放。0.0（温存）〜 1.0（必殺技/ゲージ全開放）</summary>
    public half resourceUnlock;
    /// <summary>コンボ継続。0.0（単発重視）〜 1.0（フルコンボ志向）</summary>
    public half comboCommitment;
    /// <summary>リスク許容。0.0（安全重視）〜 1.0（ハイリスクハイリターン）</summary>
    public half riskTolerance;
}
// half型 = 16bit → 10D × 2byte = 20byte/キャラ
```

**Blender との関係**:
- `PersonalityData` はベース値（LLM 出力そのまま）
- Blender は条件配列（`low_hp`, `high_resource` 等）からの **delta** を計算
- AI判定入力 = `PersonalityData` + `Blender delta`

**訓練時の使用法**:
- ランダムな性格ベクトルを生成して AI判定の入力に含める
- 同時に、性格に応じた報酬重みを適用
- 1つのモデルが全スタイルを学習し、推論時に LLM 出力の性格で制御

### 7.2 AnimationStateData（アニメーション状態）

アクションマスキングとフレームアドバンテージ計算に使用するアニメーション状態。
既存の `MotionInfo`（`AttackInfo` 内）の `preMotionDuration` / `activeMotionDuration` / `recoveryDuration` と連動する。

```csharp
/// <summary>
/// アニメーション状態データ。
/// BaseCharacter のアニメーションコントローラから毎フレーム更新される。
/// AI判定で Action Masking と frame_advantage 計算に使用。
/// </summary>
public struct AnimationStateData
{
    /// <summary>現在のアニメーションフェーズ</summary>
    public AnimationPhase currentPhase;
    /// <summary>フェーズ内の正規化時間（0.0〜1.0）</summary>
    public half normalizedTime;
    /// <summary>アクション可能になるまでのフレーム数</summary>
    public short framesUntilActionable;
    /// <summary>現在実行中の技ID（攻撃中のみ有効、それ以外は0）</summary>
    public byte currentMoveId;
    /// <summary>現在のアクションがキャンセル可能か</summary>
    public BitableBool isCancelable;
    /// <summary>コミット状態（キャンセル不可の攻撃中）か</summary>
    public BitableBool isCommitted;
}

/// <summary>アニメーションフェーズ</summary>
public enum AnimationPhase : byte
{
    /// <summary>ニュートラル — 全行動が選択可能</summary>
    Neutral,
    /// <summary>予備動作 — 発動前モーション中（MotionInfo.preMotionDuration に対応）</summary>
    Anticipation,
    /// <summary>アクティブ — 攻撃判定中（MotionInfo.activeMotionDuration に対応）</summary>
    Active,
    /// <summary>硬直 — 行動後の隙（MotionInfo.recoveryDuration に対応）。最も脆弱な状態</summary>
    Recovery,
}
```

**AI判定入力への変換（8D）**:
```
currentPhase → 4D one-hot [Neutral, Anticipation, Active, Recovery]
normalizedTime → 1D (0.0-1.0)
framesUntilActionable → 1D (正規化: frames / maxFrames)
isCancelable → 1D (0 or 1)
isCommitted → 1D (0 or 1)
```

**フレームアドバンテージ**（追加1D、AI判定内で計算）:
```
frame_advantage = (opponent.framesUntilActionable - self.framesUntilActionable) / maxFrames
// 正 = 有利（攻め継続）、負 = 不利（防御推奨）、0 = 五分
```

### 7.3 DeliberationBuffer（意図的遅延バッファ）

AI の反応速度に人間的なばらつきを導入する構造体。
AI判定の出力を即座に実行せず、状況に応じた遅延を挿入する。

```csharp
/// <summary>
/// AI の行動実行に人間的な遅延を挿入するバッファ。
/// AI判定の出力（MovementInfo）を保留し、遅延フレーム経過後に実行する。
/// </summary>
public struct DeliberationBuffer
{
    /// <summary>
    /// 基本遅延フレーム数。難易度依存。
    /// Easy = 12 (200ms@60fps), Normal = 6 (100ms), Hard = 2 (33ms)
    /// </summary>
    public byte baseDelayFrames;

    /// <summary>
    /// ランダム幅（フレーム数）。
    /// 予測通りの状況 = 少（0-2）、意外な状況 = 多（0-6）
    /// </summary>
    public byte varianceFrames;

    /// <summary>残り遅延フレーム数。0 になったら pendingAction を実行</summary>
    public byte currentDelay;

    /// <summary>保留中の行動。currentDelay == 0 で BaseCharacter に渡す</summary>
    public MovementInfo pendingAction;

    /// <summary>保留中の行動があるか</summary>
    public BitableBool hasPendingAction;
}
```

**処理フロー**:
```
AI判定出力 → judgeResult[index]
  ↓
DeliberationBuffer に格納:
  pendingAction = judgeResult[index]
  currentDelay = baseDelayFrames + Random(0, varianceFrames)
  hasPendingAction = true
  ↓
毎フレーム:
  if (hasPendingAction && currentDelay > 0) currentDelay--;
  if (hasPendingAction && currentDelay == 0)
  {
      BaseCharacter.ProcessAIResult(pendingAction);
      hasPendingAction = false;
  }
```

### 7.4 ActType 拡張

フェイントキャンセルアクションの追加。

```csharp
/// <summary>AI行動の種類</summary>
public enum ActType : byte
{
    Attack,       // 攻撃
    Move,         // 移動
    Guard,        // ガード
    Dodge,        // 回避
    Heal,         // 回復
    Buff,         // バフ
    Wait,         // 待機
    Flee,         // 逃走
    FeintCancel,  // フェイントキャンセル（Anticipation 中のみ有効）
}
```

**FeintCancel の仕様**:
- `AnimationPhase.Anticipation` 中のみ実行可能（他のフェーズでは Action Masking で無効化）
- 実行すると予備動作をキャンセルし、`AnimationPhase.Neutral` に即座に遷移
- 新規アニメーションアセット不要（既存モーションの途中キャンセル）
- `PersonalityData.riskTolerance` が高いほどフェイント選択の重みが増加
