# 04 - ダメージ・被弾システム

## 1. 概要

本システムは**与ダメージ（DamageDealer）**と**被弾（DamageReceiver）**の2つのコアコンポーネントで構成される。
攻撃判定から最終ダメージ適用まで、パリィ・ガード・多属性計算を一貫して処理する。

### 1.1 設計原則

| 原則 | 説明 |
|---|---|
| **GetComponent 禁止** | すべてのデータは `GameManager.Data`（SoACharaDataDic）に `objectHash` でアクセスする |
| **構造体ベース** | DamageData / DamageResult は struct。GC Alloc ゼロ |
| **属性別計算** | 7属性（斬撃・打撃・刺突・炎・雷・聖・闇）を個別に計算。弱点は defense[channel] の属性別低さで表現（弱点倍率の適用は仕様外）。※実装では物理タイプ(Slash/Strike/Pierce)もElement enumに統合済み |
| **単一責任** | DamageDealer は「ダメージを作る」、DamageReceiver は「ダメージを受ける」のみ |

### 1.2 システム全体の位置づけ

```
WeaponHolder（攻撃入力・コンボ管理）
    │
    ▼ AttackInfo 参照
DamageDealer（ダメージ生成・ヒットボックス管理）
    │
    │ DamageData を生成
    ▼
DamageReceiver（被弾判定・ダメージ適用）
    │
    ├─→ パリィ判定
    ├─→ ガード判定
    ├─→ 多属性ダメージ計算
    ├─→ HP減算（GameManager.Data 経由）
    ├─→ 状態異常適用
    └─→ AIイベント通知（AIManager.NotifyEvent）
```

## 2. DamageData 構造体

攻撃者から被弾者に渡されるダメージ情報の一括パケット。

```csharp
/// <summary>
/// 攻撃時に生成されるダメージ情報パケット。
/// DamageDealerが生成し、DamageReceiverに渡す。
/// </summary>
public struct DamageData
{
    /// <summary>攻撃者の objectHash</summary>
    public int attackerHash;

    /// <summary>被弾者の objectHash</summary>
    public int defenderHash;

    /// <summary>属性別ダメージ値（7属性: 斬撃/打撃/刺突/炎/雷/聖/闇）</summary>
    public ElementalStatus damage;

    /// <summary>モーション値（ダメージ倍率）</summary>
    public float motionValue;

    /// <summary>吹き飛ばし力（方向+大きさ）</summary>
    public Vector2 knockbackForce;

    /// <summary>攻撃属性（Element flags）</summary>
    public Element attackElement;

    /// <summary>付与する状態異常情報（蓄積モデル）</summary>
    public StatusEffectInfo statusEffectInfo;

    /// <summary>攻撃特性フラグ（Unparriable, SuperArmor等）</summary>
    public AttackFeature feature;

    /// <summary>アーマー削り値</summary>
    public float armorBreakValue;

    /// <summary>ジャストガード耐性（0〜100）</summary>
    public float justGuardResistance;

    /// <summary>飛翔体による攻撃かどうか</summary>
    public bool isProjectile;
}
```

### 2.2 属性システム（統合7属性）

旧設計では「属性（無・火・水・風・土・光・闇）」と「物理タイプ（斬撃・打撃・刺突）」が別軸だったが、
実装では **Element enum に物理3種+魔法4種を統合** し、`[Flags]` として複合指定も可能にした。
`WeaponPhysicalType` は廃止済みである。

```csharp
/// <summary>属性（物理3種 + 魔法4種 = 7属性）</summary>
[Flags]
public enum Element : byte
{
    None    = 0,
    Slash   = 1 << 0,  // 斬撃
    Strike  = 1 << 1,  // 打撃
    Pierce  = 1 << 2,  // 刺突
    Fire    = 1 << 3,  // 炎
    Thunder = 1 << 4,  // 雷
    Light   = 1 << 5,  // 聖
    Dark    = 1 << 6,  // 闇
}
```

### 2.3 ElementalStatus 構造体

7属性それぞれの値を保持する構造体。攻撃力・防御力・ダメージの各チャネルで使用する。

```csharp
/// <summary>
/// 7属性（斬撃/打撃/刺突/炎/雷/聖/闇）のステータス値。
/// 攻撃力・防御力・ダメージの各チャネルで使用する。
/// </summary>
[Serializable]
public struct ElementalStatus
{
    public int slash;     // 斬撃
    public int strike;    // 打撃
    public int pierce;    // 刺突
    public int fire;      // 炎
    public int thunder;   // 雷
    public int light;     // 聖
    public int dark;      // 闇

    public int Total => slash + strike + pierce + fire + thunder + light + dark;
    public int PhysicalTotal => slash + strike + pierce;
    public int MagicalTotal => fire + thunder + light + dark;

    public int Get(Element element) => element switch
    {
        Element.Slash   => slash,
        Element.Strike  => strike,
        Element.Pierce  => pierce,
        Element.Fire    => fire,
        Element.Thunder => thunder,
        Element.Light   => light,
        Element.Dark    => dark,
        _               => 0,
    };
}
```

## 3. DamageDealer（与ダメージ側）

攻撃のダメージ情報を生成し、ヒットボックスやプロジェクタイル経由で DamageReceiver に送信する。

```csharp
/// <summary>
/// 与ダメージ管理コンポーネント。
/// 攻撃情報（AttackInfo）とキャラクターの攻撃ステータスから DamageData を生成する。
/// データは全て GameManager.Data に objectHash でアクセスする。
/// </summary>
public class DamageDealer : MonoBehaviour
{
    /// <summary>所有者の objectHash（GameManager.Data へのキー）</summary>
    private int _ownerHash;

    /// <summary>現在の攻撃情報（WeaponHolder から設定される）</summary>
    private AttackInfo _currentAttackInfo;

    // ===============================================================
    //  初期化
    // ===============================================================

    public void Initialize(int ownerHash)
    {
        _ownerHash = ownerHash;
    }

    // ===============================================================
    //  ダメージ計算
    // ===============================================================

    /// <summary>
    /// 現在の AttackInfo とキャラクターの CombatStats から DamageData を生成する。
    /// </summary>
    public DamageData CalculateOutgoingDamage(int defenderHash)
    {
        // GameManager.Data 経由で CombatStats を取得（GetComponent 不使用）
        ref CombatStats combatStats =
            ref GameManager.Data.GetCombatStats(_ownerHash);

        // AttackInfo の baseDamage(ElementalStatus) をそのまま使用
        // damageMultiplier で武器攻撃力への倍率を適用

        return new DamageData
        {
            attackerHash     = _ownerHash,
            defenderHash     = defenderHash,
            damage           = _currentAttackInfo.baseDamage,
            motionValue      = _currentAttackInfo.damageMultiplier,
            knockbackForce   = _currentAttackInfo.knockbackInfo.force,
            attackElement    = _currentAttackInfo.attackElement,
            statusEffectInfo = _currentAttackInfo.statusEffectInfo,
            feature          = /* AttackFeature flags */,
            armorBreakValue  = _currentAttackInfo.armorBreakValue,
            justGuardResistance = _currentAttackInfo.justGuardResistance,
            isProjectile     = false,
        };
    }

    /// <summary>
    /// WeaponHolder から攻撃開始時に呼ばれる。
    /// </summary>
    public void SetAttackInfo(AttackInfo info)
    {
        _currentAttackInfo = info;
    }
}
```

### 3.1 ダメージ計算式

実装では静的ユーティリティクラス群（`DamageCalculator`, `HpArmorLogic`, `SituationalBonusLogic`）に分離されている。

```
【単一属性の基本ダメージ（DamageCalculator.CalculateBaseDamage）】

baseDmg = (atk^2 × motionValue) / (atk + def)
result = max(k_MinDamage, baseDmg)

【7属性合計ダメージ（DamageCalculator.CalculateTotalDamage）】

for each channel in [Slash, Strike, Pierce, Fire, Thunder, Light, Dark]:
  channelDmg = CalculateBaseDamage(atk[channel], motionValue, def[channel])
  ※ atk[channel] == 0 の属性はスキップ
  ※ 弱点倍率（weakElement ×1.5）は仕様外。弱点は defense[channel] を低く設定することで表現する。

totalDamage = max(1, Σ(channelDmg))

【HP・アーマー適用（HpArmorLogic.ApplyDamage）】

1. アーマー破壊: armorBreakValue > 0 の場合
   - 行動アーマー（actionArmor）を先に消費
   - 残りをベースアーマー（currentArmor）から減算
2. アーマーブレイクボーナス: 全アーマー ≤ 0 なら actualDamage × 1.3倍
3. HP減算: currentHp = max(0, currentHp - actualDamage)
4. キル判定: currentHp <= 0

【状況ボーナス（SituationalBonusLogic）】

- Counter: 攻撃中の敵にヒット
- Backstab: 背面からの攻撃
- StaggerHit: 怯み中の敵にヒット

【ガード判定（GuardJudgmentLogic）】

- 7属性別のガードカット率（GuardStats）を適用
- ジャストガード: 攻撃のjustGuardResistance内で成功
- ガードブレイク: スタミナ不足時

【状態異常（蓄積モデル）】

- ヒットごとに蓄積値を加算（StatusEffectInfo.accumulateValue）
- 閾値超過で発症（最大3スロット同時）
- 11種: Poison/Burn/Bleed/Stun/Freeze/Paralyze/Slow/Blind/Silence/Weakness/Curse
```

### 3.2 ヒットボックス生成

```csharp
/// <summary>
/// 攻撃アニメーションのイベントから呼ばれるヒットボックス。
/// Trigger コライダーで対象を検出し、DamageReceiver に DamageData を送信する。
/// </summary>
public class HitBox : MonoBehaviour
{
    private DamageDealer _dealer;
    private HashSet<int> _alreadyHit; // 同一攻撃で二重ヒット防止

    /// <summary>
    /// ヒットボックスを有効化する。攻撃開始時に呼ばれる。
    /// </summary>
    public void Activate(DamageDealer dealer)
    {
        _dealer = dealer;
        _alreadyHit ??= new HashSet<int>();
        _alreadyHit.Clear();
        gameObject.SetActive(true);
    }

    /// <summary>
    /// ヒットボックスを無効化する。攻撃終了時に呼ばれる。
    /// </summary>
    public void Deactivate()
    {
        gameObject.SetActive(false);
    }

    private void OnTriggerEnter2D(Collider2D other)
    {
        int targetHash = other.gameObject.GetHashCode();

        // 二重ヒット防止
        if (!_alreadyHit.Add(targetHash)) return;

        // GameManager.Data から DamageReceiver 参照を取得
        // （GetComponent ではなく objectHash 経由）
        if (GameManager.Data.TryGetDamageReceiver(targetHash,
                out DamageReceiver receiver))
        {
            DamageData data = _dealer.CalculateOutgoingDamage(targetHash);
            DamageResult result = receiver.ApplyDamage(data);

            // 与ダメージイベントを AI に通知
            if (result.totalDamage > 0)
            {
                AIManager.NotifyEvent(
                    _dealer.OwnerHash,
                    BrainEventFlagType.大ダメージを与えた,
                    targetHash
                );
            }
        }
    }
}
```

### 3.3 飛翔体のダメージ

```csharp
/// <summary>
/// 飛翔体（矢・魔法弾等）が保持するダメージ情報。
/// 生成時に DamageDealer から DamageData を受け取り、
/// 着弾時に DamageReceiver に渡す。
/// </summary>
public class ProjectileDamage : MonoBehaviour
{
    private DamageData _cachedDamage;

    /// <summary>
    /// 飛翔体生成時に DamageDealer から呼ばれる。
    /// ダメージ情報をキャッシュし、isProjectile を true に設定する。
    /// </summary>
    public void SetDamageData(DamageData data)
    {
        data.isProjectile = true;
        _cachedDamage = data;
    }

    private void OnTriggerEnter2D(Collider2D other)
    {
        int targetHash = other.gameObject.GetHashCode();

        // 味方判定（同一陣営には当たらない）
        // CharacterFlags の Belong ビット（CharacterBelong [Flags]）で判定
        ref CharacterFlags attackerFlags =
            ref GameManager.Data.GetFlags(_cachedDamage.attackerHash);
        ref CharacterFlags defenderFlags =
            ref GameManager.Data.GetFlags(targetHash);

        if (IsSameBelong(attackerFlags, defenderFlags)) return;

        _cachedDamage.defenderHash = targetHash;

        if (GameManager.Data.TryGetDamageReceiver(targetHash,
                out DamageReceiver receiver))
        {
            receiver.ApplyDamage(_cachedDamage);
        }

        // 着弾処理（エフェクト・SE・オブジェクトプールへ返却）
        OnImpact();
    }

    private bool IsSameBelong(in CharacterFlags a, in CharacterFlags b)
    {
        // 陣営（Belong）が同一なら味方と判断
        return (a.Belong & b.Belong) != 0;
    }
}
```

## 4. DamageReceiver（被弾側）

ダメージの受理・軽減・適用を一括管理するコンポーネント。

```csharp
/// <summary>
/// 被弾管理コンポーネント。
/// パリィ判定 → ガード判定 → 属性別ダメージ計算 → HP減算 → 状態異常適用
/// の順でダメージを処理する。
/// データは全て GameManager.Data に objectHash でアクセスする。
/// </summary>
public class DamageReceiver : MonoBehaviour
{
    /// <summary>所有者の objectHash（GameManager.Data へのキー）</summary>
    private int _ownerHash;

    // ---- パリィ関連 ----
    /// <summary>パリィ受付中かどうか</summary>
    private bool _isParryWindow;

    /// <summary>パリィウィンドウの残り時間</summary>
    private float _parryTimer;

    // ---- ガード関連 ----
    /// <summary>ガード中かどうか</summary>
    private bool _isGuarding;

    // ---- 定数 ----
    /// <summary>パリィ成功時の敵スタン時間（秒）</summary>
    private const float PARRY_STUN_DURATION = 1.5f;

    /// <summary>パリィ成功時の反撃ダメージ倍率</summary>
    private const float PARRY_COUNTER_MULTIPLIER = 1.5f;

    // 注: 弱点属性倍率（WEAKNESS_MULTIPLIER）は仕様上存在しない。
    //     弱点は defense[channel] を属性別に低く設定することで表現する。

    /// <summary>ガード時のダメージ軽減率（1.0 = 100%カット）</summary>
    private const float GUARD_REDUCTION = 0.7f;

    // ===============================================================
    //  初期化
    // ===============================================================

    public void Initialize(int ownerHash)
    {
        _ownerHash = ownerHash;
    }

    // ===============================================================
    //  メインダメージ処理
    // ===============================================================

    /// <summary>
    /// ダメージを適用する。計算順序:
    /// 1. パリィ判定
    /// 2. ガード判定
    /// 3. 属性別ダメージ計算（attack^2 × motionValue / (attack + defense) を 7 属性合算）
    /// 4. HP減算
    /// 5. 状態異常判定
    /// 6. 吹き飛ばし適用
    /// 7. AIイベント通知
    /// </summary>
    public DamageResult ApplyDamage(DamageData data)
    {
        var result = new DamageResult();

        // ----- Step 1: パリィ判定 -----
        if (TryParry(data, ref result))
        {
            return result; // パリィ成功 → ダメージ無効
        }

        // ----- Step 2: ガード判定 -----
        bool guarded = TryGuard(data, ref result);

        // ----- Step 3: 属性別ダメージ計算 -----
        int totalDamage = CalculateElementalDamage(data, guarded);
        result.totalDamage = totalDamage;

        // ----- Step 4: HP減算 -----
        ApplyHpReduction(totalDamage, ref result);

        // ----- Step 5: 状態異常判定 -----
        if (!guarded) // ガード中は状態異常無効
        {
            TryApplyStatusEffect(data, ref result);
        }

        // ----- Step 6: 吹き飛ばし適用 -----
        if (!guarded || result.guardBroken)
        {
            ApplyKnockback(data);
        }

        // ----- Step 7: AIイベント通知 -----
        NotifyAI(data, result);

        return result;
    }

    // ===============================================================
    //  パリィ
    // ===============================================================

    /// <summary>
    /// パリィ判定。パリィウィンドウ中かつパリィ可能な攻撃の場合に成功。
    /// </summary>
    private bool TryParry(in DamageData data, ref DamageResult result)
    {
        if (!_isParryWindow) return false;
        if (!IsParriable(data)) return false;

        result.wasParried = true;
        result.totalDamage = 0;

        // 攻撃者にスタンを適用
        ApplyParryStun(data.attackerHash);

        // パリィウィンドウを閉じる
        _isParryWindow = false;
        _parryTimer = 0f;

        return true;
    }

    // ===============================================================
    //  ガード
    // ===============================================================

    /// <summary>
    /// ガード判定。ガード中ならダメージ軽減、スタミナ超過でガードブレイク。
    /// </summary>
    private bool TryGuard(in DamageData data, ref DamageResult result)
    {
        if (!_isGuarding) return false;

        // スタミナ消費計算（CharacterVitals 内の currentStamina を使用）
        ref CharacterVitals vitals =
            ref GameManager.Data.GetVitals(_ownerHash);
        float staminaCost = CalculateGuardStaminaCost(data);

        if (vitals.currentStamina < staminaCost)
        {
            // ガードブレイク
            result.guardBroken = true;
            _isGuarding = false;
            vitals.currentStamina = 0;
            return false;
        }

        // スタミナ消費してガード成立
        vitals.currentStamina -= staminaCost;
        result.wasGuarded = true;
        return true;
    }

    // ===============================================================
    //  属性別ダメージ計算
    // ===============================================================

    /// <summary>
    /// 7属性（斬撃/打撃/刺突/炎/雷/聖/闇）それぞれで
    /// (atk^2 × motionValue) / (atk + def) を計算して合算する。
    /// 弱点は defense[channel] を低く設定することで表現（弱点倍率は仕様外）。
    /// 実装では DamageCalculator.CalculateTotalDamage() に委譲。
    /// </summary>
    private int CalculateElementalDamage(in DamageData data, bool guarded)
    {
        ref CombatStats combatStats =
            ref GameManager.Data.GetCombatStats(_ownerHash);

        int elementalTotal = 0;

        // 実装では DamageCalculator.CalculateTotalDamage() に委譲:
        // 各属性チャネル(Slash/Strike/Pierce/Fire/Thunder/Light/Dark)で
        // (atk^2 × motionValue) / (atk + def) を計算して合算。
        // 弱点倍率は仕様外。弱点は defense[channel] を低く設定することで表現する。
        int totalDamage = DamageCalculator.CalculateTotalDamage(
            data.damage, data.motionValue,
            combatStats.defense, Element.None);

        // ガード中ならダメージ軽減
        if (guarded)
        {
            totalDamage = Mathf.Max(1, (int)(totalDamage * (1f - GUARD_REDUCTION)));
        }

        return totalDamage;
    }

    /// <summary>
    /// 単一属性のダメージ計算。
    /// finalDamage = max(1, attackDamage - defenderDef)
    /// 弱点倍率は仕様外（弱点は defenderDef[channel] を低く設定することで表現）。
    /// 状態異常 Weakness 中なら追加倍率。
    /// </summary>
    private int CalcSingleElement(
        int atkDamage, int defValue, Element element)
    {
        if (atkDamage <= 0) return 0;

        int raw = Mathf.Max(1, atkDamage - defValue);

        // 状態異常 Weakness による被ダメージ増加
        ref CharacterStatusEffects effects =
            ref GameManager.Data.GetStatusEffects(_ownerHash);
        if (effects.HasActive(StatusEffectId.Weakness))
        {
            raw = (int)(raw * effects.GetModifier(StatusEffectId.Weakness));
        }

        return raw;
    }

    // ===============================================================
    //  HP減算
    // ===============================================================

    /// <summary>
    /// HP・アーマー適用。実装では HpArmorLogic.ApplyDamage() に委譲。
    /// アーマー破壊→アーマーブレイクボーナス→HP減算→キル判定の順。
    /// HP は必ず 0 以上にクランプされる。
    /// </summary>
    private void ApplyHpReduction(int totalDamage, ref DamageResult result)
    {
        ref CharacterVitals vitals =
            ref GameManager.Data.GetVitals(_ownerHash);

        // HpArmorLogic.ApplyDamage() がアーマー消費→HP減算→クランプを一括処理
        var (actualDmg, isKill, armorBroken) = HpArmorLogic.ApplyDamage(
            ref vitals.currentHp, ref vitals.currentArmor,
            totalDamage, data.armorBreakValue, ref actionArmor);

        result.totalDamage = actualDmg;
        result.isKill = isKill;
    }

    // ===============================================================
    //  状態異常
    // ===============================================================

    private void TryApplyStatusEffect(in DamageData data, ref DamageResult result)
    {
        if (data.statusEffectInfo.effect == StatusEffectId.None) return;

        // 蓄積モデル: ヒットごとに蓄積値を加算し、閾値超過で発症
        ref CharacterStatusEffects effects =
            ref GameManager.Data.GetStatusEffects(_ownerHash);

        effects.Accumulate(
            data.statusEffectInfo.effect,
            data.statusEffectInfo.accumulateValue,
            data.statusEffectInfo   // 発症時のパラメータ
        );
        // → 蓄積値 ≥ 閾値 なら内部で自動的に発症スロットに登録される
    }

    // ===============================================================
    //  吹き飛ばし
    // ===============================================================

    private void ApplyKnockback(in DamageData data)
    {
        if (data.knockbackForce <= 0f) return;

        // BaseCharacter の物理挙動に吹き飛ばしを適用
        // GameManager.Data 経由で BaseCharacter 参照を取得
        if (GameManager.Data.TryGetManaged(_ownerHash,
                out BaseCharacter character))
        {
            character.ApplyExternalForce(
                data.knockbackDir * data.knockbackForce
            );
        }
    }

    // ===============================================================
    //  AI通知
    // ===============================================================

    private void NotifyAI(in DamageData data, in DamageResult result)
    {
        // 被弾者 → AIManager に被弾イベントを通知
        AIManager.NotifyEvent(
            _ownerHash,
            BrainEventFlagType.大ダメージを受けた,
            data.attackerHash
        );

        // 死亡時はキル通知
        if (result.isKill)
        {
            AIManager.NotifyEvent(
                data.attackerHash,
                BrainEventFlagType.キャラを倒した,
                _ownerHash
            );
        }
    }

    // ===============================================================
    //  パリィ補助メソッド
    // ===============================================================

    /// <summary>
    /// パリィウィンドウを開始する。ガード入力直後に呼ばれる。
    /// </summary>
    public void StartParryWindow(float duration)
    {
        _isParryWindow = true;
        _parryTimer = duration;
    }

    /// <summary>ガード状態を設定する。</summary>
    public void SetGuarding(bool guarding)
    {
        _isGuarding = guarding;
    }

    /// <summary>
    /// パリィ可能な攻撃かどうかを判定する。
    /// </summary>
    private bool IsParriable(in DamageData data)
    {
        // AttackFeature.Unparriable フラグで判定
        if ((data.feature & AttackFeature.Unparriable) != 0) return false;
        // 飛翔体はパリィ不可（設定で変更可能にする場合はここを拡張）
        if (data.isProjectile) return false;
        return true;
    }

    /// <summary>
    /// パリィ成功時に攻撃者にスタンを適用する。
    /// </summary>
    private void ApplyParryStun(int attackerHash)
    {
        // 攻撃者にスタン状態異常を即時発症させる
        ref CharacterStatusEffects attackerEffects =
            ref GameManager.Data.GetStatusEffects(attackerHash);
        attackerEffects.ForceApply(
            StatusEffectId.Stun,
            PARRY_STUN_DURATION
        );
    }

    private void Update()
    {
        // パリィウィンドウのタイマー管理
        if (_isParryWindow)
        {
            _parryTimer -= Time.deltaTime;
            if (_parryTimer <= 0f)
            {
                _isParryWindow = false;
            }
        }
    }

    /// <summary>
    /// ガード時のスタミナ消費量を計算する。
    /// 盾のガード効率と防御力で軽減される。
    /// </summary>
    private float CalculateGuardStaminaCost(in DamageData data)
    {
        // 基本コスト = 合計ダメージの一定割合
        float baseCost = data.damage.Total * 0.5f;

        // 盾のガード効率による軽減
        ref EquipmentStatus equipStatus =
            ref GameManager.Data.GetEquipmentStatus(_ownerHash);
        float shieldReduction = equipStatus.guardEfficiency;

        // 防御力による軽減
        ref CombatStats combatStats =
            ref GameManager.Data.GetCombatStats(_ownerHash);
        float defRatio = Mathf.Clamp01(
            (float)combatStats.defense.Total /
            Mathf.Max(1, data.damage.Total) * 0.3f);

        return baseCost * (1f - shieldReduction) * (1f - defRatio);
    }
}
```

### 6.2 パリィ成功時の効果

パリィが成功すると以下の3つの効果が発生する。

```
パリィ成功
    │
    ├─ ① ダメージ完全無効
    │     result.totalDamage = 0
    │     result.wasParried = true
    │
    ├─ ② 攻撃者にスタンを適用
    │     StatusEffectId.Stun を ForceApply（蓄積を経ずに即発症）
    │     持続時間: PARRY_STUN_DURATION (1.5秒)
    │     → スタン中は行動不能
    │
    └─ ③ 反撃ボーナス
          次の攻撃のダメージ倍率: PARRY_COUNTER_MULTIPLIER (1.5倍)
          持続時間: スタン時間と同等
          → DamageDealer 側で倍率を参照
```

```csharp
/// <summary>
/// パリィ成功時の処理まとめ。
/// </summary>
private void OnParrySuccess(in DamageData data, ref DamageResult result)
{
    // ① ダメージ無効
    result.wasParried = true;
    result.totalDamage = 0;

    // ② 攻撃者スタン
    ApplyParryStun(data.attackerHash);

    // ③ 反撃ボーナス（CharacterFlags にBrainEventフラグ設定）
    ref CharacterFlags selfFlags =
        ref GameManager.Data.GetFlags(_ownerHash);
    // パリィ成功フラグを設定 → DamageDealer が次の攻撃時に参照
    selfFlags.BrainEventFlags |= (byte)BrainEventFlagType.攻撃対象指定;

    // パリィエフェクト・SE 再生
    // EffectManager.Play(ParryEffect, transform.position);
}
```

### 6.3 パリィ可能な攻撃の条件

| 条件 | パリィ可否 | 理由 |
|---|---|---|
| AttackFeature.Unparriable なし | **可能** | 近接攻撃は基本的にパリィ可能 |
| AttackFeature.Unparriable あり | **不可** | フラグで個別に制御 |
| 飛翔体（isProjectile = true） | **不可** | 矢・魔法弾等はパリィ不可 |
| AttackInfo.isParriable | **設定次第** | AttackInfo 側でもパリィ可否を設定可能 |

## 7. ガードシステム

### 7.1 ガード中のダメージ軽減率

```
ガード中のダメージ計算:

  通常ダメージ = Σ max(1, atkDamage[e] - def[e])
  ガード後ダメージ = 通常ダメージ × (1 - GUARD_REDUCTION)
                   = 通常ダメージ × 0.3
  ※ 弱点倍率は仕様外（弱点は defense[channel] を低く設定することで表現）
```

| 項目 | 値 | 説明 |
|---|---|---|
| 基本軽減率 | 70% | `GUARD_REDUCTION = 0.7f` |
| 最低ダメージ | 1 | ガード中でも最低1ダメージは受ける |
| 状態異常 | 無効 | ガード成立中は状態異常を完全ブロック |
| 吹き飛ばし | 無効 | ガード成立中は吹き飛ばし無効 |
| スタミナ消費 | あり | ダメージ量に応じたスタミナを消費 |

### 7.2 ガードブレイク（スタミナ超過時）

```
ガードブレイク判定:

  ガードスタミナコスト = ダメージ合計 × 0.5 × (1 - 防御補正)

  if (currentStamina < ガードスタミナコスト):
      ガードブレイク発生
      ├─ ガード解除 (_isGuarding = false)
      ├─ スタミナ = 0
      ├─ ダメージ軽減なし（フルダメージ適用）
      ├─ 吹き飛ばし適用
      └─ スタン状態（短時間行動不能）
```

```

│               ガード状態遷移図                    │
│                                                  │

│      │                                           │
│      ▼                                           │
│   ┌──────────┐   パリィウィンドウ中   ┌───────┐  │
│   │ パリィ   │ ──────────────────→  │ ガード │  │
│   │ 受付中   │   ウィンドウ終了       │  中    │  │
│   └──────────┘                       └───┬───┘  │
```

### 7.3 属性別ガード耐性

ガード時のスタミナ消費は**盾のガード効率**と**防御力**で軽減される。

```
ガードスタミナコスト計算:

  baseCost = ダメージ合計 × 0.5
  盾軽減 = shield.guardEfficiency (0.0〜1.0)
  防御補正 = clamp(防御力合計 / ダメージ合計 × 0.3, 0, 1)
  最終コスト = baseCost × (1 - 盾軽減) × (1 - 防御補正)

  例: ダメージ合計 = 100, guardEfficiency = 0.4, 防御力合計 = 50
      baseCost = 50
      盾軽減後 = 50 × (1 - 0.4) = 30
      防御補正 = clamp(50/100 × 0.3, 0, 1) = 0.15
      最終コスト = 30 × 0.85 = 25.5
```

| 盾効率 | 防御力/ダメージ比 | 最終コスト(ダメージ100時) |
|---|---|---|
| 0.0 | 0% | 50.0 |
| 0.4 | 0% | 30.0 |
| 0.4 | 50% | 25.5 |
| 0.7 | 100% | 10.5 |
| 1.0 | 任意 | 0（完全防御） |

## 8. 状態異常管理システム

### 8.1 設計方針

従来のビットフラグ + 確率判定方式を廃止し、**蓄積→発症モデル**を採用する。
攻撃ヒットごとに蓄積値を加算し、閾値を超えた時点で状態異常が発症する。
これにより、ボスや高耐性キャラへの段階的な状態異常適用や、
連続攻撃による状態異常の確実な蓄積が実現できる。

### 8.2 StatusEffectId（状態異常ID）

旧 SpecialEffect enum を廃止。

```csharp
/// <summary>

/// SoAコンテナ上の CharacterStatusEffects で複数同時管理。
/// </summary>
public enum StatusEffectId : byte
{
    None = 0,

    // ─── 持続ダメージ系 (DoT) ───
    Poison,       // 毒: 毎秒固定ダメージ
    Burn,         // 炎上: 毎秒固定ダメージ + 防御低下
    Bleed,        // 出血: 移動時に追加ダメージ

    // ─── 行動阻害系 (Debilitation) ───
    Stun,         // スタン: 完全行動不能（短時間）
    Freeze,       // 凍結: 移動不能 + 被ダメージ増加
    Paralyze,     // 麻痺: 確率で行動失敗
    Slow,         // 鈍足: 移動速度低下

    // ─── 能力低下系 (Weakness) ───
    Blind,        // 暗闇: AI索敵範囲低下
    Silence,      // 沈黙: 魔法使用不可
    Weakness,     // 脆弱: 被ダメージ増加
    Curse,        // 呪い: HP回復不可
}

/// <summary>状態異常のカテゴリ分類</summary>
public enum StatusEffectCategory : byte
{
    DamageOverTime,   // 持続ダメージ系
    Debilitation,     // 行動阻害系
    WeaknessType,     // 能力低下系
}
```

```
│  accumulation += adjustedValue                   │
│                                                  │
│  if (accumulation >= threshold):                 │
│    → 発症: StatusEffectSlot に登録               │
│    → accumulation = 0 にリセット                 │
│    → 既に発症中 && stackable:                    │
│        currentStack++ (maxStack上限)             │
│        remainingDuration をリセット               │
│    → activeFlags に該当ビットをセット            │
└─────────────────┬───────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────┐
│ 毎フレーム更新 (StatusEffectManager.Update)     │
│                                                  │
│  for each activeSlot:                            │
│    remainingDuration -= deltaTime                │
│    if (remainingDuration <= 0):                  │
│      → 解除: slot.id = None                     │
│      → activeFlags から該当ビットをクリア        │
│      → continue                                 │
│                                                  │
│    tickTimer -= deltaTime                        │
│    if (tickTimer <= 0):                          │
│      → DoT処理: HP -= damagePerTick × stack     │
│      → tickTimer += tickInterval                │
│                                                  │
│  for each accumSlot:                             │
│    accumulation -= decayRate × deltaTime         │
│    accumulation = max(0, accumulation)           │
└─────────────────────────────────────────────────┘
```

### 8.6 状態異常効果テーブル（仮数値）

| 状態異常 | カテゴリ | 効果 | スタック | 蓄積閾値 | 減衰率(/秒) | 持続時間 | tick間隔 | 解除条件 |
|---|---|---|---|---|---|---|---|---|
| Poison | DoT | 毎秒HP2%減少 | 3段階(×1.0/1.5/2.0) | 100 | 5 | 8秒 | 1.0秒 | 時間 / 解毒 |

| Stun | 阻害 | 完全行動不能 | 不可 | 200 | 20 | 1.5秒 | - | 時間 |
| Freeze | 阻害 | 移動不能+被ダメ×1.3 | 不可 | 120 | 10 | 3秒 | - | 時間 / 被弾 |
| Paralyze | 阻害 | 30%確率で行動失敗 | 不可 | 100 | 8 | 4秒 | - | 時間 |
| Slow | 阻害 | 移動速度50%低下 | 不可 | 50 | 5 | 5秒 | - | 時間 |

※ 蓄積閾値はキャラクターの `statusResistance` で倍率補正される:
  `実効閾値 = 基本閾値 × (1 + statusResistance)`

### 8.7 StatusEffectManager

```csharp
/// <summary>
/// 全キャラクターの状態異常を毎フレーム更新するマネージャー。
/// GameManager の子コンポーネント。SoAコンテナを直接操作する。
/// </summary>
public class StatusEffectManager : MonoBehaviour
{
    public void UpdateAllEffects(float deltaTime)
    {
        var data = GameManager.Data;
        for (int i = 0; i < data.Count; i++)
        {
            ref CharacterStatusEffects effects = ref data.GetStatusEffectsAt(i);
            if (effects.activeFlags == 0 &&
                effects.accum0.targetEffect == StatusEffectId.None &&
                effects.accum1.targetEffect == StatusEffectId.None &&
                effects.accum2.targetEffect == StatusEffectId.None &&
                effects.accum3.targetEffect == StatusEffectId.None)
                continue; // 処理不要

            // 発症中スロットの更新
            UpdateSlot(ref effects.slot0, ref effects, i, deltaTime);
            UpdateSlot(ref effects.slot1, ref effects, i, deltaTime);
            UpdateSlot(ref effects.slot2, ref effects, i, deltaTime);
            UpdateSlot(ref effects.slot3, ref effects, i, deltaTime);

            // 蓄積値の自然減衰
            DecayAccumulation(ref effects.accum0, deltaTime);
            DecayAccumulation(ref effects.accum1, deltaTime);
            DecayAccumulation(ref effects.accum2, deltaTime);
            DecayAccumulation(ref effects.accum3, deltaTime);
        }
    }

    private void UpdateSlot(ref StatusEffectSlot slot,
        ref CharacterStatusEffects effects, int charIndex, float dt)
    {
        if (slot.id == StatusEffectId.None) return;

        slot.remainingDuration -= dt;
        if (slot.remainingDuration <= 0f)
        {
            effects.activeFlags &= (ushort)~(1 << (int)slot.id);
            slot.id = StatusEffectId.None;
            return;
        }

        // DoT処理（tickInterval > 0 の場合のみ）
        if (slot.tickInterval > 0f)
        {
            slot.tickTimer -= dt;
            if (slot.tickTimer <= 0f)
            {
                float damage = slot.damagePerTick * slot.currentStack;
                ref CharacterBaseInfo baseInfo =
                    ref GameManager.Data.GetBaseInfoAt(charIndex);
                baseInfo.currentHp -= (int)damage;
                slot.tickTimer += slot.tickInterval;
            }
        }
    }

    private void DecayAccumulation(ref StatusAccumulation accum, float dt)
    {
        if (accum.targetEffect == StatusEffectId.None) return;
        accum.accumulation = Mathf.Max(0f,
            accum.accumulation - accum.decayRate * dt);
        if (accum.accumulation <= 0f)
            accum.targetEffect = StatusEffectId.None; // スロット解放
    }
}
```

## 9. DamageResult

ダメージ処理の結果を呼び出し元に返す構造体。

```csharp
/// <summary>
/// DamageReceiver.ApplyDamage の戻り値。
/// 攻撃側（DamageDealer / HitBox）がこの結果を受けて
/// エフェクト表示、AI通知、コンボ継続判定などを行う。
/// </summary>
public struct DamageResult
{
    /// <summary>実際に適用された合計ダメージ量</summary>
    public int totalDamage;

    /// <summary>パリィで無効化されたかどうか</summary>
    public bool wasParried;

    /// <summary>ガードで軽減されたかどうか</summary>
    public bool wasGuarded;

    /// <summary>ガードブレイクが発生したかどうか</summary>
    public bool guardBroken;

    /// <summary>この攻撃で対象が死亡したかどうか</summary>
    public bool killed;

    /// <summary>蓄積により発症した状態異常（発症なしなら StatusEffectId.None）</summary>
    public StatusEffectId triggeredEffect;
}
```

### 9.1 DamageResult の使用例

```csharp
// HitBox 内での使用例
DamageResult result = receiver.ApplyDamage(data);

if (result.wasParried)
{
    // パリィエフェクト再生
    EffectManager.Play(EffectType.Parry, hitPoint);
    SoundManager.Play(SoundType.ParrySuccess);
}
else if (result.wasGuarded)
{
    // ガードエフェクト再生
    EffectManager.Play(EffectType.Guard, hitPoint);
    SoundManager.Play(SoundType.GuardHit);

    if (result.guardBroken)
    {
        // ガードブレイクエフェクト
        EffectManager.Play(EffectType.GuardBreak, hitPoint);
        SoundManager.Play(SoundType.GuardBreak);
    }
}
else
{
    // 通常ヒットエフェクト
    EffectManager.Play(EffectType.Hit, hitPoint);
    SoundManager.Play(SoundType.Hit);

    if (result.killed)
    {
        // 撃破エフェクト
        EffectManager.Play(EffectType.Kill, hitPoint);
    }
}

// ダメージ数値表示
if (result.totalDamage > 0)
{
    DamageNumberUI.Show(hitPoint, result.totalDamage,
        result.triggeredEffect != StatusEffectId.None);
}
```

## 13. アーマーシステム

### 13.1 二層アーマー構造

キャラクターのアーマーは**ベースアーマー**と**行動アーマー**の二層で構成される。

| 層 | 説明 | 設定箇所 |
|---|---|---|
| ベースアーマー | キャラクター固有の基礎アーマー値。プレイヤーは装備/コアで変動 | CharacterInfo.maxArmor |
| 行動アーマー | 行動中に一時的に付与されるアーマー。ActionEffect で設定 | ActionEffect (type=Armor) |

### 13.2 アーマー消費優先度

```
被弾時のアーマー削り処理:

  1. 行動アーマーが有効か確認（ActionEffectProcessor.Evaluate）
  2. 行動アーマー > 0 → 行動アーマーから優先消費
  3. 行動アーマーで吸収しきれない分 → ベースアーマーから消費
  4. 両方とも 0 以下 → アーマーブレイク（1.3倍ダメージボーナス）

  ※ 行動アーマーが残っている限りベースアーマーは減らない
```

### 13.3 ベースアーマー回復

```
回復条件:
  - 被弾後 armorRecoveryDelay 秒経過で回復開始
  - 回復速度: armorRecoveryRate/秒
  - maxArmor を超過しない

  CharacterInfo:
    maxArmor: float            // ベースアーマー最大値
    armorRecoveryRate: float   // 回復速度（/秒）
    armorRecoveryDelay: float  // 被弾後の回復開始遅延（秒）
```

### 13.4 行動アーマーは回復しない

行動アーマーは ActionEffect の有効区間のみ存在する一時的な値。
行動が終了すれば自動的に消滅し、回復処理は不要。

## 14. 行動特殊効果（ActionEffect）システム

### 14.1 概要

全行動（攻撃、回避、ガード、スキル、魔法など）に共通で「開始時間〜持続時間」の特殊効果を複数設定できる汎用システム。

```csharp
[Serializable]
public struct ActionEffect
{
    public ActionEffectType type;   // 効果の種類
    public float startTime;         // 発生タイミング（秒）
    public float duration;          // 持続時間（秒）
    public float value;             // 効果量
}

public enum ActionEffectType : byte
{
    Armor,            // 行動アーマー（value = アーマー量）
    SuperArmor,       // スーパーアーマー（怯まない）
    Invincible,       // 完全無敵
    DamageReduction,  // ダメージ軽減（value = 軽減率 0-1）
    GuardPoint,       // ガードポイント（特定フレームだけガード判定）
}
```

### 14.2 設定箇所

| データ | 用途 |
|---|---|
| AttackInfo.motionInfo.actionEffects | 攻撃全体の特殊効果 |
| AttackMotionData.actionEffects | コンボステップごとの特殊効果 |
| AbilityData（回避等） | Ability個別の特殊効果 |

### 14.3 効果の複合ルール

| 効果タイプ | 同時に複数ある場合 |
|---|---|
| Armor | **合算**（20 + 15 = 35） |
| DamageReduction | **最大値**を採用 |
| SuperArmor / Invincible / GuardPoint | **OR**（1つでもあればtrue） |

### 14.4 使用例

**大振り攻撃（予備動作中アーマー→攻撃中スーパーアーマー）**
```
actionEffects[0] = { Armor, start=0.0, dur=0.3, value=30 }
actionEffects[1] = { SuperArmor, start=0.3, dur=0.5 }
```

**回避ロール（序盤に無敵フレーム）**
```
actionEffects[0] = { Invincible, start=0.05, dur=0.1 }
```

**ガードポイント付きカウンター**
```
actionEffects[0] = { GuardPoint, start=0.0, dur=0.2 }
```

**ボスのチャージ攻撃（チャージ中ダメージ軽減50%）**
```
actionEffects[0] = { DamageReduction, start=0.0, dur=1.5, value=0.5 }
```

### 14.5 処理クラス

`ActionEffectProcessor`（static class）が ActionEffect[] と経過時間から `EffectState` を返す。
DamageReceiver は被弾時にこの EffectState を参照して無敵判定・ダメージ軽減・アーマー優先消費を行う。

## 15. 被弾リアクションシステム

### 15.1 概要

被弾時のキャラクターリアクション（怯み、吹き飛ばし、ガードブレイク）を決定するシステム。
`HitReactionLogic`（static class）がアーマー状態・特殊効果・攻撃属性からリアクション種別を判定する。

### 15.2 HitReaction enum

| 値 | 説明 | 対応ActState |
|---|---|---|
| None | リアクションなし | 状態遷移なし |
| Flinch | 怯み（短い硬直 ~0.3秒） | ActState.Flinch |
| Knockback | 吹き飛ばし（着地→起き上がり完了まで） | ActState.Knockbacked |
| GuardBreak | ガードブレイク（固定1秒硬直） | ActState.GuardBroken |

### 15.3 判定フロー

```
被弾
 ├─ ガードブレイク → HitReaction.GuardBreak
 ├─ ガード成功（Guarded/JustGuard/EnhancedGuard） → HitReaction.None
 ├─ SuperArmor → HitReaction.None
 ├─ ヒットスタン中（Flinch/GuardBroken/Stunned）+ 吹き飛ばし力あり
 │   ├─ KnockbackImmunity → HitReaction.Flinch
 │   └─ 通常 → HitReaction.Knockback（アーマー無視）
 ├─ アーマー（ベース + アクション） > 0 → HitReaction.None
 ├─ 吹き飛ばし力あり
 │   ├─ KnockbackImmunity → HitReaction.Flinch
 │   └─ 通常 → HitReaction.Knockback
 └─ それ以外 → HitReaction.Flinch
```

### 15.4 ActState 拡張

| ActState | 説明 | 硬直時間 |
|---|---|---|
| Flinch | 怯み | 0.3秒（固定） |
| Knockbacked | 吹き飛ばし | 着地 + 起き上がりモーション完了まで |
| GuardBroken | ガードブレイク | 1.0秒（固定） |
| Stunned | スタン（状態異常蓄積による気絶） | 状態異常のduration依存 |

### 15.5 起き上がり中の保護

Knockback状態の起き上がりフェーズ中:
- 怯まない（Flinch無効）
- アーマー削りなし
- 状態機械側で管理（DamageReceiver外）

### 15.6 ヒットスタン連鎖

Flinch/GuardBroken/Stunned 中に吹き飛ばし攻撃を受けた場合:
- アーマーを**無視**してKnockbackに遷移（KnockbackImmunity時はFlinch維持）
- SuperArmor中は免除

### 15.7 KnockbackImmunity（吹き飛ばし無効）

`ActionEffectType.KnockbackImmunity` で付与。
- 吹き飛ばし攻撃がFlinchに変換される
- 一部の敵は永続（解除不能）
- プレイヤーはコア装備で付与可能
- SuperArmorとは異なる（KnockbackImmunityでも怯みは発生する）

### 15.8 吹き飛ばし判定基準

攻撃データの `knockbackForce` ベクトルの大きさで判定:
- `knockbackForce.sqrMagnitude > 0.01` → 吹き飛ばし属性あり
- それ以外 → 吹き飛ばし属性なし（通常攻撃 → Flinch）
- 吹き飛ばし方向は打ち上げも含む（Launch = Knockbackの方向違い）

## 16. ガード方向・ガード攻撃

### 16.1 ガード方向（GuardDirection）

| 値 | 説明 |
|---|---|
| Front | 前方のみガード |
| Back | 後方のみガード |
| Both | 前後両方ガード |

`GuardStats.guardDirection` に設定。攻撃方向と不一致の場合 `GuardResult.NoGuard`。
攻撃方向は `DamageData.knockbackForce` の向きとキャラのfacing（`localScale.x`）を比較して判定。

### 16.2 ガード攻撃（GuardAttack）

`ActionEffectType.GuardAttack` で付与される特殊効果。
- ガード中にこの効果がアクティブな場合、`attackPower > guardStrength` でもブレイクしない
- `GuardResult.GuardBreak` → `GuardResult.Guarded` に昇格

### 16.3 ジャストガード回復

ジャストガード成功時に以下を回復:
- スタミナ: +15（maxStaminaでクランプ）
- アーマー: +10（maxArmorでクランプ）
- `GuardJudgmentLogic.ApplyJustGuardRecovery()` で適用

## 17. 状況ダメージボーナス

### 17.1 概要

特定の状況で攻撃がヒットした場合にダメージ倍率ボーナスが適用される。
**重複なし**: 複数条件を同時に満たす場合は最大倍率のもののみ適用。

### 17.2 ボーナス種別

| 種別 | 倍率 | 条件 | 優先度 |
|---|---|---|---|
| Counter | 1.3x | 攻撃中の敵にヒット（AttackPrep/Attacking/AttackRecovery） | 1（最高） |
| Backstab | 1.25x | 背面からの攻撃 | 2 |
| StaggerHit | 1.2x | 怯み/スタン/ガードブレイク中の敵にヒット | 3（最低） |

### 17.3 適用条件

- **ガード成功時は適用しない**: `GuardResult.NoGuard` または `GuardResult.GuardBreak` の場合のみ
- ガード軽減の**前**にrawDamageに乗算
- `DamageResult.situationalBonus` にどのボーナスが適用されたか記録（UI表示用）

### 17.4 実装

`SituationalBonusLogic.Evaluate()` — 純粋ロジック
- 入力: isTargetAttacking, isAttackFromBehind, isTargetInHitstun
- 出力: (float multiplier, SituationalBonus bonus)

`SituationalBonusLogic.IsTargetAttacking()` — ActStateが攻撃中か判定
- `AttackPrep`, `Attacking`, `AttackRecovery` → true
