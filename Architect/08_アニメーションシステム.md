# 08 - アニメーションシステム

## 1. 概要

本ドキュメントでは、全キャラクター共通のアニメーション管理アーキテクチャを定義する。
AnimatorControllerの状態設計、ActionAnimationProfileによるクリップ管理、AnimatorBridgeによるパラメータ制御を一貫して扱う。

### 1.1 設計原則

| 原則 | 説明 |
|---|---|
| **共通AnimatorBridge** | 全キャラクターが同一のAnimatorBridgeコンポーネントを使用する。キャラ固有のC#コードは書かない |
| **ベースController + OverrideController** | 1つのベースAnimatorControllerを全キャラで共有し、AnimatorOverrideControllerでクリップを差し替える |
| **パラメータ駆動** | 移動系ステートはパラメータ（Speed, IsGrounded等）で遷移。AnyState矢印は被弾リアクション等の共通割り込みのみ |
| **Actionサブステートマシン** | キャラ固有の行動（攻撃・スキル・魔法等）はサブステート内のスロット（Action_0~7）で管理 |
| **Profile一元管理** | アニメーションクリップの割り当てはActionAnimationProfile（ScriptableObject）で一元管理。AIInfoとAnimatorの不整合を防止 |

### 1.2 システム全体の位置づけ

```
ActionBase / IAbility / HitReactionLogic
        │
        │ TriggerAction(type), SetSpeed(x) 等を呼ぶ
        ▼
   AnimatorBridge（共通MonoBehaviour）
        │
        │ animator.SetFloat, SetTrigger 等に変換
        ▼
   Animator（Unity標準）
        │
        │ AnimatorOverrideController でクリップ差し替え済み
        ▼
   AnimationClip 再生
```

### 1.3 関心の分離

| 層 | 責務 | 例 |
|---|---|---|
| **ActState（ゲームプレイ層）** | 行動可否の判定。「今ガード可能か？」「行動中断されたか？」 | ActState.Flinch → 行動不可 |
| **AnimatorBridge（橋渡し層）** | ゲームロジックのパラメータをAnimatorに伝達 | SetSpeed(3.5f), TriggerFlinch() |
| **AnimatorController（視覚層）** | 視覚カテゴリの状態遷移。「走りアニメ」「怯みアニメ」 | Speed > 0 → Run ステート |

ActStateは「行動の種類」（Patrol/Flee/Flank等）を区別しない。
AnimatorControllerも区別しない。全て同じ「走り」として扱う。

## 2. AnimatorController 共通ステート一覧

### 2.1 移動系ステート（パラメータ駆動で遷移）

| ステート | 遷移条件 | 備考 |
|---|---|---|
| **Idle** | Speed == 0 && IsGrounded | デフォルトステート |
| **Run** | Speed > 0 && IsGrounded | 全地上キャラ共通 |
| **Dash** | IsDashing == true | 無敵フレーム付き、スタミナ消費 |
| **Jump** | !IsGrounded && VelocityY > 0 | 上昇中 |
| **Fall** | !IsGrounded && VelocityY <= 0 | 下降中 |
| **Landing** | IsGrounded（trueに変化） | 着地モーション。短い再生後 → Idle/Run |

※ 2Dボーンアニメーションのためブレンドツリーは使用しない。各ステートは個別クリップ。

### 2.2 防御系ステート（パラメータ駆動）

| ステート | 遷移条件 | 備考 |
|---|---|---|
| **Guard** | IsGuarding == true | ガード持ちのみ使用。持たないキャラは無視 |
| **ParryWindow** | ParryTrigger | ガード開始直後のパリィ受付モーション |

### 2.3 被弾リアクション系ステート（AnyState → Trigger で強制割り込み）

| ステート | トリガー | 硬直時間 | 出口 |
|---|---|---|---|
| **Flinch** | FlinchTrigger | 0.3秒固定 | → Idle |
| **Knockback** | KnockbackTrigger | 着地まで（空中） | → KnockbackLanding |
| **KnockbackLanding** | IsGrounded == true（Knockback中） | 着地衝撃 | → GetUp |
| **GetUp** | KnockbackLanding完了後 | 起き上がりモーション | → Idle（無敵付き） |
| **GuardBroken** | GuardBrokenTrigger | 1.0秒固定 | → Idle |
| **Stunned** | StunnedTrigger | 状態異常duration依存 | → Idle |
| **Dead** | IsDead == true | 永続 | 遷移なし |

### 2.4 Actionサブステートマシン

| ステート | 用途 |
|---|---|
| **Action_0 ~ Action_7** | キャラ固有行動（攻撃、スキル、キャスト等） |

ActionTrigger + ActionType(int) で進入。モーション終了 → Exit → Idle。
OverrideControllerでクリップ差し替え。使わないスロットは空のまま。

#### スロット分割規約（プレイヤー）

```
Action_0 ~ Action_4:  右手武器用（5枠）
Action_5 ~ Action_7:  左手装備用（3枠）
```

| 装備パターン | 右 (0~4) | 左 (5~7) |
|---|---|---|
| 両手大剣 | 5枠フル使用 | 未使用 |
| 片手剣 + 盾 | 斬り×2, 突き, スキル | バッシュ, パリィ |
| 片手剣 + 魔導書 | 斬り×2, 突き, スキル | 魔法詠唱, 魔法発動 |
| 片手剣のみ | 斬り×3, スキル | 未使用 |

#### 多フェーズ行動（詠唱→発動、チャージ→解放）

ActionSlotの `animSlotIndex`（メインフェーズ）と `subAnimSlotIndex`（サブフェーズ）でスロットを2つ使用する。

```
通常攻撃:     animSlotIndex=0, subAnimSlotIndex=-1
  → Action_0 再生 → 終了

魔法:         animSlotIndex=4, subAnimSlotIndex=5
  → Action_4（詠唱ループ） → 詠唱完了 → Action_5（発動） → 終了

チャージ攻撃: animSlotIndex=1, subAnimSlotIndex=2
  → Action_1（溜めループ） → 解放 → Action_2（振り下ろし） → 終了
```

### 2.5 特殊演出リザーブ（ステートのみ用意、クリップは空）

| ステート | 想定用途 | 備考 |
|---|---|---|
| **Special_0** | 形態変化・覚醒演出 | ボス用 |
| **Special_1** | 登場演出 | ボス・中ボス用 |
| **Special_2** | 撃破演出（Deadと別） | 特殊死亡モーション |
| **Special_3** | 予備 | |

AnyState → SpecialTrigger + SpecialType(int) で進入。

                    │          AnyState                    │

## 5. ActionAnimationProfile（ScriptableObject）

キャラクター（または装備）ごとに1つ作成する。アクションスロットのクリップ割り当てと移動系クリップを一元管理する。

### 5.1 構造

```csharp
/// <summary>
/// キャラクター（or 装備）ごとのアニメーションクリップ設定。
/// 移動系の共通クリップ + Actionスロットへのクリップ割り当てを一元管理する。
/// AnimatorBridge がこのProfileを受け取り、OverrideControllerを自動生成する。
/// </summary>
[CreateAssetMenu(menuName = "Game/ActionAnimationProfile")]
public class ActionAnimationProfile : ScriptableObject
{
    // ========== 移動系クリップ（持ち方で変わる） ==========

    [Header("移動系")]
    public AnimationClip idle;
    public AnimationClip run;
    public AnimationClip dash;
    public AnimationClip jump;
    public AnimationClip fall;
    public AnimationClip landing;
    public AnimationClip guard;

    // ========== Actionスロット ==========

    [Header("アクション")]
    public SlotEntry[] actionSlots;

    [Serializable]
    public struct SlotEntry
    {
        /// <summary>人間用ラベル（"斬り1段目", "火球詠唱" 等）</summary>
        public string label;

        /// <summary>Action_N の N</summary>
        public int slotIndex;

        /// <summary>再生するアニメーションクリップ</summary>
        public AnimationClip clip;

        /// <summary>詠唱・チャージ用ループフラグ</summary>
        public bool isLoop;
    }
}
```

### 5.2 設定フロー

```
1. ActionAnimationProfile を作る（Inspectorで設定）
   ┌──────────────────────────────────────────────┐
   │ SkeletonWarrior_Profile.asset                │
   │                                              │
   │ [移動系]                                      │
   │   idle: skeleton_idle.anim                   │
   │   run:  skeleton_run.anim                    │
   │   dash: skeleton_dash.anim                   │
   │   ...                                        │
   │                                              │
   │ [アクション]                                   │
   │   [0] label:"斬り"    slot:0  clip:slash     │
   │   [1] label:"突き"    slot:1  clip:thrust    │
   │   [2] label:"盾殴り"  slot:2  clip:bash      │
   └──────────────────────────────────────────────┘

2. ランタイム初期化時にAnimatorBridgeがProfileを受け取る
   → OverrideControllerにクリップを自動セット

3. AIInfo側のActionSlotは同じslotIndexを参照
   → animSlotIndex: 0 は Profile の slot:0 と一致
```

### 5.3 Profileが唯一の定義元

```
           ActionAnimationProfile（1つの場所）
            ┌──────────────┐
            │ slot:0 = 斬り │──→ clip: slash.anim
            │ slot:1 = 突き │──→ clip: thrust.anim
            └──────┬───────┘
                   │
        ┌──────────┼──────────┐
        ▼                     ▼
  AIInfo.ActionSlot     AnimatorOverride
  animSlotIndex: 0      Action_0 → slash
  animSlotIndex: 1      Action_1 → thrust
```

AIInfoはslot番号を参照するだけ。OverrideControllerはProfileから自動生成。
手動で両方設定する必要がないので不整合が起きない。

## 6. プレイヤーの装備パターン

### 6.1 持ち方による分類

| 持ち方 | 移動モーション | 右手アクション | 左手アクション |
|---|---|---|---|
| 両手持ち | 両手用idle/run | Action_0~4 | 未使用 |
| 片手持ち（左なし） | 片手用idle/run | Action_0~4 | 未使用 |
| 片手 + 左装備 | 片手用idle/run | Action_0~4 | Action_5~7 |

### 6.2 Profile合成（右装備 + 左装備）

```
右 (剣Profile):              左 (盾Profile):
  idle: idle_onehanded         ← 右が優先（全身モーション）
  run:  run_onehanded          ← 右が優先
  Action_0: 斬り1段            Action_5: シールドバッシュ
  Action_1: 斬り2段            Action_6: パリィ構え
  Action_2: 突き
  Action_3: スキル
```

右ProfileがIdle/Run等の全身モーションを提供し、左Profileはアクションスロットのみ追加する。

### 6.3 装備変更時の処理

```csharp
// 武器変更時
public void OnWeaponChanged(WeaponInfo rightWeapon, EquipmentInfo leftEquip)
{
    bridge.ApplyDualProfile(rightWeapon.actionAnimProfile, leftEquip?.actionAnimProfile);
}
```

## 7. AnimatorBridge（共通コンポーネント）

### 7.1 クラス定義

```csharp
/// <summary>
/// 全キャラクター共通のAnimator制御コンポーネント。
/// ゲームロジック（ActionBase, IAbility, HitReactionLogic等）からの
/// パラメータ設定を受け、Animatorに伝達する。
/// キャラ固有のC#コードは不要。差分はAnimatorOverrideControllerのクリップ差し替えで吸収する。
/// </summary>
public class AnimatorBridge : MonoBehaviour
{
    [SerializeField] private Animator _animator;

    // ========== パラメータハッシュ（定数、全キャラ共通） ==========

    private static readonly int s_Speed = Animator.StringToHash("Speed");
    private static readonly int s_VelocityY = Animator.StringToHash("VelocityY");
    private static readonly int s_IsGrounded = Animator.StringToHash("IsGrounded");
    private static readonly int s_IsDashing = Animator.StringToHash("IsDashing");
    private static readonly int s_IsGuarding = Animator.StringToHash("IsGuarding");
    private static readonly int s_IsDead = Animator.StringToHash("IsDead");
    private static readonly int s_ActionTrigger = Animator.StringToHash("Action");
    private static readonly int s_ActionType = Animator.StringToHash("ActionType");
    private static readonly int s_FlinchTrigger = Animator.StringToHash("Flinch");
    private static readonly int s_KnockbackTrigger = Animator.StringToHash("Knockback");
    private static readonly int s_GuardBrokenTrigger = Animator.StringToHash("GuardBroken");
    private static readonly int s_StunnedTrigger = Animator.StringToHash("Stunned");
    private static readonly int s_ParryTrigger = Animator.StringToHash("Parry");
    private static readonly int s_SpecialTrigger = Animator.StringToHash("Special");
    private static readonly int s_SpecialType = Animator.StringToHash("SpecialType");

    // ========== Profile適用 ==========

    /// <summary>
    /// 単一Profileを適用する（敵・NPC、両手武器プレイヤー）。
    /// </summary>
    public void ApplyProfile(ActionAnimationProfile profile)
    {
        ApplyDualProfile(profile, null);
    }

    /// <summary>
    /// 右装備 + 左装備のProfileを合成して適用する。
    /// 移動系クリップは右Profileから取得。左ProfileはActionスロットのみ追加。
    /// </summary>
    public void ApplyDualProfile(
        ActionAnimationProfile rightProfile,
        ActionAnimationProfile leftProfile)
    {
        AnimatorOverrideController overrideCtrl =
            new AnimatorOverrideController(_animator.runtimeAnimatorController);

        // 移動系は右Profileから（全身モーション）
        if (rightProfile.idle != null)    overrideCtrl["Idle"] = rightProfile.idle;
        if (rightProfile.run != null)     overrideCtrl["Run"] = rightProfile.run;
        if (rightProfile.dash != null)    overrideCtrl["Dash"] = rightProfile.dash;
        if (rightProfile.jump != null)    overrideCtrl["Jump"] = rightProfile.jump;
        if (rightProfile.fall != null)    overrideCtrl["Fall"] = rightProfile.fall;
        if (rightProfile.landing != null) overrideCtrl["Landing"] = rightProfile.landing;
        if (rightProfile.guard != null)   overrideCtrl["Guard"] = rightProfile.guard;

        // 右手アクション
        foreach (ActionAnimationProfile.SlotEntry entry in rightProfile.actionSlots)
        {
            overrideCtrl[$"Action_{entry.slotIndex}"] = entry.clip;
        }

        // 左手アクション
        if (leftProfile != null)
        {
            foreach (ActionAnimationProfile.SlotEntry entry in leftProfile.actionSlots)
            {
                overrideCtrl[$"Action_{entry.slotIndex}"] = entry.clip;
            }
        }

        _animator.runtimeAnimatorController = overrideCtrl;
    }

    // ========== 移動系 ==========

    public void SetSpeed(float speed) => _animator.SetFloat(s_Speed, speed);
    public void SetVerticalVelocity(float vy) => _animator.SetFloat(s_VelocityY, vy);
    public void SetGrounded(bool grounded) => _animator.SetBool(s_IsGrounded, grounded);
    public void SetDashing(bool dashing) => _animator.SetBool(s_IsDashing, dashing);
    public void SetGuarding(bool guard) => _animator.SetBool(s_IsGuarding, guard);

    // ========== Action（サブステート進入） ==========

    /// <summary>
    /// Actionサブステートのスロットを指定してアニメーション再生。
    /// ActionBase handlerから slot.animSlotIndex を渡す。
    /// </summary>
    public void TriggerAction(int slotIndex)
    {
        _animator.SetInteger(s_ActionType, slotIndex);
        _animator.SetTrigger(s_ActionTrigger);
    }

    // ========== 共通リアクション（AnyStateから割り込み） ==========

    public void TriggerFlinch() => _animator.SetTrigger(s_FlinchTrigger);
    public void TriggerKnockback() => _animator.SetTrigger(s_KnockbackTrigger);
    public void TriggerGuardBroken() => _animator.SetTrigger(s_GuardBrokenTrigger);
    public void TriggerStunned() => _animator.SetTrigger(s_StunnedTrigger);
    public void TriggerParry() => _animator.SetTrigger(s_ParryTrigger);
    public void SetDead(bool dead) => _animator.SetBool(s_IsDead, dead);

    // ========== 特殊演出 ==========

    public void TriggerSpecial(int specialType)
    {
        _animator.SetInteger(s_SpecialType, specialType);
        _animator.SetTrigger(s_SpecialTrigger);
    }
}
```

## 8. ActionSlot の拡張

### 8.1 アニメーションスロット追加

```csharp
[Serializable]
public struct ActionSlot
{
    /// <summary>行動の種別（Attack/Cast/Instant/Sustained/Broadcast）</summary>
    public ActionExecType execType;

    /// <summary>行動のサブタイプID</summary>
    public int paramId;

    /// <summary>行動のパラメータ値（強度・持続時間等）</summary>
    public float paramValue;

    /// <summary>UI表示名</summary>
    public string displayName;

    /// <summary>メインアニメーションスロット（Action_Nの N）。-1 = アニメなし</summary>
    public int animSlotIndex;

    /// <summary>サブアニメーションスロット（詠唱→発動の発動側）。-1 = 単一フェーズ</summary>
    public int subAnimSlotIndex;
}
```

### 8.2 ActionBase handler での使用

```csharp
// 通常攻撃の場合
public override void Execute(int ownerHash, int targetHash, ActionSlot slot)
{
    // メインアニメーション再生
    if (slot.animSlotIndex >= 0)
    {
        _bridge.TriggerAction(slot.animSlotIndex);
    }
    // ... 攻撃ロジック
}

// 魔法詠唱の場合
public override void Execute(int ownerHash, int targetHash, ActionSlot slot)
{
    // 詠唱ループ開始
    if (slot.animSlotIndex >= 0)
    {
        _bridge.TriggerAction(slot.animSlotIndex);
    }
    // ... 詠唱タイマー開始
}

// 詠唱完了時
private void OnCastComplete(ActionSlot slot)
{
    // 発動アニメーション再生
    if (slot.subAnimSlotIndex >= 0)
    {
        _bridge.TriggerAction(slot.subAnimSlotIndex);
    }
    // ... 魔法発動ロジック
}
```

## 9. キャラ種別ごとの設定差分

### 9.1 全キャラ共通（C#コード変更不要）

| キャラ種別 | Profile設定 | AnimatorController |
|---|---|---|
| スライム | SlimeProfile（Action_0: 体当たり） | ベースController + Override |
| 骸骨剣士 | SkeletonProfile（Action_0~2） | ベースController + Override |
| ボス（形態1） | BossPhase1Profile（Action_0~7フル） | ベースController + Override |
| ボス（形態2） | 形態変化時にBossPhase2Profileに差し替え | 同上 |
| プレイヤー（大剣） | GreatswordProfile（両手用idle/run + Action_0~4） | ベースController + Override |
| プレイヤー（剣+盾） | SwordProfile + ShieldProfile 合成 | ベースController + Override |

### 9.2 特殊ケース（AnimatorBridge継承が必要な場合）

- 多関節ボス（部位別アニメーション）
- 騎乗キャラ（騎手 + 騎獣で別Animator）

上記のような標準パラメータセットでは表現できないケースのみ、AnimatorBridgeを継承して拡張する。

## 10. エフェクト・音声管理（ハイブリッド方式）

### 10.1 設計方針

| 担当 | 役割 |
|---|---|
| **AnimationClip（AnimationEvent）** | 「いつ」再生するかを決定。`OnActionEvent(int eventId)` を発火 |
| **ActionAnimationProfile（ActionEventData）** | 「何を」再生するかを決定。VFX/SFXのAddressable参照を保持 |

クリップを差し替えると対応するProfileのイベントデータも切り替わるため、不整合が起きない。

### 10.2 ActionEventData 構造体

```csharp
[Serializable]
public struct ActionEventData
{
    /// <summary>AnimationEventのintパラメータと一致させる</summary>
    public int eventId;

    /// <summary>再生するVFXプレハブ（Addressable）</summary>
    public AssetReferenceGameObject vfxRef;

    /// <summary>VFX生成位置（キャラ相対）</summary>
    public Vector2 vfxOffset;

    /// <summary>再生するSE（Addressable）</summary>
    public AssetReferenceT<AudioClip> sfxRef;

    /// <summary>SE音量</summary>
    [Range(0f, 1f)]
    public float sfxVolume;
}
```

### 10.3 設定例

```
剣の斬りクリップ:
  フレーム5:  AnimationEvent → OnActionEvent(0)  → swing_sfx再生
  フレーム12: AnimationEvent → OnActionEvent(1)  → slash_vfx生成

盾バッシュクリップ:
  フレーム3:  AnimationEvent → OnActionEvent(0)  → bash_sfx + bash_vfx
```

### 10.4 共通リアクションエフェクト

被弾リアクション（Flinch, Knockback等）のVFX/SFXはProfile外でAnimatorBridgeのSerializeFieldに直接設定する。
全キャラ共通のため、Profileに持たせると冗長になる。

```csharp
[Header("共通リアクションエフェクト")]
[SerializeField] private AssetReferenceGameObject _flinchVfx;
[SerializeField] private AssetReferenceGameObject _knockbackVfx;

[Header("共通リアクション音声")]
[SerializeField] private AssetReferenceT<AudioClip> _flinchSfx;
[SerializeField] private AssetReferenceT<AudioClip> _knockbackSfx;
```
