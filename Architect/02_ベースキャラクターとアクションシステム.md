# 02 - ベースキャラクターとアクションシステム

## 1. 概要

本ドキュメントでは、全キャラクターの基盤となる **BaseCharacter** クラスと、行動を拡張する **アクションシステム（Ability）** の設計を定義する。

設計の核心は以下の3点である。

| 原則 | 説明 |
|---|---|
| **データは持たない** | BaseCharacter は `objectHash` と `charIndex` のみ保持。HP/MP/スタミナ等のデータは `GameManager.Data`（SoACharaDataDic）経由でアクセスする |
| **コンポーネント拡張** | ジャンプ・ダッシュ・飛行等のアクションは `IAbility` コンポーネントとして GameObject に追加。キャラ種別ごとに組み合わせを変える |
| **AI と入力の統一** | AI 判定結果（MovementInfo）もプレイヤー入力も、同じ Ability インターフェースを通じて実行される |

既存の `BaseController` を発展させ、コーギーエンジンの `CorgiController` + `CharacterAbility` を完全に置き換える。

## 2. BaseCharacter クラス

### 2.1 クラス定義

```csharp
using UnityEngine;

namespace CharacterController
{
    /// <summary>
    /// 全キャラクター共通の基盤クラス。
    /// 自身のデータは保持せず、SoAコンテナへのインデックスのみを持つ。
    /// AI判定結果（MovementInfo）を受け取り、Abilityコンポーネントに処理を委譲する。
    /// </summary>
    public class BaseCharacter : MonoBehaviour
    {
        // ========== 識別子 ==========

        /// <summary>SoACharaDataDic 内のインデックス。全データアクセスの起点。</summary>
        public int CharIndex { get; private set; }

        /// <summary>GameObject のハッシュ値。ハッシュテーブル経由のアクセスに使用。</summary>
        public int ObjectHash { get; private set; }

        // ========== ScriptableObject 参照 ==========

        /// <summary>キャラクター固有設定（HP/MP基本値、移動パラメータ等）</summary>
        [SerializeField] private CharacterInfo _characterInfo;

        /// <summary>AI設定（判定ルール、モード、クールタイム）</summary>
        [SerializeField] private AIInfo _aiInfo;

        // ========== AI判定結果 ==========

        /// <summary>AI判定から受け取った最新の判定結果</summary>
        public MovementInfo LastJudgeResult { get; set; }

        // ========== コンポーネント参照 ==========

        private Rigidbody2D _rb;
        private IAbility[] _abilities;
        private WeaponHolder _weaponHolder;
        private DamageReceiver _damageReceiver;
        private SensorSystem _sensorSystem;

        // ========== 状態プロパティ ==========

        /// <summary>接地判定</summary>
        public bool IsGrounded { get; private set; }

        /// <summary>右向きかどうか</summary>
        public bool IsFacingRight { get; private set; }

        /// <summary>生存判定。SoAコンテナのHP値を直接参照する。</summary>
        public bool IsAlive => GameManager.Data.GetBaseInfo(ObjectHash).currentHp > 0;
    }
}
```

**設計意図**: `BaseCharacter` は「自分が誰であるか（ObjectHash / CharIndex）」だけを知っている。HP や MP のフィールドを持たないため、データの二重管理が発生しない。

### 2.2 初期化フロー

```csharp
/// <summary>
/// キャラクター生成時に呼ばれる初期化処理。
/// SoAコンテナへの登録と、全コンポーネントの取得・初期化を行う。
/// </summary>
protected void Initialize()
{
    // 1. ハッシュ値を取得
    ObjectHash = gameObject.GetHashCode();

    // 2. CharacterInfo から各構造体を生成
    var baseInfo = _characterInfo.CreateBaseInfo();       // CharacterBaseInfo
    var solidData = _characterInfo.CreateSolidData();     // SolidData
    var moveStatus = _characterInfo.CreateMoveStatus();   // MoveStatus
    var atkStatus = _characterInfo.CreateAtkStatus();     // CharacterAtkStatus
    var defStatus = _characterInfo.CreateDefStatus();     // CharacterDefStatus
    var staminaInfo = _characterInfo.CreateStaminaInfo(); // StaminaInfo
    var stateInfo = new CharacterStateInfo();             // 初期状態
    var coldLog = new CharacterColdLog();                 // クールダウン初期値
    var recognition = new RecognitionData();              // 認識データ（空）

    // 3. AIManager 経由で SoACharaDataDic に一括登録
    //    戻り値がコンテナ内インデックスとなる
    CharIndex = AIManager.instance.CharacterAdd(
        ObjectHash, this,
        baseInfo, solidData, atkStatus, defStatus,
        stateInfo, moveStatus, coldLog, recognition, staminaInfo
    );

    // 4. コンポーネント取得
    _rb = GetComponent<Rigidbody2D>();
    _abilities = GetComponents<IAbility>();
    _weaponHolder = GetComponent<WeaponHolder>();
    _damageReceiver = GetComponent<DamageReceiver>();
    _sensorSystem = GetComponent<SensorSystem>();

    // 5. 各 Ability を初期化（BaseCharacter 参照を渡す）
    foreach (var ability in _abilities)
    {
        ability.Initialize(this);
    }

    // 6. WeaponHolder にオーナーハッシュを設定
    if (_weaponHolder != null)
    {
        _weaponHolder.SetOwner(ObjectHash);
    }
}
```

### 2.3 データアクセス（GameManager.Data 経由）

BaseCharacter および関連コンポーネントは、全てのデータを `GameManager.Data`（SoACharaDataDic のラッパー）経由で取得する。GetComponent によるコンポーネント検索は一切行わない。

```csharp

    => GameManager.Data.GetStaminaInfo(ObjectHash).currentStamina;

    => GameManager.Data.GetBaseInfo(ObjectHash).hpRatio;

/// <summary>MP割合を取得（0〜100）。</summary>

    => GameManager.Data.GetBaseInfo(ObjectHash).mpRatio;
```


**データアクセスパターンの比較**:

```
【従来方式（NG）】
BaseCharacter がローカルに HP を保持
  → 変更時に SoAコンテナと同期が必要
  → 同期漏れでデータ不整合のリスク

【本設計（OK）】
BaseCharacter は ObjectHash のみ保持
  → GameManager.Data.GetBaseInfo(ObjectHash).currentHp
  → 常に SoAコンテナの値を直接参照
  → データの一元管理が保証される
```

### 2.4 毎フレーム処理

```csharp
/// <summary>
/// FixedUpdate で物理演算に同期した処理を行う。
/// AI判定結果 → Ability実行 → 物理適用 の順で処理する。
/// </summary>
private void FixedUpdate()
{
    if (!IsAlive) return;

    // 1. 接地判定を更新
    UpdateGroundCheck();

    // 2. AI判定結果を処理（AIキャラの場合）
    //    プレイヤーの場合は InputHandler が LastJudgeResult を書き換える
    ProcessMovementInfo();

    // 3. 向きの更新
    UpdateFacingDirection();
}

/// <summary>
/// AI判定結果（MovementInfo）に基づいてアクションを実行する。
/// actNum を AbilityType にマッピングし、対応する Ability に処理を委譲。
/// </summary>
private void ProcessMovementInfo()
{
    var info = LastJudgeResult;

    // 判定結果にモード変更がある場合
    if ((info.result & JudgeResult.モード変更あり) != 0)
    {
        HandleModeChange(info.changeMode);
    }

    // actNum に対応するアビリティを実行
    var abilityType = MapActNumToAbility(info.actNum);
    ExecuteAbility(abilityType, info);

    // 判定結果のデータ反映
    info.JudgeUpdate(ObjectHash);
}

/// <summary>
/// AbilityType に対応する IAbility を検索して実行する。
/// </summary>
private void ExecuteAbility(AbilityType type, MovementInfo info)
{
    for (int i = 0; i < _abilities.Length; i++)
    {
        if (_abilities[i].Type == type && _abilities[i].CanExecute())
        {
            _abilities[i].Execute(info);
            return;
        }
    }
}

/// <summary>
/// 接地判定。Raycast でキャラクター足元の地形を検出する。
/// </summary>
private void UpdateGroundCheck()
{
    // 足元から下方向に短い Ray を飛ばす
    var origin = (Vector2)transform.position + Vector2.down * 0.1f;
    var hit = Physics2D.Raycast(origin, Vector2.down, 0.2f, LayerMask.GetMask("Ground"));
    IsGrounded = hit.collider != null;
}

/// <summary>
/// 移動方向に応じて向きを反転する。
/// </summary>
private void UpdateFacingDirection()
{
    float vx = _rb.linearVelocityX;
    if (Mathf.Abs(vx) > 0.01f)
    {
        bool shouldFaceRight = vx > 0;
        if (IsFacingRight != shouldFaceRight)
        {
            IsFacingRight = shouldFaceRight;
            var scale = transform.localScale;
            scale.x = IsFacingRight ? Mathf.Abs(scale.x) : -Mathf.Abs(scale.x);
            transform.localScale = scale;
        }
    }
}
```

**フレーム処理フロー**:

```
[FixedUpdate]
  │
  ├── UpdateGroundCheck()
  │     └── Physics2D.Raycast で接地判定
  │
  ├── ProcessMovementInfo()
  │     ├── JudgeResult フラグ確認
  │     ├── actNum → AbilityType マッピング
  │     ├── ExecuteAbility() で対応 Ability 呼び出し
  │     └── JudgeUpdate() でデータ反映
  │
  └── UpdateFacingDirection()
        └── Rigidbody2D の速度から向き更新
```

## 3. HP / MP / スタミナ

### 3.1 データ構造（CharacterBaseInfo, StaminaInfo）

HP と MP は `CharacterBaseInfo` 構造体に格納される。この構造体は BrainStatus 内に既に定義されており、SoACharaDataDic で配列管理されている。

```csharp
/// <summary>
/// キャラクターの基本情報。HP、MP、およびそれらの割合を保持する。
/// SoACharaDataDic の UnsafeList<CharacterBaseInfo> で一括管理。
/// 28 byte（パディング含む）
/// </summary>
[Serializable]
[StructLayout(LayoutKind.Sequential)]
public struct CharacterBaseInfo
{
    /// <summary>最大体力</summary>
    public int maxHp;

    /// <summary>現在の体力</summary>
    public int currentHp;

    /// <summary>最大魔力</summary>
    public int maxMp;

    /// <summary>現在の魔力</summary>
    public int currentMp;

    /// <summary>HPの割合（0〜100）。AI判定で高速比較用。</summary>
    public byte hpRatio;

    /// <summary>MPの割合（0〜100）。</summary>
    public byte mpRatio;

    // ---- ユーティリティ ----

    /// <summary>HP割合を再計算する。HP変動時に呼ぶ。</summary>
    public void RecalculateHpRatio()
    {
        hpRatio = maxHp > 0 ? (byte)(currentHp * 100 / maxHp) : (byte)0;
    }

    /// <summary>MP割合を再計算する。</summary>
    public void RecalculateMpRatio()
    {
        mpRatio = maxMp > 0 ? (byte)(currentMp * 100 / maxMp) : (byte)0;
    }
}
```

スタミナは独立した `StaminaInfo` 構造体で管理する。飛行やダッシュなどの継続消費系アクションで使用される。

```csharp
/// <summary>
/// スタミナ情報。ダッシュ・飛行等の継続消費アクションで使用。
/// HP/MP とは更新頻度が異なるため独立構造体とする。
/// </summary>
[Serializable]
[StructLayout(LayoutKind.Sequential)]
public struct StaminaInfo
{
    /// <summary>最大スタミナ</summary>
    public float maxStamina;

    /// <summary>現在のスタミナ</summary>
    public float currentStamina;

    /// <summary>スタミナ自動回復速度（/秒）</summary>
    public float regenRate;

    /// <summary>回復開始までの待機時間（秒）</summary>
    public float regenDelay;

    /// <summary>最後にスタミナを消費した時刻</summary>
    public float lastConsumeTime;

    /// <summary>スタミナ枯渇中フラグ（0: 通常, 1: 枯渇）</summary>
    public byte isExhausted;

    // ---- 操作メソッド ----

    /// <summary>
    /// スタミナを消費する。枯渇した場合は isExhausted を立てる。
    /// </summary>
    /// <returns>消費に成功したか</returns>
    public bool Consume(float amount, float currentTime)
    {
        if (isExhausted == 1) return false;
        if (currentStamina < amount)
        {
            isExhausted = 1;
            return false;
        }
        currentStamina -= amount;
        lastConsumeTime = currentTime;
        return true;
    }

    /// <summary>
    /// スタミナの自動回復を処理する。regenDelay 経過後に回復開始。
    /// </summary>
    public void Regenerate(float currentTime, float deltaTime)
    {
        if (currentTime - lastConsumeTime < regenDelay) return;

        currentStamina = Mathf.Min(currentStamina + regenRate * deltaTime, maxStamina);

        // 一定割合まで回復したら枯渇状態を解除
        if (isExhausted == 1 && currentStamina >= maxStamina * 0.2f)
        {
            isExhausted = 0;
        }
    }
}
```

### 3.2 リソース消費フロー

HP/MP/スタミナの変更は全て SoACharaDataDic 経由で行う。ローカル変数へのコピーは禁止。

```csharp
// ========== HP消費（ダメージ受け） ==========
public static class ResourceOperations
{
    /// <summary>
    /// HPダメージを適用する。
    /// ref 参照で直接 SoA コンテナの値を書き換える。
    /// </summary>
    public static void ApplyHpDamage(int objectHash, int damage)
    {
        ref var info = ref GameManager.Data.GetBaseInfo(objectHash);
        info.currentHp = Mathf.Max(0, info.currentHp - damage);
        info.RecalculateHpRatio();
    }

    /// <summary>
    /// HP回復を適用する。最大HPを超えない。
    /// </summary>
    public static void ApplyHpHeal(int objectHash, int amount)
    {
        ref var info = ref GameManager.Data.GetBaseInfo(objectHash);
        info.currentHp = Mathf.Min(info.maxHp, info.currentHp + amount);
        info.RecalculateHpRatio();
    }

    /// <summary>
    /// MP消費を試みる。不足時は false を返す。
    /// </summary>
    public static bool TryConsumeMp(int objectHash, int cost)
    {
        ref var info = ref GameManager.Data.GetBaseInfo(objectHash);
        if (info.currentMp < cost) return false;
        info.currentMp -= cost;
        info.RecalculateMpRatio();
        return true;
    }

    /// <summary>
    /// スタミナ消費を試みる。枯渇中は false を返す。
    /// </summary>
    public static bool TryConsumeStamina(int objectHash, float amount)
    {
        ref var stamina = ref GameManager.Data.GetStaminaInfo(objectHash);
        return stamina.Consume(amount, Time.time);
    }
}
```

**リソース消費シーケンス**:

```
DashAbility.Execute()
  │
  ├── ResourceOperations.TryConsumeStamina(ObjectHash, dashCost)
  │     ├── ref stamina = ref GameManager.Data.GetStaminaInfo(hash)
  │     ├── stamina.Consume(amount, Time.time)
  │     └── return true / false
  │
  ├── [成功] → ダッシュ実行（Rigidbody2D に力を加える）
  └── [失敗] → ダッシュ不可（スタミナ枯渇通知）
```

## 4. アクションシステム（Ability）

### 4.1 IAbility インターフェース

全てのアビリティが実装すべきインターフェース。BaseCharacter は `IAbility[]` として管理し、AbilityType で検索・実行する。

```csharp
/// <summary>
/// アビリティの共通インターフェース。
/// BaseCharacter は IAbility[] を保持し、actNum → AbilityType マッピングで
/// 対応するアビリティを実行する。
/// </summary>
public interface IAbility
{
    /// <summary>アビリティの種別</summary>
    AbilityType Type { get; }

    /// <summary>アビリティが属する排他グループ</summary>
    AbilityExclusiveGroup ExclusiveGroup { get; }

    /// <summary>現在実行可能かどうか</summary>
    bool CanExecute();

    /// <summary>現在実行中かどうか</summary>
    bool IsExecuting { get; }

    /// <summary>BaseCharacter からの初期化</summary>
    void Initialize(BaseCharacter owner);

    /// <summary>アビリティを実行する</summary>
    void Execute(MovementInfo info);

    /// <summary>アビリティを中断する</summary>
    void Cancel();

    /// <summary>毎フレームの更新処理（実行中のみ呼ばれる）</summary>
    void Tick(float deltaTime);
}
```

### 4.2 AbilityBase 基底クラス

各アビリティの共通処理を AbilityBase にまとめる。個別アビリティはこれを継承する。

```csharp
/// <summary>
/// アビリティの共通基底クラス。
/// MonoBehaviour かつ IAbility を実装し、GameObject にアタッチ可能。
/// クールダウン管理、排他制御チェック、オーナー参照を共通化する。
/// </summary>
public abstract class AbilityBase : MonoBehaviour, IAbility
{
    // ========== IAbility 実装 ==========

    public abstract AbilityType Type { get; }
    public virtual AbilityExclusiveGroup ExclusiveGroup => AbilityExclusiveGroup.None;
    public bool IsExecuting { get; protected set; }

    // ========== 設定 ==========

    [SerializeField, Header("クールダウン時間（秒）")]
    protected float _cooldownTime = 0f;

    // ========== 内部状態 ==========

    protected BaseCharacter _owner;
    protected int _ownerHash;
    private float _lastExecuteTime = float.MinValue;

    // ========== 初期化 ==========

    public virtual void Initialize(BaseCharacter owner)
    {
        _owner = owner;
        _ownerHash = owner.ObjectHash;
    }

    // ========== 実行判定 ==========

    /// <summary>
    /// 実行可能かどうかを判定する。
    /// クールダウン中でないこと、排他グループの他アビリティが実行中でないこと、
    /// 個別条件を満たすことをチェックする。
    /// </summary>
    public virtual bool CanExecute()
    {
        // クールダウンチェック
        if (Time.time - _lastExecuteTime < _cooldownTime) return false;

        // 排他グループチェック
        if (ExclusiveGroup != AbilityExclusiveGroup.None)
        {
            if (AbilityExclusiveManager.IsGroupActive(_ownerHash, ExclusiveGroup))
                return false;
        }

        // 個別条件チェック（サブクラスでオーバーライド）
        return CanExecuteInternal();
    }

    /// <summary>サブクラス固有の実行条件。デフォルトは true。</summary>
    protected virtual bool CanExecuteInternal() => true;

    // ========== 実行 ==========

    public void Execute(MovementInfo info)
    {
        IsExecuting = true;
        _lastExecuteTime = Time.time;

        // 排他グループに登録
        if (ExclusiveGroup != AbilityExclusiveGroup.None)
        {
            AbilityExclusiveManager.SetGroupActive(_ownerHash, ExclusiveGroup, Type);
        }

        OnExecute(info);
    }

    /// <summary>サブクラスで実装する実行処理</summary>
    protected abstract void OnExecute(MovementInfo info);

    // ========== キャンセル ==========

    public void Cancel()
    {
        if (!IsExecuting) return;
        IsExecuting = false;

        // 排他グループから解除
        if (ExclusiveGroup != AbilityExclusiveGroup.None)
        {
            AbilityExclusiveManager.ClearGroup(_ownerHash, ExclusiveGroup);
        }

        OnCancel();
    }

    /// <summary>サブクラスで実装するキャンセル処理</summary>
    protected virtual void OnCancel() { }

    // ========== 更新 ==========

    public virtual void Tick(float deltaTime) { }

    // ========== ユーティリティ ==========

    /// <summary>実行完了時に呼ぶ。IsExecuting を false にし排他解除する。</summary>
    protected void Complete()
    {
        IsExecuting = false;
        if (ExclusiveGroup != AbilityExclusiveGroup.None)
        {
            AbilityExclusiveManager.ClearGroup(_ownerHash, ExclusiveGroup);
        }
    }
}
```

### 4.3 AbilityType 列挙

```csharp
/// <summary>
/// アビリティの種別。actNum からのマッピングにも使用する。
/// </summary>
public enum AbilityType : byte
{
    None = 0,

    // ---- 基本移動 ----
    Walk = 1,           // 歩行
```

### 5.1 JumpAbility（ジャンプ - 多段ジャンプ対応）

```csharp
/// <summary>
/// ジャンプアビリティ。多段ジャンプに対応。
/// ジャンプ高さ・回数は CharacterInfo の MoveStatus から取得。
/// </summary>
public class JumpAbility : AbilityBase
{
    public override AbilityType Type => AbilityType.Jump;
    public override AbilityExclusiveGroup ExclusiveGroup => AbilityExclusiveGroup.Aerial;

    [SerializeField, Header("最大ジャンプ回数")]
    private int _maxJumpCount = 2;

    [SerializeField, Header("2段目以降のジャンプ力倍率")]
    private float _multiJumpRatio = 0.75f;

    private int _currentJumpCount;
    private Rigidbody2D _rb;

    public override void Initialize(BaseCharacter owner)
    {
        base.Initialize(owner);
        _rb = owner.GetComponent<Rigidbody2D>();
    }

    /// <summary>接地時にジャンプ回数をリセットする。</summary>
    private void FixedUpdate()
    {
        if (_owner.IsGrounded && !IsExecuting)
        {
            _currentJumpCount = 0;
        }
    }

    protected override bool CanExecuteInternal()
    {
        // 地上ジャンプ、または多段ジャンプ残りがある場合
        return _currentJumpCount < _maxJumpCount;
    }

    protected override void OnExecute(MovementInfo info)
    {
        // MoveStatus からジャンプ高さを取得（SoAコンテナ経由）
        ref var moveStatus = ref GameManager.Data.GetMoveStatus(_ownerHash);
        float jumpForce = moveStatus.jumpHeight;

        // 2段目以降は倍率を適用
        if (_currentJumpCount > 0)
        {
            jumpForce *= _multiJumpRatio;
        }

        // 垂直速度をリセットしてからジャンプ力を付与
        _rb.linearVelocityY = 0f;
        _rb.AddForce(Vector2.up * jumpForce, ForceMode2D.Impulse);

        _currentJumpCount++;

        // ジャンプは瞬間的なアクションなので即完了
        Complete();
    }
}
```

**多段ジャンプの状態遷移**:

```
[接地] jumpCount = 0
  │
  ├── Execute() → 1段目ジャンプ (jumpForce = jumpHeight)
  │   jumpCount = 1
  │
  ├── Execute() → 2段目ジャンプ (jumpForce = jumpHeight * 0.75)
  │   jumpCount = 2
  │
  ├── CanExecute() == false （回数上限）
  │
  └── [着地] → jumpCount = 0 にリセット
```

### 5.2 DashAbility（ダッシュ - 無敵フレーム対応）

```csharp
/// <summary>
/// ダッシュアビリティ。スタミナ消費 + 無敵フレーム対応。
/// ダッシュ速度は MoveStatus.dashSpeed から取得。
/// </summary>
public class DashAbility : AbilityBase
{
    public override AbilityType Type => AbilityType.Dash;
    public override AbilityExclusiveGroup ExclusiveGroup => AbilityExclusiveGroup.Movement;

    [SerializeField, Header("ダッシュ持続時間（秒）")]
    private float _dashDuration = 0.2f;

    [SerializeField, Header("スタミナ消費量")]
    private float _staminaCost = 15f;

    [SerializeField, Header("無敵フレーム開始（秒）")]
    private float _invincibleStart = 0.05f;

    [SerializeField, Header("無敵フレーム終了（秒）")]
    private float _invincibleEnd = 0.15f;

    private Rigidbody2D _rb;
    private DamageReceiver _damageReceiver;
    private float _elapsedTime;
    private float _dashDirection;

    public override void Initialize(BaseCharacter owner)
    {
        base.Initialize(owner);
        _rb = owner.GetComponent<Rigidbody2D>();
        _damageReceiver = owner.GetComponent<DamageReceiver>();
    }

    protected override bool CanExecuteInternal()
    {
        // スタミナ枯渇チェック
        ref var stamina = ref GameManager.Data.GetStaminaInfo(_ownerHash);
        return stamina.isExhausted == 0 && stamina.currentStamina >= _staminaCost;
    }

    protected override void OnExecute(MovementInfo info)
    {
        // スタミナ消費
        if (!ResourceOperations.TryConsumeStamina(_ownerHash, _staminaCost))
        {
            Complete();
            return;
        }

        // ダッシュ方向の決定（向いている方向）
        _dashDirection = _owner.IsFacingRight ? 1f : -1f;
        _elapsedTime = 0f;

        // MoveStatus からダッシュ速度を取得
        ref var moveStatus = ref GameManager.Data.GetMoveStatus(_ownerHash);
        _rb.linearVelocityX = _dashDirection * moveStatus.dashSpeed;

        // 重力を一時的に無効化
        _rb.gravityScale = 0f;
        _rb.linearVelocityY = 0f;
    }

    public override void Tick(float deltaTime)
    {
        if (!IsExecuting) return;

        _elapsedTime += deltaTime;

        // 無敵フレーム管理
        if (_elapsedTime >= _invincibleStart && _elapsedTime <= _invincibleEnd)
        {
            _damageReceiver?.SetInvincible(true);
        }
        else
        {
            _damageReceiver?.SetInvincible(false);
        }

        // ダッシュ終了判定
        if (_elapsedTime >= _dashDuration)
        {
            EndDash();
        }
    }

    protected override void OnCancel()
    {
        EndDash();
    }

    private void EndDash()
    {
        _rb.gravityScale = 1f;
        _damageReceiver?.SetInvincible(false);
        Complete();
    }
}
```

**ダッシュのタイムライン**:

```
t=0.00  Execute()     → スタミナ消費、ダッシュ開始
t=0.05  invStart      → 無敵フレーム開始
t=0.15  invEnd        → 無敵フレーム終了
t=0.20  dashDuration  → ダッシュ終了、重力復帰
```

### 5.3 FlyAbility（飛行 - スタミナ継続消費）

```csharp
/// <summary>
/// 飛行アビリティ。スタミナを継続消費しながら自由飛行する。
/// SolidData.canFly が true のキャラクターのみ使用可能。
/// スタミナ枯渇で強制終了し、落下する。
/// </summary>
public class FlyAbility : AbilityBase
{
    public override AbilityType Type => AbilityType.Fly;
    public override AbilityExclusiveGroup ExclusiveGroup => AbilityExclusiveGroup.Aerial;

    [SerializeField, Header("飛行速度")]
    private float _flySpeed = 5f;

    [SerializeField, Header("スタミナ消費速度（/秒）")]
    private float _staminaCostPerSecond = 10f;

    [SerializeField, Header("上昇時の追加スタミナ倍率")]
    private float _ascendCostMultiplier = 1.5f;

    private Rigidbody2D _rb;

    public override void Initialize(BaseCharacter owner)
    {
        base.Initialize(owner);
        _rb = owner.GetComponent<Rigidbody2D>();
    }

    protected override bool CanExecuteInternal()
    {
        // 飛行可能なキャラクターかチェック（SolidData.canFly）
        ref var solid = ref GameManager.Data.GetSolidData(_ownerHash);
        if (solid.canFly == 0) return false;

        // スタミナ枯渇チェック
        ref var stamina = ref GameManager.Data.GetStaminaInfo(_ownerHash);
        return stamina.isExhausted == 0;
    }

    protected override void OnExecute(MovementInfo info)
    {
        // 重力を無効化して飛行モードに入る
        _rb.gravityScale = 0f;
    }

    public override void Tick(float deltaTime)
    {
        if (!IsExecuting) return;

        // 飛行方向の計算（AI判定またはプレイヤー入力から）
        var moveDir = GetFlyDirection();

        // スタミナ消費（上昇時は追加消費）
        float cost = _staminaCostPerSecond * deltaTime;
        if (moveDir.y > 0) cost *= _ascendCostMultiplier;

        if (!ResourceOperations.TryConsumeStamina(_ownerHash, cost))
        {
            // スタミナ枯渇 → 飛行終了、落下開始
            ForceLand();
            return;
        }

        // 飛行移動
        _rb.linearVelocity = moveDir.normalized * _flySpeed;
    }

    /// <summary>飛行方向を取得する。</summary>
    private Vector2 GetFlyDirection()
    {
        var info = _owner.LastJudgeResult;
        // targetHash からターゲット位置を取得し、方向ベクトルを計算
        // （簡略化: AIの場合はターゲット方向、プレイヤーは入力方向）
        return new Vector2(info.targetDistance > 0 ? 1f : -1f, 0f);
    }

    protected override void OnCancel()
    {
        RestoreGravity();
    }

    /// <summary>スタミナ枯渇による強制着陸。</summary>
    private void ForceLand()
    {
        RestoreGravity();
        Complete();
    }

    private void RestoreGravity()
    {
        _rb.gravityScale = 1f;
    }
}
```

**飛行状態マシン**:

```
[地上/空中] ── CanExecute() ──→ [飛行中]
                                    │
                        ┌───────────┼───────────┐
                        │           │           │
                   Tick()で     スタミナ     Cancel()
                   継続飛行     枯渇         （被弾等）
                        │           │           │
                        │       ForceLand()     │
                        │           │           │
                        └───────────┼───────────┘
                                    │
                                    ▼
                            [重力復帰 → 落下]
```

### 5.4 WarpAbility（ワープ - MP消費瞬間移動）

```csharp
/// <summary>
/// ワープアビリティ。MP を消費して瞬間移動する。
/// ターゲット位置またはワープポイントへ即座に移動。
/// 地形判定を行い、壁の中にワープしないことを保証する。
/// </summary>
public class WarpAbility : AbilityBase
{
    public override AbilityType Type => AbilityType.Warp;
    public override AbilityExclusiveGroup ExclusiveGroup => AbilityExclusiveGroup.Movement;

    [SerializeField, Header("MP消費量")]
    private int _mpCost = 30;

    [SerializeField, Header("最大ワープ距離")]
    private float _maxWarpDistance = 10f;

    [SerializeField, Header("ワープ前の予備動作時間（秒）")]
    private float _castTime = 0.3f;

    [SerializeField, Header("ワープ後の硬直時間（秒）")]
    private float _recoveryTime = 0.2f;

    [SerializeField, Header("ワープ先安全チェック用レイヤー")]
    private LayerMask _obstacleLayer;

    private float _elapsedTime;
    private Vector2 _warpTarget;
    private WarpPhase _phase;

    private enum WarpPhase { Cast, Warp, Recovery }

    protected override bool CanExecuteInternal()
    {
        // MP チェック
        ref var baseInfo = ref GameManager.Data.GetBaseInfo(_ownerHash);
        return baseInfo.currentMp >= _mpCost;
    }

    protected override void OnExecute(MovementInfo info)
    {
        _elapsedTime = 0f;
        _phase = WarpPhase.Cast;

        // ワープ先を計算
        _warpTarget = CalculateWarpTarget(info);
    }

    public override void Tick(float deltaTime)
    {
        if (!IsExecuting) return;
        _elapsedTime += deltaTime;

        switch (_phase)
        {
            case WarpPhase.Cast:
                // 予備動作中（エフェクト再生等）
                if (_elapsedTime >= _castTime)
                {
                    PerformWarp();
                    _phase = WarpPhase.Recovery;
                    _elapsedTime = 0f;
                }
                break;

            case WarpPhase.Recovery:
                // 硬直中
                if (_elapsedTime >= _recoveryTime)
                {
                    Complete();
                }
                break;
        }
    }

    /// <summary>ワープ先を計算する。地形に埋まらない安全な位置を返す。</summary>
    private Vector2 CalculateWarpTarget(MovementInfo info)
    {
        // ターゲット方向に最大距離ワープ
        float direction = info.targetDistance > 0 ? 1f : -1f;
        Vector2 origin = _owner.transform.position;
        Vector2 desired = origin + Vector2.right * direction * _maxWarpDistance;

        // 地形チェック: 障害物がある場合はその手前に補正
        var hit = Physics2D.Raycast(origin, (desired - origin).normalized,
            _maxWarpDistance, _obstacleLayer);
        if (hit.collider != null)
        {
            desired = hit.point - (desired - origin).normalized * 0.5f;
        }

        return desired;
    }

    /// <summary>実際のワープ処理。MP消費と位置変更を行う。</summary>
    private void PerformWarp()
    {
        // MP消費
        if (!ResourceOperations.TryConsumeMp(_ownerHash, _mpCost))
        {
            // MP不足（実行中に消費された場合）→ キャンセル
            Complete();
            return;
        }

        // 瞬間移動
        _owner.transform.position = _warpTarget;

        // 移動速度をリセット
        var rb = _owner.GetComponent<Rigidbody2D>();
        rb.linearVelocity = Vector2.zero;
    }
}
```

### 5.5 LadderClimbAbility（はしご登り）

```csharp
/// <summary>
/// はしご登りアビリティ。はしごオブジェクトとの接触時に
/// 垂直移動を可能にする。重力を無効化し、上下入力で移動。
/// </summary>
public class LadderClimbAbility : AbilityBase
{
    public override AbilityType Type => AbilityType.LadderClimb;
    public override AbilityExclusiveGroup ExclusiveGroup => AbilityExclusiveGroup.Movement;

    [SerializeField, Header("はしご登り速度")]
    private float _climbSpeed = 3f;

    [SerializeField, Header("はしごレイヤー")]
    private LayerMask _ladderLayer;

    private Rigidbody2D _rb;
    private Collider2D _currentLadder;
    private float _originalGravityScale;

    public override void Initialize(BaseCharacter owner)
    {
        base.Initialize(owner);
        _rb = owner.GetComponent<Rigidbody2D>();
        _originalGravityScale = _rb.gravityScale;
    }

    protected override bool CanExecuteInternal()
    {
        // はしごに接触しているかチェック
        return _currentLadder != null;
    }

    protected override void OnExecute(MovementInfo info)
    {
        // 重力を無効化
        _rb.gravityScale = 0f;
        _rb.linearVelocity = Vector2.zero;

        // はしごの中心X座標にスナップ
        var pos = _owner.transform.position;
        pos.x = _currentLadder.bounds.center.x;
        _owner.transform.position = pos;
    }

    public override void Tick(float deltaTime)
    {
        if (!IsExecuting) return;

        // はしごから離れたら終了
        if (_currentLadder == null)
        {
            ExitLadder();
            return;
        }

        // 上下移動（AI の actNum または入力から方向を取得）
        float verticalInput = GetVerticalInput();
        _rb.linearVelocityY = verticalInput * _climbSpeed;
        _rb.linearVelocityX = 0f;

        // はしごの上端・下端を超えたら離脱
        float yPos = _owner.transform.position.y;
        var bounds = _currentLadder.bounds;
        if (yPos > bounds.max.y + 0.1f || yPos < bounds.min.y - 0.1f)
        {
            ExitLadder();
        }
    }

    /// <summary>上下入力を取得する。</summary>
    private float GetVerticalInput()
    {
        // MovementInfo から判定（actNum が上移動か下移動か）
        var info = _owner.LastJudgeResult;
        // 簡略化: targetDistance の符号で上下を判断
        return info.targetDistance > 0 ? 1f : -1f;
    }

    /// <summary>はしごから離脱する。</summary>
    private void ExitLadder()
    {
        _rb.gravityScale = _originalGravityScale;
        Complete();
    }

    // ---- トリガー検知 ----

    private void OnTriggerEnter2D(Collider2D other)
    {
        if (((1 << other.gameObject.layer) & _ladderLayer) != 0)
        {
            _currentLadder = other;
        }
    }

    private void OnTriggerExit2D(Collider2D other)
    {
        if (other == _currentLadder)
        {
            _currentLadder = null;
        }
    }
}
```

## 6. アビリティの組み合わせ管理

### 6.1 排他制御（AbilityExclusiveGroup）

同時に実行できないアビリティをグループで管理する。同じ排他グループに属するアビリティは、1つだけが実行可能。

```csharp
/// <summary>
/// アビリティの排他グループ。
/// 同一グループのアビリティは同時に実行できない。
/// </summary>
[Flags]
public enum AbilityExclusiveGroup : byte
{
    None = 0,

    /// <summary>移動系（ダッシュ、ワープ、はしご登りは排他）</summary>
    Movement = 1 << 0,

    /// <summary>空中系（ジャンプ、飛行は排他）</summary>
    Aerial = 1 << 1,

    /// <summary>戦闘系（攻撃、スキル、ガードは排他）</summary>
    Combat = 1 << 2,
}

/// <summary>
/// 排他グループの状態を管理する静的クラス。
/// キャラごとにどのグループがアクティブかを追跡する。
/// </summary>
public static class AbilityExclusiveManager
{
    // キャラハッシュ → アクティブなグループフラグ
    private static readonly Dictionary<int, AbilityExclusiveGroup> _activeGroups = new();

    /// <summary>指定グループがアクティブか確認する。</summary>
    public static bool IsGroupActive(int ownerHash, AbilityExclusiveGroup group)
    {
        return _activeGroups.TryGetValue(ownerHash, out var active)
               && (active & group) != 0;
    }

    /// <summary>グループをアクティブにする。</summary>
    public static void SetGroupActive(int ownerHash, AbilityExclusiveGroup group, AbilityType type)
    {
        if (!_activeGroups.ContainsKey(ownerHash))
            _activeGroups[ownerHash] = AbilityExclusiveGroup.None;

        _activeGroups[ownerHash] |= group;
    }

    /// <summary>グループを解除する。</summary>
    public static void ClearGroup(int ownerHash, AbilityExclusiveGroup group)
    {
        if (_activeGroups.ContainsKey(ownerHash))
        {
            _activeGroups[ownerHash] &= ~group;
        }
    }

    /// <summary>キャラ削除時に全グループを解除する。</summary>
    public static void RemoveCharacter(int ownerHash)
    {
        _activeGroups.Remove(ownerHash);
    }
}
```

**排他制御の例**:

```
[ダッシュ中] ExclusiveGroup = Movement
  │
  ├── WarpAbility.CanExecute() → false（Movement グループがアクティブ）
  ├── LadderClimbAbility.CanExecute() → false（Movement グループがアクティブ）
  ├── JumpAbility.CanExecute() → true（Aerial グループは未使用）
  └── DashAbility.Complete() → Movement グループ解除
```

### 6.3 AI からのアビリティ制御（actNum → AbilityType mapping）

AI判定が出力する `MovementInfo.actNum` は行動番号であり、これを `AbilityType` にマッピングして対応するアビリティを実行する。マッピングは `CharacterInfo`（ScriptableObject）で定義し、キャラクター種別ごとに自由に設定できる。

```csharp
/// <summary>
/// actNum と AbilityType のマッピング情報。
/// CharacterInfo 内に配列として保持される。
/// </summary>
[Serializable]
public struct ActAbilityMapping
{
    /// <summary>AI判定が出力する行動番号</summary>
    public byte actNum;

    /// <summary>対応する AbilityType</summary>
    public AbilityType abilityType;

    /// <summary>追加パラメータ（方向、強度等）</summary>
    public float parameter;
}

// ========== BaseCharacter でのマッピング処理 ==========

/// <summary>
/// actNum を AbilityType に変換する。
/// マッピングテーブルは CharacterInfo に定義されている。
/// </summary>
private AbilityType MapActNumToAbility(byte actNum)
{
    // CharacterInfo のマッピングテーブルを検索
    var mappings = _characterInfo.ActAbilityMappings;
    for (int i = 0; i < mappings.Length; i++)
    {
        if (mappings[i].actNum == actNum)
        {
            return mappings[i].abilityType;
        }
    }
    return AbilityType.Idle; // マッピングなし → 待機
}
```

**AI → Ability 実行フロー**:

```
AIBrain
  │ 出力: MovementInfo { actNum = 3, targetHash = 12345, ... }
  │
  ▼
BaseCharacter.ProcessMovementInfo()
  │
  ├── MapActNumToAbility(3) → AbilityType.Dash
  │     （CharacterInfo のマッピング: actNum=3 は Dash）
  │
  ├── ExecuteAbility(AbilityType.Dash, info)
  │     ├── _abilities から DashAbility を検索
  │     ├── DashAbility.CanExecute() → true
  │     └── DashAbility.Execute(info)
  │
  └── info.JudgeUpdate(ObjectHash) → データ反映
```

## 7. 地形判定との連携

### 7.1 接地判定（GroundCheck）

```csharp
/// <summary>
/// 接地判定の詳細情報。BaseCharacter が毎フレーム更新する。
```

| 地形状態 | Jump | Dash | Fly | Warp | LadderClimb |

### 7.3 地形判定コード

```csharp
/// <summary>
/// BaseCharacter の接地判定メソッド（詳細版）。
/// 複数の Ray を使って地形情報を取得する。
/// </summary>
private void UpdateGroundCheck()
{
    var origin = (Vector2)transform.position;
    var size = _colliderSize; // BoxCollider2D のサイズ

    // 足元に複数レイを飛ばして接地判定
    float rayLength = 0.15f;
    float halfWidth = size.x * 0.4f;

    // 中央・左・右の3本レイ
    bool hitCenter = Physics2D.Raycast(origin, Vector2.down, rayLength, _groundLayer);
    bool hitLeft = Physics2D.Raycast(origin + Vector2.left * halfWidth, Vector2.down, rayLength, _groundLayer);
    bool hitRight = Physics2D.Raycast(origin + Vector2.right * halfWidth, Vector2.down, rayLength, _groundLayer);

    var prevGrounded = IsGrounded;
    IsGrounded = hitCenter || hitLeft || hitRight;

    // 地面の法線を取得（坂道判定用）
    if (IsGrounded)
    {
        var hit = Physics2D.Raycast(origin, Vector2.down, rayLength, _groundLayer);
        _groundInfo.groundNormal = hit.normal;
        _groundInfo.groundAngle = Vector2.Angle(hit.normal, Vector2.up);
        _groundInfo.groundLayer = hit.collider.gameObject.layer;
        _groundInfo.isOnPassThroughPlatform =
            hit.collider.gameObject.layer == LayerMask.NameToLayer("PassThrough");
    }

    // 着地イベント
    if (!prevGrounded && IsGrounded)
    {
        OnLanded();
    }
}

/// <summary>着地時のコールバック。各 Ability に通知する。</summary>
private void OnLanded()
{
    for (int i = 0; i < _abilities.Length; i++)
    {
        if (_abilities[i].IsExecuting)
        {
            // 飛行中に着地した場合はキャンセル
            if (_abilities[i].Type == AbilityType.Fly)
            {
                _abilities[i].Cancel();
            }
        }
    }
}
```

## 8. アニメーション状態公開API

AI判定のアクションマスキングとフレームアドバンテージ計算のため、BaseCharacter はアニメーション状態を SoA コンテナに毎フレーム書き戻す。

> **関連**: `AnimationStateData` 構造体は `06_情報クラス群 7.2節` で定義。`AnimationPhase` enum も同節を参照。

### 8.1 アニメーション状態のトラッキング

```csharp
/// <summary>
/// BaseCharacter に追加するアニメーション状態のトラッキング処理。
/// MotionInfo（攻撃のタイムライン）の preMotion / active / recovery と連動し、
/// 現在のアニメーションフェーズを SoACharaDataDic に書き戻す。
/// </summary>
public partial class BaseCharacter : MonoBehaviour
{
    // ========== アニメーション状態 ==========

    /// <summary>現在のアニメーションフェーズ</summary>
    private AnimationPhase _currentPhase = AnimationPhase.Neutral;

    /// <summary>現在フェーズの残りフレーム数</summary>
    private int _remainingFrames;

    /// <summary>現在フェーズ内の経過時間</summary>
    private float _phaseElapsed;

    /// <summary>現在フェーズの総時間</summary>
    private float _phaseDuration;

    /// <summary>現在実行中の技ID（攻撃中のみ、0 = なし）</summary>
    private byte _currentMoveId;

    /// <summary>現在の攻撃がキャンセル可能か</summary>
    private bool _isCancelable;

    /// <summary>
    /// 攻撃開始時に呼ばれる。MotionInfo の各フェーズ時間を設定する。
    /// WeaponHolder.Attack() / IAbility.Execute() から呼び出される。
    /// </summary>
    public void BeginAttackMotion(MotionInfo motion, byte moveId, bool cancelable)
    {
        _currentPhase = AnimationPhase.Anticipation;
        _phaseDuration = motion.preMotionDuration;
        _phaseElapsed = 0f;
        _remainingFrames = Mathf.CeilToInt(
            (motion.preMotionDuration + motion.activeMotionDuration + motion.recoveryDuration) * 60f);
        _currentMoveId = moveId;
        _isCancelable = cancelable;
    }

    /// <summary>
    /// FixedUpdate 内で毎フレーム呼ばれる。
    /// フェーズ遷移と SoACharaDataDic への書き戻しを行う。
    /// </summary>
    private void UpdateAnimationState()
    {
        if (_currentPhase == AnimationPhase.Neutral) return;

        float dt = Time.fixedDeltaTime;
        _phaseElapsed += dt;
        _remainingFrames = Mathf.Max(0, _remainingFrames - 1);

        // フェーズ遷移
        if (_phaseElapsed >= _phaseDuration)
        {
            _phaseElapsed -= _phaseDuration;
            _currentPhase = _currentPhase switch
            {
                AnimationPhase.Anticipation => AnimationPhase.Active,
                AnimationPhase.Active       => AnimationPhase.Recovery,
                AnimationPhase.Recovery     => AnimationPhase.Neutral,
                _ => AnimationPhase.Neutral,
            };

            // 次フェーズの duration を設定（省略: MotionInfo から取得）
            _isCancelable = false;  // Active 以降はキャンセル不可
            if (_currentPhase == AnimationPhase.Neutral)
            {
                _currentMoveId = 0;
                _remainingFrames = 0;
            }
        }

        // SoACharaDataDic に書き戻し
        ref var animData = ref GameManager.Data.GetAnimationState(ObjectHash);
        animData.currentPhase = _currentPhase;
        animData.normalizedTime = (half)(_phaseElapsed / Mathf.Max(_phaseDuration, 0.001f));
        animData.framesUntilActionable = (short)_remainingFrames;
        animData.currentMoveId = _currentMoveId;
        animData.isCancelable = _isCancelable;
        animData.isCommitted = (_currentPhase == AnimationPhase.Active);
    }
}
```

### 8.2 フェイントキャンセルの処理

AI が `ActType.FeintCancel` を選択した場合、BaseCharacter は Anticipation フェーズを即座に中断して Neutral に戻す。

```csharp
/// <summary>
/// フェイントキャンセル。Anticipation 中のみ有効。
/// 予備動作をキャンセルして Neutral に即座復帰する。
/// </summary>
public bool TryFeintCancel()
{
    if (_currentPhase != AnimationPhase.Anticipation) return false;
    if (!_isCancelable) return false;

    _currentPhase = AnimationPhase.Neutral;
    _remainingFrames = 0;
    _currentMoveId = 0;
    _phaseElapsed = 0f;

    // SoA 書き戻し
    ref var animData = ref GameManager.Data.GetAnimationState(ObjectHash);
    animData.currentPhase = AnimationPhase.Neutral;
    animData.framesUntilActionable = 0;
    animData.currentMoveId = 0;
    animData.isCancelable = false;
    animData.isCommitted = false;

    return true;
}
```

### 8.3 FixedUpdate への統合

```csharp
private void FixedUpdate()
{
    if (!IsAlive) return;

    UpdateGroundCheck();
    UpdateAnimationState();   // ← 追加: アニメーション状態トラッキング
    ProcessMovementInfo();
    UpdateFacingDirection();
}
```
