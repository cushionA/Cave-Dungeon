# 03 - 武器・攻撃・戦闘システム

## 1. 概要

コーギーエンジンの武器システム（Weapon / MeleeWeapon / ProjectileWeapon）を完全に置き換える戦闘システムである。

**WeaponHolder** が装備中の武器と攻撃入力を一元管理し、各武器は **IWeaponBehaviour** インターフェースを実装する。攻撃の詳細データはすべて **AttackInfo**（ScriptableObject）に定義し、コード変更なしでバランス調整が可能。

コンボ入力・チャージ攻撃・ジャンプ攻撃を含む多彩な攻撃手段を提供しつつ、AI が攻撃情報を参照して回避・距離管理を判断できる構造とする。

すべてのキャラクターデータは `GameManager.Data`（SoACharaDataDic）に格納され、各コンポーネントは `objectHash` を用いてアクセスする。

## 2. WeaponHolder

WeaponHolder は BaseCharacter にアタッチされる MonoBehaviour で、武器の装備・切り替え・攻撃入力の受付を担当する。

```csharp
/// <summary>
/// 武器の装備・切り替え・攻撃入力を管理するコンポーネント。
/// 攻撃力等のステータスは SoACharaDataDic から objectHash 経由で取得する。
/// </summary>
public class WeaponHolder : MonoBehaviour
{
    [SerializeField] private int _maxEquipSlots = 2;

    private int _ownerHash;                      // owner の objectHash（データアクセス用）
    private IWeaponBehaviour _currentWeapon;      // 現在選択中の武器
    private IWeaponBehaviour[] _equippedWeapons;  // 装備中の武器スロット
    private int _currentSlotIndex;                // 現在の武器スロットインデックス
    private ComboSystem _comboSystem;             // コンボ入力管理

    // --- 攻撃力はコンテナから取得 ---
    // GameManager.Data.GetAtkStatus(_ownerHash)
    // GameManager.Data.GetBaseInfo(_ownerHash)

    /// <summary>
    /// BaseCharacter.Initialize() から呼ばれる初期化処理。
    /// </summary>
    public void Initialize(int ownerHash, WeaponInfo[] initialWeapons)
    {
        _ownerHash = ownerHash;
        _equippedWeapons = new IWeaponBehaviour[_maxEquipSlots];
        _comboSystem = new ComboSystem();

        for (int i = 0; i < initialWeapons.Length && i < _maxEquipSlots; i++)
        {
            _equippedWeapons[i] = WeaponFactory.Create(initialWeapons[i], this);
        }

        if (_equippedWeapons.Length > 0)
        {
            _currentSlotIndex = 0;
            _currentWeapon = _equippedWeapons[0];
        }
    }

    /// <summary>
    /// 攻撃リクエスト。プレイヤー入力またはAI判定から呼ばれる。
    /// </summary>
    public void RequestAttack(AttackType type)
    {
        if (_currentWeapon == null) return;
        if (!CanAttack()) return;

        // コンボシステムに入力を評価させる
        int attackInfoIndex = _comboSystem.EvaluateInput(type, _currentWeapon.GetComboRoutes());
        if (attackInfoIndex >= 0)
        {
            _currentWeapon.ExecuteAttack(attackInfoIndex);
        }
    }

    /// <summary>
    /// 攻撃可能かの判定。スタミナ・状態異常等を確認する。
    /// </summary>
    private bool CanAttack()
    {
        ref var stateInfo = ref GameManager.Data.GetStateInfo(_ownerHash);
        if (stateInfo.HasFlag(CharacterStateFlag.Stun)) return false;
        if (stateInfo.HasFlag(CharacterStateFlag.Dead)) return false;

        ref var staminaInfo = ref GameManager.Data.GetStaminaInfo(_ownerHash);
        if (staminaInfo.currentStamina <= 0f) return false;

        return true;
    }

    public int OwnerHash => _ownerHash;
    public IWeaponBehaviour CurrentWeapon => _currentWeapon;
}
```

### 2.1 武器切り替え

武器切り替えは入力またはAI指示により実行される。切り替え中は攻撃不可となり、コンボはリセットされる。

```csharp
public class WeaponHolder : MonoBehaviour
{
    // --- 武器切り替え ---

    [SerializeField] private float _switchCooldown = 0.3f;
    private float _lastSwitchTime;

    /// <summary>
    /// 次の武器スロットに切り替える。
    /// </summary>
    public void SwitchToNextWeapon()
    {
        if (Time.time - _lastSwitchTime < _switchCooldown) return;
        if (_currentWeapon != null && _currentWeapon.IsAttacking) return;

        int nextIndex = (_currentSlotIndex + 1) % _maxEquipSlots;
        SwitchToSlot(nextIndex);
    }

    /// <summary>
    /// 指定スロットの武器に切り替える。
    /// </summary>
    public void SwitchToSlot(int slotIndex)
    {
        if (slotIndex < 0 || slotIndex >= _maxEquipSlots) return;
        if (_equippedWeapons[slotIndex] == null) return;
        if (slotIndex == _currentSlotIndex) return;

        // 現在の武器を非アクティブに
        _currentWeapon?.OnUnequip();

        // コンボリセット
        _comboSystem.Reset();

        // 新しい武器をアクティブに
        _currentSlotIndex = slotIndex;
        _currentWeapon = _equippedWeapons[slotIndex];
        _currentWeapon.OnEquip();

        _lastSwitchTime = Time.time;
    }

    /// <summary>
    /// 武器スロットに武器を装備する。
    /// </summary>
    public void EquipWeapon(int slotIndex, WeaponInfo weaponInfo)
    {
        if (slotIndex < 0 || slotIndex >= _maxEquipSlots) return;

        // 既存の武器を破棄
        if (_equippedWeapons[slotIndex] != null)
        {
            _equippedWeapons[slotIndex].OnDispose();
        }

        _equippedWeapons[slotIndex] = WeaponFactory.Create(weaponInfo, this);

        // 現在のスロットなら即座に装備
        if (slotIndex == _currentSlotIndex)
        {
            _currentWeapon = _equippedWeapons[slotIndex];
            _currentWeapon.OnEquip();
        }
    }
}
```

### 2.2 攻撃入力の受付

攻撃入力はプレイヤーの場合 InputSystem から、AIキャラクターの場合 MovementInfo の行動番号から受け取る。

```csharp
public class WeaponHolder : MonoBehaviour
{
    // --- 攻撃入力の受付 ---

    private bool _isCharging;
    private float _chargeStartTime;
    private AttackType _chargingType;

    /// <summary>
    /// 毎フレームの更新処理。BaseCharacter.Update() から呼ばれる。
    /// </summary>
    public void WeaponUpdate()
    {
        // チャージ中の更新
        if (_isCharging)
        {
            float chargeTime = Time.time - _chargeStartTime;
            _currentWeapon?.OnChargeUpdate(chargeTime);
        }

        // コンボタイマー更新
        _comboSystem.Update(Time.deltaTime);
    }

    /// <summary>
    /// チャージ開始（ボタン押下時）。
    /// </summary>
    public void StartCharge(AttackType type)
    {
        if (!CanAttack()) return;
        if (_currentWeapon == null) return;

        _isCharging = true;
        _chargeStartTime = Time.time;
        _chargingType = type;
        _currentWeapon.OnChargeStart(type);
    }

    /// <summary>
    /// チャージ解放（ボタンリリース時）。
    /// </summary>
    public void ReleaseCharge()
    {
        if (!_isCharging) return;

        float chargeTime = Time.time - _chargeStartTime;
        _isCharging = false;
        _currentWeapon?.OnChargeRelease(_chargingType, chargeTime);
    }

    /// <summary>
    /// AI からの攻撃指示を受け取る。
    /// MovementInfo の行動番号から AttackType を決定する。
    /// </summary>
    public void ProcessAIAttackCommand(int actionId)
    {
        AttackType type = ActionIdToAttackType(actionId);
        RequestAttack(type);
    }

    private AttackType ActionIdToAttackType(int actionId)
    {
        // actionId と AttackType のマッピングは AIInfo で定義
        return (AttackType)actionId;
    }
}
```

### 3.3 WeaponBase 基底クラス

```csharp
/// <summary>
/// 武器の基底クラス。共通処理を実装する。
/// </summary>
public abstract class WeaponBase : MonoBehaviour, IWeaponBehaviour
{
    [SerializeField] protected WeaponInfo _weaponInfo;
    [SerializeField] protected AttackInfo[] _attackInfos;
    [SerializeField] protected ComboRoute[] _comboRoutes;

    protected WeaponHolder _holder;
    protected bool _isAttacking;
    protected int _currentAttackIndex;
    protected Coroutine _attackCoroutine;

    public string WeaponName => _weaponInfo.weaponName;
    public bool IsAttacking => _isAttacking;

    public ComboRoute[] GetComboRoutes() => _comboRoutes;
    public AttackInfo[] GetAttackInfos() => _attackInfos;

    /// <summary>
    /// WeaponHolder からの初期化。
    /// </summary>
    public virtual void Setup(WeaponHolder holder)
    {
        _holder = holder;
    }

    // --- ライフサイクル ---
    public virtual void OnEquip()
    {
        gameObject.SetActive(true);
    }

    public virtual void OnUnequip()
    {
        CancelAttack();
        gameObject.SetActive(false);
    }

    public virtual void OnDispose()
    {
        CancelAttack();
        Destroy(gameObject);
    }

    // --- 攻撃実行 ---
    public virtual void ExecuteAttack(int attackInfoIndex)
    {
        if (attackInfoIndex < 0 || attackInfoIndex >= _attackInfos.Length) return;

        CancelAttack();
        _currentAttackIndex = attackInfoIndex;
        _attackCoroutine = StartCoroutine(AttackSequence(_attackInfos[attackInfoIndex]));
    }

    /// <summary>
    /// 攻撃シーケンスの共通フロー。
    /// サブクラスで個別のフェーズをオーバーライド可能。
    /// </summary>
    protected virtual IEnumerator AttackSequence(AttackInfo attackInfo)
    {
        _isAttacking = true;

        // 1. 発動前モーション
        yield return PlayPreMotion(attackInfo.motionInfo);

        // 2. 溜め時間（チャージ攻撃の場合）
        if (attackInfo.motionInfo.chargeTime > 0f)
        {
            yield return new WaitForSeconds(attackInfo.motionInfo.chargeTime);
        }

        // 3. ヒットボックス生成 + ダメージ判定
        CreateHitbox(attackInfo);

        // 4. 発動時モーション + エフェクト + サウンド
        yield return PlayActivationMotion(attackInfo);

        // 5. 飛翔体生成（遠距離武器の場合）
        if (attackInfo.projectileInfo.isProjectile)
        {
            SpawnProjectile(attackInfo);
        }

        // 6. 攻撃終了
        _isAttacking = false;
    }

    /// <summary>
    /// 発動前モーション再生。
    /// </summary>
    protected virtual IEnumerator PlayPreMotion(MotionInfo motionInfo)
    {
        // アニメーター経由でモーション再生
        // motionInfo.preMotionClip, motionInfo.preMotionDuration
        yield return new WaitForSeconds(motionInfo.preMotionDuration);
    }

    /// <summary>
    /// 発動時モーション・エフェクト・サウンド再生。
    /// </summary>
    protected virtual IEnumerator PlayActivationMotion(AttackInfo attackInfo)
    {
        // モーション再生
        yield return new WaitForSeconds(attackInfo.motionInfo.activationDuration);
    }

    /// <summary>
    /// ヒットボックスを生成してダメージ判定を行う。
    /// </summary>
    protected abstract void CreateHitbox(AttackInfo attackInfo);

    /// <summary>
    /// 飛翔体を生成する（遠距離武器用）。
    /// </summary>
    protected virtual void SpawnProjectile(AttackInfo attackInfo)
    {
        // サブクラスで実装
    }

    /// <summary>
    /// 現在の攻撃をキャンセルする。
    /// </summary>
    protected void CancelAttack()
    {
        if (_attackCoroutine != null)
        {
            StopCoroutine(_attackCoroutine);
            _attackCoroutine = null;
        }
        _isAttacking = false;
    }

    // --- チャージ ---
    public virtual void OnChargeStart(AttackType type) { }
    public virtual void OnChargeUpdate(float chargeTime) { }
    public virtual void OnChargeRelease(AttackType type, float chargeTime) { }

    // --- ジャンプ攻撃 ---
    public virtual void ExecuteJumpAttack(AttackType type) { }
}
```

## 4. 攻撃種類

### 4.1 弱攻撃 (LightAttack)

素早い攻撃。威力は低いが隙が少なく、コンボの起点となる。

| 項目 | 値 |
|---|---|
| 入力 | 攻撃ボタン短押し |
| 発動前モーション | 短い（0.05〜0.1秒） |
| 威力倍率 | 1.0x |
| スタミナ消費 | 低 |
| コンボ | 起点・繋ぎとして使用可能 |

```csharp
/// <summary>
/// 弱攻撃の実行例。WeaponBase を継承した武器内での処理。
/// </summary>
public void OnLightAttackInput()
{
    // WeaponHolder 経由で攻撃リクエスト
    _holder.RequestAttack(AttackType.LightAttack);
}
```

### 4.2 強攻撃 (HeavyAttack)

重い一撃。威力が高いが発動が遅く、隙が大きい。

| 項目 | 値 |
|---|---|
| 入力 | 攻撃ボタン長押し → 離す、または別ボタン |
| 発動前モーション | 長い（0.2〜0.5秒） |
| 威力倍率 | 1.8〜2.5x |
| スタミナ消費 | 中〜高 |
| コンボ | フィニッシュとして使用可能。吹き飛ばし付き |

```csharp
/// <summary>
/// 強攻撃の実行例。
/// </summary>
public void OnHeavyAttackInput()
{
    _holder.RequestAttack(AttackType.HeavyAttack);
}
```

### 4.3 武器固有スキル (WeaponSkill)

各武器タイプに固有の特殊攻撃。MP を消費する。

| 項目 | 値 |
|---|---|
| 入力 | スキルボタン |
| MP消費 | 武器・スキルごとに異なる |
| クールタイム | あり（AttackInfo で定義） |
| 効果 | 武器種ごとに異なる（範囲攻撃、バフ、特殊移動等） |

```csharp
/// <summary>
/// 武器固有スキルの実行処理。
/// MP 消費とクールタイムの確認を行ったうえで発動する。
/// </summary>
public void ExecuteWeaponSkill(int skillIndex)
{
    AttackInfo skillInfo = _attackInfos[skillIndex];

    // MP消費チェック
    ref var baseInfo = ref GameManager.Data.GetBaseInfo(_holder.OwnerHash);
    if (baseInfo.currentMp < skillInfo.mpCost)
    {
        return; // MP不足
    }

    // クールタイムチェック
    if (_skillCooldownTimers[skillIndex] > 0f)
    {
        return; // クールタイム中
    }

    // MP消費
    baseInfo.currentMp -= skillInfo.mpCost;

    // クールタイム開始
    _skillCooldownTimers[skillIndex] = skillInfo.cooldownTime;

    // 攻撃実行
    ExecuteAttack(skillIndex);
}
```

### 4.4 チャージ攻撃 (ChargeAttack)

ボタンを押し続けることで溜め、離した時の溜め時間に応じて威力が変化する。

| 項目 | 値 |
|---|---|
| 入力 | 攻撃ボタン長押し → 離す |
| 最小チャージ時間 | 0.5秒（これ未満は通常攻撃扱い） |
| 最大チャージ時間 | 2.0〜3.0秒（武器による） |
| 威力倍率 | 1.0x〜3.0x（チャージ時間に比例） |
| スタミナ消費 | チャージ中に持続消費 |

```csharp
/// <summary>
/// チャージ攻撃の実装例。WeaponBase のオーバーライド。
/// </summary>
public class ChargeableWeaponBase : WeaponBase
{
    [SerializeField] private float _minChargeTime = 0.5f;
    [SerializeField] private float _maxChargeTime = 2.5f;
    [SerializeField] private float _chargeStaminaCostPerSec = 5f;

    [SerializeField] private int _chargeAttackInfoIndex;     // チャージ攻撃用 AttackInfo のインデックス
    [SerializeField] private AnimationCurve _chargeDamageCurve; // 溜め時間→威力倍率カーブ

    private bool _isCharging;
    private float _currentChargeTime;

    public override void OnChargeStart(AttackType type)
    {
        _isCharging = true;
        _currentChargeTime = 0f;
        // チャージ開始エフェクト・SE
    }

    public override void OnChargeUpdate(float chargeTime)
    {
        if (!_isCharging) return;

        _currentChargeTime = chargeTime;

        // チャージ中のスタミナ消費
        ref var staminaInfo = ref GameManager.Data.GetStaminaInfo(_holder.OwnerHash);
        staminaInfo.currentStamina -= _chargeStaminaCostPerSec * Time.deltaTime;

        // スタミナ切れでチャージ強制解除
        if (staminaInfo.currentStamina <= 0f)
        {
            OnChargeRelease(AttackType.ChargeLight, _currentChargeTime);
            return;
        }

        // チャージ段階のビジュアルフィードバック
        float chargeRatio = Mathf.Clamp01(_currentChargeTime / _maxChargeTime);
        UpdateChargeVisual(chargeRatio);
    }

    public override void OnChargeRelease(AttackType type, float chargeTime)
    {
        if (!_isCharging) return;
        _isCharging = false;

        // 最小チャージ時間未満は通常攻撃
        if (chargeTime < _minChargeTime)
        {
            AttackType normalType = (type == AttackType.ChargeLight)
                ? AttackType.LightAttack
                : AttackType.HeavyAttack;
            _holder.RequestAttack(normalType);
            return;
        }

        // チャージ倍率の計算
        float chargeRatio = Mathf.Clamp01(chargeTime / _maxChargeTime);
        float damageMultiplier = _chargeDamageCurve.Evaluate(chargeRatio);

        // チャージ攻撃実行（倍率付き）
        ExecuteChargeAttack(_chargeAttackInfoIndex, damageMultiplier);
    }

    /// <summary>
    /// チャージ倍率を適用した攻撃を実行する。
    /// </summary>
    private void ExecuteChargeAttack(int attackInfoIndex, float damageMultiplier)
    {
        AttackInfo info = _attackInfos[attackInfoIndex];
        // DamageDealer にチャージ倍率を渡す
        StartCoroutine(ChargeAttackSequence(info, damageMultiplier));
    }

    private IEnumerator ChargeAttackSequence(AttackInfo attackInfo, float multiplier)
    {
        _isAttacking = true;

        yield return PlayPreMotion(attackInfo.motionInfo);

        // チャージ攻撃のヒットボックス生成（倍率付き）
        DamageDealer.CreateHitbox(attackInfo, _holder.OwnerHash, multiplier);

        yield return PlayActivationMotion(attackInfo);

        _isAttacking = false;
    }

    /// <summary>
    /// チャージ段階に応じたビジュアル更新。
    /// </summary>
    private void UpdateChargeVisual(float chargeRatio)
    {
        // エフェクトの大きさ・色をチャージ率に応じて変更
    }
}
```

### 4.5 ジャンプ攻撃

空中で発動する攻撃。弱攻撃と強攻撃で挙動が大きく異なる。

#### 4.5.1 ジャンプ中弱攻撃

空中で素早い攻撃を繰り出す。滞空時間に若干の変化を与える。

| 項目 | 値 |
|---|---|
| 入力 | 空中で弱攻撃ボタン |
| 威力倍率 | 0.8〜1.0x |
| 特徴 | 空中で素早い一撃。移動慣性を維持 |
| 回数制限 | 1回/ジャンプ（再ジャンプでリセット） |

#### 4.5.2 ジャンプ中強攻撃（落下攻撃）

急降下して着地時に範囲ダメージを与える強力な攻撃。

| 項目 | 値 |
|---|---|
| 入力 | 空中で強攻撃ボタン |
| 威力倍率 | 2.0〜3.0x（落下距離に応じて変動） |
| 特徴 | 急降下 → 着地時に範囲ダメージ |
| 落下速度 | 通常落下の 2〜3 倍 |
| 範囲 | 着地地点を中心に円形（武器による） |

```csharp
/// <summary>
/// ジャンプ攻撃の実装。WeaponBase を継承した武器内での処理。
/// </summary>
public class JumpAttackHandler
{
    private WeaponBase _weapon;
    private WeaponHolder _holder;
    private bool _hasUsedJumpAttack;      // ジャンプ中の攻撃使用済みフラグ
    private float _diveFallSpeed = 30f;   // 落下攻撃の降下速度
    private float _diveStartHeight;       // 落下攻撃開始時の高さ

    [SerializeField] private int _jumpLightAttackIndex;   // ジャンプ弱攻撃の AttackInfo インデックス
    [SerializeField] private int _jumpHeavyAttackIndex;   // ジャンプ強攻撃の AttackInfo インデックス

    /// <summary>
    /// ジャンプ開始時に呼ばれる。攻撃使用フラグをリセットする。
    /// </summary>
    public void OnJumpStart()
    {
        _hasUsedJumpAttack = false;
    }

    /// <summary>
    /// ジャンプ攻撃の実行。
    /// </summary>
    public void ExecuteJumpAttack(AttackType type)
    {
        if (_hasUsedJumpAttack) return;
        _hasUsedJumpAttack = true;

        switch (type)
        {
            case AttackType.JumpLight:
                ExecuteJumpLightAttack();
                break;
            case AttackType.JumpHeavy:
                ExecuteJumpHeavyAttack();
                break;
        }
    }

    /// <summary>
    /// ジャンプ弱攻撃。空中で素早い攻撃を行う。
    /// </summary>
    private void ExecuteJumpLightAttack()
    {
        AttackInfo info = _weapon.GetAttackInfos()[_jumpLightAttackIndex];
        _weapon.ExecuteAttack(_jumpLightAttackIndex);
    }

    /// <summary>
    /// ジャンプ強攻撃（落下攻撃）。
    /// 急降下して着地時に範囲ダメージを与える。
    /// </summary>
    private void ExecuteJumpHeavyAttack()
    {
        _diveStartHeight = _weapon.transform.position.y;
        _weapon.StartCoroutine(DiveAttackSequence());
    }

    /// <summary>
    /// 落下攻撃シーケンス。
    /// </summary>
    private IEnumerator DiveAttackSequence()
    {
        AttackInfo info = _weapon.GetAttackInfos()[_jumpHeavyAttackIndex];

        // 1. 発動前モーション（空中で構える）
        yield return new WaitForSeconds(info.motionInfo.preMotionDuration);

        // 2. 急降下開始 - Rigidbody2D の速度を下方向に設定
        var rb = _weapon.GetComponentInParent<Rigidbody2D>();
        rb.linearVelocity = new Vector2(0f, -_diveFallSpeed);

        // 3. 着地待ち
        yield return WaitForGrounded(rb);

        // 4. 着地時 - 落下距離に応じたダメージ計算
        float fallDistance = _diveStartHeight - _weapon.transform.position.y;
        float distanceMultiplier = Mathf.Clamp(fallDistance / 10f, 1.0f, 3.0f);

        // 5. 範囲ダメージのヒットボックス生成
        DamageDealer.CreateAreaHitbox(
            info,
            _weapon.transform.position,
            info.hitboxConfig.radius,
            _holder.OwnerHash,
            distanceMultiplier
        );

        // 6. 着地エフェクト・画面揺れ
        EffectManager.Play(info.effectSoundInfo.impactEffect, _weapon.transform.position);
        CameraShaker.Shake(info.hitEffectConfig.screenShakeIntensity);

        // 7. 着地後の硬直
        yield return new WaitForSeconds(info.motionInfo.recoveryDuration);
    }

    /// <summary>
    /// 接地するまで待機する。
    /// </summary>
    private IEnumerator WaitForGrounded(Rigidbody2D rb)
    {
        while (rb.linearVelocity.y < -0.1f)
        {
            yield return null;
        }
    }
}
```

## 5. コンボシステム

### 5.1 コンボルート定義 (ComboRoute)

コンボはデータ駆動で定義する。各武器が複数のコンボルートを持ち、プレイヤーの入力パターンに応じて分岐する。

```csharp
/// <summary>
/// コンボの1ステップを定義する構造体。
/// </summary>
[System.Serializable]
public struct ComboStep
{
    /// <summary>このステップの入力タイプ（弱/強）。</summary>
    public AttackType inputType;

    /// <summary>このステップで実行する AttackInfo のインデックス。</summary>
    public int attackInfoIndex;

    /// <summary>次の入力を受け付ける時間（秒）。この時間内に次の入力がなければコンボ終了。</summary>
    public float inputWindow;

    /// <summary>他の行動（回避等）でキャンセル可能になるまでの時間（秒）。</summary>
    public float cancelWindow;
}

/// <summary>
/// コンボルート全体を定義するクラス。
/// 各武器が複数の ComboRoute を持つ。
/// </summary>
[System.Serializable]
public class ComboRoute
{
    /// <summary>コンボルートの識別名。</summary>
    public string routeName;

    /// <summary>コンボの各ステップ。</summary>
    public ComboStep[] steps;
}
```

### 5.2 コンボ入力判定

```csharp
/// <summary>
/// コンボの入力判定と状態管理を行うクラス。
/// 入力されたタイプに応じて最適なコンボルートを選択する。
/// </summary>
public class ComboSystem
{
    private int _currentStepIndex;               // 現在のコンボステップ
    private int _currentRouteIndex;              // 現在のコンボルートインデックス（-1: 未開始）
    private float _inputTimer;                   // 次入力受付タイマー
    private bool _isWaitingForNextInput;         // 次入力待ちフラグ
    private List<int> _validRouteIndices;        // 現在有効なコンボルートの候補リスト

    public ComboSystem()
    {
        _currentStepIndex = -1;
        _currentRouteIndex = -1;
        _validRouteIndices = new List<int>();
    }

    /// <summary>
    /// 攻撃入力を評価し、対応する AttackInfo のインデックスを返す。
    /// コンボが成立しない場合は -1 を返す。
    /// </summary>
    public int EvaluateInput(AttackType inputType, ComboRoute[] routes)
    {
        int nextStep = _currentStepIndex + 1;

        if (_currentStepIndex < 0)
        {
            // コンボ開始：最初のステップが合致するルートを全て候補にする
            _validRouteIndices.Clear();
            for (int i = 0; i < routes.Length; i++)
            {
                if (routes[i].steps.Length > 0 &&
                    routes[i].steps[0].inputType == inputType)
                {
                    _validRouteIndices.Add(i);
                }
            }
        }
        else if (!_isWaitingForNextInput)
        {
            // 入力受付時間外
            Reset();
            return EvaluateInput(inputType, routes); // 新規コンボとして再評価
        }
        else
        {
            // コンボ継続：次のステップが合致する候補に絞り込む
            for (int i = _validRouteIndices.Count - 1; i >= 0; i--)
            {
                int routeIdx = _validRouteIndices[i];
                ComboRoute route = routes[routeIdx];

                // ステップ数が足りないか、入力タイプが合わない場合は候補から除外
                if (route.steps.Length <= nextStep ||
                    route.steps[nextStep].inputType != inputType)
                {
                    _validRouteIndices.RemoveAt(i);
                }
            }
        }

        // 候補がない場合はコンボ不成立
        if (_validRouteIndices.Count == 0)
        {
            Reset();
            return -1;
        }

        // 最初の候補ルートを採用
        int selectedRoute = _validRouteIndices[0];
        ComboStep step = routes[selectedRoute].steps[nextStep];

        // 状態更新
        _currentStepIndex = nextStep;
        _currentRouteIndex = selectedRoute;

        // 次の入力受付タイマーを設定
        if (step.inputWindow > 0f)
        {
            _inputTimer = step.inputWindow;
            _isWaitingForNextInput = true;
        }
        else
        {
            // コンボの最終ステップ
            _isWaitingForNextInput = false;
        }

        return step.attackInfoIndex;
    }

    /// <summary>
    /// 毎フレームの更新。入力受付タイマーを減算する。
    /// </summary>
    public void Update(float deltaTime)
    {
        if (!_isWaitingForNextInput) return;

        _inputTimer -= deltaTime;
        if (_inputTimer <= 0f)
        {
            Reset();
        }
    }

    /// <summary>
    /// コンボ状態をリセットする。
    /// </summary>
    public void Reset()
    {
        _currentStepIndex = -1;
        _currentRouteIndex = -1;
        _inputTimer = 0f;
        _isWaitingForNextInput = false;
        _validRouteIndices.Clear();
    }

    /// <summary>
    /// 現在のコンボステップ数を取得する。
    /// </summary>
    public int CurrentStep => _currentStepIndex;

    /// <summary>
    /// 現在キャンセル可能かどうか。
    /// </summary>
    public bool CanCancel(ComboRoute[] routes)
    {
        if (_currentRouteIndex < 0 || _currentStepIndex < 0) return true;

        ComboStep step = routes[_currentRouteIndex].steps[_currentStepIndex];
        return _inputTimer <= step.inputWindow - step.cancelWindow;
    }
}
```

### 5.3 コンボのリセット条件

コンボは以下の条件でリセットされる。

| 条件 | 説明 |
|---|---|
| 入力受付時間超過 | `inputWindow` 以内に次の入力がなかった場合 |
| 被弾 | ダメージを受けた場合（スーパーアーマー時を除く） |
| 武器切り替え | 別の武器に切り替えた場合 |
| 回避・ダッシュ | キャンセルウィンドウ内で回避した場合（コンボリセット） |
| スタミナ切れ | スタミナが 0 になった場合 |
| 状態異常 | スタン、凍結等の行動不能状態 |

```csharp

{
    ref var stateInfo = ref GameManager.Data.GetStateInfo(_holder.OwnerHash);
    if (!stateInfo.HasFlag(CharacterStateFlag.SuperArmor))
    {
        _comboSystem.Reset();
        _currentWeapon?.CancelAttack();
    }
}
```

## 8. 武器種別の例

### 8.1 近接武器 (MeleeWeapon)

近距離でヒットボックスを生成する武器タイプ。

```csharp
/// <summary>
/// 近接武器の実装。剣・斧・槍など。
/// 武器の前方にヒットボックスを生成してダメージ判定を行う。
/// </summary>
public class MeleeWeapon : WeaponBase
{
    [Header("近接武器設定")]
    [SerializeField] private Vector2 _hitboxSize = new Vector2(2f, 1f);
    [SerializeField] private Vector2 _hitboxOffset = new Vector2(1f, 0f);
    [SerializeField] private LayerMask _targetLayer;

    /// <summary>
    /// 近接攻撃のヒットボックス生成。
    /// キャラクターの向きに応じてヒットボックスの位置を調整する。
    /// </summary>
    protected override void CreateHitbox(AttackInfo attackInfo)
    {
        // キャラクターの向きに応じたオフセット
        float direction = _holder.transform.localScale.x > 0 ? 1f : -1f;
        Vector2 center = (Vector2)_holder.transform.position
                         + new Vector2(_hitboxOffset.x * direction, _hitboxOffset.y);

        // 範囲内の対象を検出
        Collider2D[] hits = Physics2D.OverlapBoxAll(center, _hitboxSize, 0f, _targetLayer);

        foreach (var hit in hits)
        {
            var receiver = hit.GetComponent<DamageReceiver>();
            if (receiver == null) continue;

            // ダメージデータの構築
            DamageData damage = BuildDamageData(attackInfo);
            receiver.ApplyDamage(damage);
        }
    }

    /// <summary>
    /// AttackInfo とキャラクターの攻撃力から DamageData を構築する。
    /// </summary>
    private DamageData BuildDamageData(AttackInfo attackInfo)
    {
        ref var atkStatus = ref GameManager.Data.GetAtkStatus(_holder.OwnerHash);

        ElementalDamage baseDamage = atkStatus.CalculateOutgoingDamage(attackInfo);

        return new DamageData
        {
            damage = baseDamage,
            knockbackForce = attackInfo.knockbackInfo.knockbackForce,
            knockbackDir = attackInfo.knockbackInfo.knockbackDirection,
            physicalType = attackInfo.physicalType,
            statusEffectInfo = attackInfo.statusEffectInfo,
            attackerHash = _holder.OwnerHash,
        };
    }
}
```

#### 近接武器のバリエーション

| 武器 | 特徴 | コンボ | 固有スキル |
|---|---|---|---|
| 剣 | バランス型。攻撃速度・威力が中間 | 弱×3、弱弱強 | 回転斬り（周囲範囲攻撃） |
| 斧 | 高威力・低速。吹き飛ばし強め | 弱×2、強 | 地割れ（前方直線範囲） |
| 槍 | リーチが長い。ヒットボックスが縦長 | 弱×4（連続突き） | 突進突き（前方移動しながら攻撃） |

### 8.2 遠距離武器 (RangedWeapon)

飛翔体を生成する武器タイプ。

```csharp
/// <summary>
/// 遠距離武器の実装。弓・銃・杖など。
/// 飛翔体（Projectile）を生成して攻撃する。
/// </summary>
public class RangedWeapon : WeaponBase
{
    [Header("遠距離武器設定")]
    [SerializeField] private Transform _muzzlePoint;         // 発射位置
    [SerializeField] private GameObject _projectilePrefab;   // 飛翔体プレハブ
    [SerializeField] private int _poolSize = 10;             // オブジェクトプール数

    private ObjectPool<Projectile> _projectilePool;

    public override void OnEquip()
    {
        base.OnEquip();
        // オブジェクトプールの初期化
        _projectilePool = new ObjectPool<Projectile>(
            _projectilePrefab.GetComponent<Projectile>(), _poolSize);
    }

    /// <summary>
    /// 遠距離攻撃ではヒットボックスを直接生成せず、飛翔体に委譲する。
    /// </summary>
    protected override void CreateHitbox(AttackInfo attackInfo)
    {
        // 遠距離武器はヒットボックスを直接生成しない
        // SpawnProjectile で飛翔体にダメージ判定を持たせる
    }

    /// <summary>
    /// 飛翔体の生成。AttackInfo の ProjectileInfo に基づいて設定する。
    /// </summary>
    protected override void SpawnProjectile(AttackInfo attackInfo)
    {
        Projectile proj = _projectilePool.Get();
        if (proj == null) return;

        // 発射位置と方向
        float direction = _holder.transform.localScale.x > 0 ? 1f : -1f;
        Vector2 spawnPos = _muzzlePoint.position;
        Vector2 shootDir = new Vector2(direction, 0f);

        // ProjectileInfo から設定を適用
        ProjectileInfo projInfo = attackInfo.projectileInfo;
        proj.Initialize(
            spawnPos,
            shootDir,
            projInfo.speed,
            projInfo.lifetime,
            projInfo.isPiercing,
            projInfo.trajectoryType,
            attackInfo,
            _holder.OwnerHash
        );

        proj.gameObject.SetActive(true);
    }

    public override void OnDispose()
    {
        _projectilePool?.Dispose();
        base.OnDispose();
    }
}

/// <summary>
/// 飛翔体。生成後に自律的に飛行し、衝突時にダメージ判定を行う。
/// </summary>
public class Projectile : MonoBehaviour
{
    private Vector2 _direction;
    private float _speed;
    private float _lifetime;
    private float _elapsedTime;
    private bool _isPiercing;
    private TrajectoryType _trajectoryType;
    private AttackInfo _attackInfo;
    private int _ownerHash;

    /// <summary>
    /// 飛翔体の初期化。
    /// </summary>
    public void Initialize(
        Vector2 position, Vector2 direction, float speed, float lifetime,
        bool isPiercing, TrajectoryType trajectoryType,
        AttackInfo attackInfo, int ownerHash)
    {
        transform.position = position;
        _direction = direction.normalized;
        _speed = speed;
        _lifetime = lifetime;
        _elapsedTime = 0f;
        _isPiercing = isPiercing;
        _trajectoryType = trajectoryType;
        _attackInfo = attackInfo;
        _ownerHash = ownerHash;
    }

    private void Update()
    {
        _elapsedTime += Time.deltaTime;
        if (_elapsedTime >= _lifetime)
        {
            ReturnToPool();
            return;
        }

        // 軌道タイプに応じた移動
        Vector2 velocity = CalculateVelocity();
        transform.Translate(velocity * Time.deltaTime);
    }

    /// <summary>
    /// 軌道タイプに応じた速度計算。
    /// </summary>
    private Vector2 CalculateVelocity()
    {
        switch (_trajectoryType)
        {
            case TrajectoryType.Straight:
                return _direction * _speed;

            case TrajectoryType.Arc:
                float gravity = -9.8f;
                float t = _elapsedTime;
                return new Vector2(
                    _direction.x * _speed,
                    _direction.y * _speed + gravity * t);

            case TrajectoryType.Homing:
                // ターゲット追尾（最寄りの敵を追尾）
                // 実装は SensorSystem からターゲット情報を取得
                return _direction * _speed;

            default:
                return _direction * _speed;
        }
    }

    private void OnTriggerEnter2D(Collider2D collision)
    {
        var receiver = collision.GetComponent<DamageReceiver>();
        if (receiver == null) return;

        // ダメージ適用
        ref var atkStatus = ref GameManager.Data.GetAtkStatus(_ownerHash);
        ElementalDamage baseDamage = atkStatus.CalculateOutgoingDamage(_attackInfo);

        DamageData damage = new DamageData
        {
            damage = baseDamage,
            knockbackForce = _attackInfo.knockbackInfo.knockbackForce,
            knockbackDir = _attackInfo.knockbackInfo.knockbackDirection,
            physicalType = _attackInfo.physicalType,
            statusEffectInfo = _attackInfo.statusEffectInfo,
            attackerHash = _ownerHash,
        };

        receiver.ApplyDamage(damage);

        // 貫通でなければ消滅
        if (!_isPiercing)
        {
            ReturnToPool();
        }
    }

    private void ReturnToPool()
    {
        gameObject.SetActive(false);
    }
}
```

#### 遠距離武器のバリエーション

| 武器 | 特徴 | 飛翔体軌道 | 固有スキル |
|---|---|---|---|
| 弓 | 中距離・高速弾。チャージで威力増加 | 直線 / 放物線 | 矢の雨（範囲降り注ぎ） |
| 銃 | 長距離・即着弾。連射可能 | 直線 | 貫通弾（複数敵貫通） |
| 杖 | 魔法弾。追尾あり。MP消費が大きい | 追尾 / 直線 | 魔法陣展開（範囲持続ダメージ） |

## 9. AI認識用データ

AttackInfo に含まれる情報は、AI が戦闘判断を行う際にも参照される。AI判定内で敵の攻撃データを分析し、回避・距離管理・優先度判断を行う。

### 9.1 AI が参照する攻撃情報

| AttackInfo のフィールド | AI の利用方法 |
|---|---|
| `motionInfo.preMotionDuration` | 発動前モーション時間 → **回避タイミングの計算**。この時間内に回避行動を開始すれば攻撃を避けられる |
| `projectileInfo.trajectoryType` | 飛翔体の軌道タイプ → **回避方向の計算**。直線弾は上下回避、追尾弾は距離を取る等 |
| `projectileInfo.speed` | 飛翔体の速度 → **回避可否の判定**。速度が高すぎる場合は回避よりガードを選択 |
| `knockbackInfo.knockbackForce` | 吹き飛ばし有無 → **距離管理の判断**。吹き飛ばし攻撃を持つ敵には近距離を維持しない |
| `hitboxConfig.radius` | 範囲ダメージ半径 → **安全距離の計算**。範囲外に退避する |
| `statusEffectInfo.effectType` | 状態異常有無 → **行動優先度の判断**。スタン付き攻撃は最優先で回避する |
| `staminaCost` | スタミナ消費量 → **攻撃後の隙の推測**。高消費攻撃の後は反撃チャンスと判断 |

### 9.2 AI の回避判断フロー

```
敵の攻撃モーション検知（SensorSystem 経由）
  │
  ▼
敵の現在の武器・攻撃の AttackInfo を取得
  │
  ▼
preMotionDuration から回避猶予時間を計算
  │  ※ 猶予時間 > 自キャラの回避発動時間 → 回避可能
  │  ※ 猶予時間 < 自キャラの回避発動時間 → ガードまたは被弾受け入れ
  │
  ▼
飛翔体ありの場合
  │  ├── trajectoryType = Straight → 上下に回避
  │  ├── trajectoryType = Arc     → 前方に詰めて回避
  │  └── trajectoryType = Homing  → 障害物利用またはガード
  │
  ▼
範囲攻撃の場合
  │  └── areaRadius の外に退避
  │
  ▼
回避行動を MovementInfo に書き込み
```

### 9.3 AI の攻撃選択への影響

```csharp
/// <summary>
/// AI判定内での攻撃選択判断（概念コード）。
/// 敵の AttackInfo を参照して最適な攻撃を選択する。
/// </summary>
// AI判定処理内
{
    // 敵の防御状態を確認
    bool enemyIsGuarding = targetStateInfo.HasFlag(CharacterStateFlag.Guard);
    bool enemyHasSuperArmor = targetStateInfo.HasFlag(CharacterStateFlag.SuperArmor);

    // 吹き飛ばし攻撃で敵のガードを崩す
    if (enemyIsGuarding)
    {
        // 吹き飛ばし付きの強攻撃を選択
        selectedAction = ActionType.HeavyAttack;
    }

    // 敵のスタミナが低い場合はラッシュ
    if (targetStaminaInfo.currentStamina < targetStaminaInfo.maxStamina * 0.2f)
    {
        selectedAction = ActionType.LightAttackCombo;
    }

    // 状態異常付与可能な攻撃を優先
    if (targetStateInfo.HasFlag(CharacterStateFlag.Poisoned))
    {
        // 既に毒状態なら他の状態異常を狙う
        selectedAction = FindAttackWithStatusEffect(StatusEffectType.Freeze);
    }
}
```

## 11. フェイントキャンセルシステム

### 11.1 概要

フェイントキャンセルは、攻撃の **Anticipation（予備動作）フェーズ** 中にモーションを中断し、Neutral 状態に復帰するアクションである。追加アニメーションアセットは不要で、既存アニメーションの途中キャンセルのみで実現する。

| 項目 | 内容 |
|---|---|
| 発動条件 | `AnimationPhase == Anticipation` かつ `isCancelable == true` |
| ActType | `ActType.FeintCancel`（06_情報クラス群.md で定義済み） |
| 復帰先 | `AnimationPhase.Neutral` |
| 追加コスト | スタミナ微消費（`feintStaminaCost`） |
| 硬直 | 短い復帰硬直（`feintRecoveryFrames`、デフォルト4F = 約66ms） |

### 11.2 WeaponHolder へのフェイント統合

```csharp
public class WeaponHolder : MonoBehaviour
{
    // === 既存フィールド省略 ===

    [Header("フェイント設定")]
    [SerializeField] private float _feintStaminaCost = 5f;  // フェイント時のスタミナ消費
    [SerializeField] private int _feintRecoveryFrames = 4;  // 復帰硬直（フレーム数）

    /// <summary>
    /// フェイントキャンセルを試行する。
    /// Anticipation フェーズかつ isCancelable の場合のみ成功。
    /// </summary>
    /// <returns>フェイント成功の場合 true</returns>
    public bool TryFeintCancel()
    {
        // AnimationStateData を取得
        var animState = GameManager.Data.GetAnimationStateData(_ownerHash);

        // Anticipation フェーズかつキャンセル可能な場合のみ
        if (animState.currentPhase != AnimationPhase.Anticipation)
            return false;
        if (!animState.isCancelable)
            return false;

        // スタミナチェック
        var staminaInfo = GameManager.Data.GetStaminaInfo(_ownerHash);
        if (staminaInfo.currentStamina < _feintStaminaCost)
            return false;

        // --- フェイント実行 ---

        // 1. 現在の攻撃をキャンセル
        if (_currentWeapon != null)
        {
            _currentWeapon.CancelAttack();
        }

        // 2. コンボリセット
        _comboSystem.Reset();

        // 3. スタミナ消費
        staminaInfo.currentStamina -= _feintStaminaCost;
        GameManager.Data.SetStaminaInfo(_ownerHash, staminaInfo);

        // 4. アニメーション状態を復帰硬直に遷移
        var newAnimState = new AnimationStateData
        {
            currentPhase = AnimationPhase.Recovery,  // 短い復帰硬直
            normalizedTime = (half)0f,
            framesUntilActionable = (short)_feintRecoveryFrames,
            currentMoveId = 0,
            isCancelable = false,
            isCommitted = false
        };
        GameManager.Data.SetAnimationStateData(_ownerHash, newAnimState);

        // 5. イベント発火（AI のエピソード記憶等で使用）
        AIManager.Instance.RaiseBrainEvent(
            _ownerHash,
            BrainEventFlagType.FeintExecuted
        );

        return true;
    }
}
```

### 11.3 攻撃実行フローへの組み込み

既存の攻撃実行フロー（セクション7）に対し、Anticipation フェーズ中にフェイントキャンセルの分岐が追加される。

```

  │
  ▼

  │
  ▼

  │  ※ isCancelable = true（cancelWindow 内）
  │
  ├──[フェイント入力 / AI FeintCancel 判定]──→ TryFeintCancel()
  │                                              │
  │                                              ▼
  │                                         攻撃キャンセル
  │                                         コンボリセット
  │                                         短い復帰硬直（4F）
  │                                              │

  │

  │  ※ 以降は通常攻撃フロー
  ▼

```

### 11.4 AI によるフェイント使用

AI判定内でフェイントキャンセルは `ActType.FeintCancel` として選択される。アクションマスキング（05_AIシステム.md セクション10参照）により、Anticipation フェーズ以外では自動的に無効化される。

```csharp
// AI判定処理内 — フェイント判定（概念コード）
{
    // フェイントは Anticipation 中のみ有効（アクションマスキングで保証）
    // 性格ベクトルがフェイント傾向に影響
    float feintBias = personalityData.riskTolerance * 0.5f
                    + (1.0f - personalityData.aggression) * 0.3f;

    // 状況要因
    bool enemyIsGuarding = targetStateInfo.HasFlag(CharacterStateFlag.Guard);
    bool hasFrameDisadvantage = frameAdvantage < -0.1f;

    // ガード中の相手にはフェイント有効（ガード解除を誘う）
    if (enemyIsGuarding)
        feintBias += 0.2f;

    // フレーム不利な場合はフェイント→退避が有効
    if (hasFrameDisadvantage)
        feintBias += 0.15f;

    // バイアス + PersonalityData で最終判定
    // ※ ルールベースでフェイントの有効性を評価
}
```

**性格ベクトルとの関係**:

| PersonalityData フィールド | フェイントへの影響 |
|---|---|
| `riskTolerance` 高 | フェイント使用頻度増（駆け引き重視） |
| `aggression` 低 | フェイント傾向増（慎重なスタイル） |
| `comboCommitment` 低 | フェイント傾向増（コミットせず様子見） |
| `distancePreference` 高 | フェイント→退避の組み合わせ傾向増 |

### 11.5 フェイントと対戦の駆け引き

フェイントシステムは以下の心理的駆け引きを生み出す。

```
攻撃者                            防御者
  │                                │
  ├── Anticipation 開始 ──────→ 回避/ガード準備
  │                                │
  ├── [A] そのまま攻撃 ──────→ 回避成功 → 攻撃者不利
  │                                │
  └── [B] FeintCancel ───────→ 回避空振り → 防御者硬直
        └── 即座に再攻撃 ────→ 被弾
```

**対戦への効果**:
- 「予備動作を見てから回避」の安定行動を崩す
- 相手の反応パターン（即回避 / 待ち / 読み）に対する適応が生まれる

### 13.5 押し出しシステム

攻撃移動中にキャラクター同士が重なった場合の処理。

```
押し出し判定:
  攻撃移動中にヒットボックスがキャラクターに接触した場合:

  ① ContactType.Normal の場合:
     攻撃者の移動を停止し、押し出しなし

  ② ContactType.Through の場合:
     攻撃者は移動を継続。接触した敵にダメージを適用し、
     被弾者を hitbox の外に押し出す。
     押し出し方向 = normalize(defender.position - attacker.position)
     押し出し距離 = hitbox.size / 2 + characterRadius

  ③ ContactType.Explosive の場合:
     接触点を中心に範囲ダメージ。攻撃者は移動を停止。
     範囲内の全キャラクターにダメージを適用。
```

### 14.6 Job System による並列移動処理

```csharp
/// <summary>
/// 弾丸移動の Job。IJobParallelFor で全弾丸を並列処理する。
/// </summary>
[BurstCompile]
public struct ProjectileMoveJob : IJobParallelFor
{
    public NativeArray<ProjectileInstance> projectiles;
    [ReadOnly] public NativeArray<float2> targetPositions; // 追尾用
    public float deltaTime;

    public void Execute(int index)
    {
        var p = projectiles[index];
        if (p.state != ProjectileState.Active) return;

        // 軌道タイプ別の速度計算
        float2 newVelocity = p.trajectory switch
        {
            TrajectoryType.Straight  => p.velocity,
            TrajectoryType.Parabolic => CalculateParabolic(p),
            TrajectoryType.Homing    => CalculateHoming(p, targetPositions),
            TrajectoryType.Wave      => CalculateWave(p),
            TrajectoryType.Boomerang => CalculateBoomerang(p),
            TrajectoryType.HomingZ   => CalculateHomingZ(p, targetPositions),
            TrajectoryType.Rain      => CalculateRain(p),
            TrajectoryType.Stop      => CalculateStop(p),
            _ => p.velocity,
        };

        p.velocity = newVelocity;
        p.position += p.velocity * deltaTime;
        p.remainingLifetime -= deltaTime;
        p.elapsedTime += deltaTime;

        if (p.remainingLifetime <= 0f)
            p.state = ProjectileState.Deactivating;

        projectiles[index] = p;
    }

    // --- 軌道計算メソッド（省略: 各タイプの物理演算） ---
}
```

### 14.8 ProjectileManager 処理フロー

```
┌─ ProjectileManager.Update() ──────────────────────────────────┐
│                                                                │
│  1. ターゲット位置キャッシュ更新                               │
│     追尾弾のターゲット位置を NativeArray に書き込み            │
│                                                                │
│  2. Job スケジュール                                           │
│     ProjectileMoveJob をスケジュール                           │
│     （全弾丸の位置・速度を並列更新）                           │
│                                                                │
│  3. Job 完了待ち                                               │
│     jobHandle.Complete()                                       │
│                                                                │
│  4. 衝突判定（メインスレッド）                                 │
│     for each active projectile:                                │
│       Physics2D.OverlapCircle で衝突チェック                   │
│       → ヒット: DamageReceiver.ApplyDamage()                  │
│       → 貫通チェック: pierceRemaining--                        │
│       → 範囲ダメージ: aoeRadius > 0 でエリアダメージ           │
│       → 子弾生成: OnHit トリガーで子弾をスポーン               │
│       → 支援弾の場合: SupportEffectInfo を味方に適用           │
│                                                                │
│  5. ライフタイム切れ処理                                       │
│     for each deactivating projectile:                          │
│       → 子弾生成: OnExpire トリガーで子弾をスポーン            │
│       → 着弾エフェクト再生                                     │
│       → プールに返却                                           │
│                                                                │
│  6. GameObject 同期                                            │
│     NativeArray の位置データを Transform に書き戻し            │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```
